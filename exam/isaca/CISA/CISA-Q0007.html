To preserve the integrity of a message, the message is passed through an algorithm called a cryptographic hash function. The function creates a compressed image of the message, called a digest. To check the integrity of a message, the receiver runs the cryptographic hash function again and compares the new digest with the previous one.====>Underline Integrity and digest which is an image If both are the same, the receiver is sure that the original message has not been changed. To ensure message integrity and message origin authentication, Message Authentication Code (MAC) is created using a combination of hash function and secret key. A sender sends the message and the MAC to a receiver over the insecure channel. The receiver separates the message from the MAC. He then makes a new MAC from the message and the secret key. Bob then compares the newly created MAC with the one received. If the two MACs match, the message is authentic and has not been modiï¬ed by an adversary.

To confirm the integrity of a hashed message, the receiver should generate a hash value of the received message using the same hashing algorithm used by the sender. This will create a numerical representation of the file, known as a hash value or message digest. The receiver can then compare this hash value to the hash value generated by the sender. If the hash values match, it indicates that the message has not been tampered with and the integrity of the message has been maintained. Using a different hashing algorithm or creating a binary image of the file may not result in a meaningful comparison between the hash values generated by the sender and receiver, and will not provide assurance of message integrity.
