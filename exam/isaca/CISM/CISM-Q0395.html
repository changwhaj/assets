  • A. Set targets against which the organization's information security function can be evaluated.
  • B. Prevent potential undesirable events from affecting information security.
  • C. Identify risk events that have already occurred from affecting information security.
  • D. Establish the process for setting organizational objectives in light of information security risk.

A. 조직의 정보 보안 기능을 평가할 수 있는 목표를 설정합니다. 정보 보안 위험 관리의 핵심 성과 지표(KPI)는 주로 조직의 정보 보안 기능을 평가할 수 있는 목표 또는 벤치마크를 설정하는 목적으로 사용됩니다. KPI는 조직이 보안 상태를 평가하고, 진행 상황을 추적하고, 보안 조치 개선에 대해 정보에 입각한 결정을 내리는 데 도움이 되는 측정 가능한 지표와 목표를 제공합니다. 바람직하지 않은 이벤트(옵션 B 및 C)를 예방하고 식별하는 것이 정보 보안의 중요한 목표인 반면, KPI의 주요 이점은 평가 및 지속적인 개선을 위한 프레임워크를 구축하는 것입니다. 옵션 D는 정보 보안 위험 관리에서 KPI의 주요 이점과 직접적인 관련이 없습니다.

A. Set targets against which the organization's information security function can be evaluated. Key performance indicators (KPIs) in information security risk management primarily serve the purpose of setting targets or benchmarks against which an organization's information security function can be evaluated. KPIs provide measurable metrics and goals that help organizations assess their security posture, track progress, and make informed decisions about improving security measures. While preventing and identifying undesirable events (options B and C) are important goals of information security, the primary benefit of KPIs is to establish a framework for evaluation and continuous improvement. Option D is not directly related to the primary benefit of KPIs in information security risk management
