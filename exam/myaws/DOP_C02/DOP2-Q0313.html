질문 #: 313
주제 #: 1

보안팀은 Amazon EventBridge가 특정 이벤트와 일치할 때 AWS Step Functions 워크플로를 호출하는 워크플로를 설정합니다. 이벤트는 여러 AWS 서비스에서 생성될 수 있습니다. AWS CloudTrail은 사용자 활동을 기록합니다.

보안팀은 일부 중요한 이벤트가 예상대로 워크플로를 호출하지 않는다는 것을 알아차립니다. CloudTrail 로그는 누락된 이벤트와 관련된 직접적인 오류를 나타내지 않습니다.

누락된 이벤트 호출의 근본 원인을 식별할 단계 조합은 무엇입니까? (세 가지를 선택하십시오.)

  • A. 이벤트 버스에서 EventBridge 스키마 검색을 활성화하여 이벤트 패턴이 예상 스키마와 일치하는지 확인합니다. 가장 많이 투표된
  • B. Amazon CloudWatch를 구성하여 EventBridge 메트릭과 Step Functions 메트릭을 모니터링합니다. 이벤트 패턴과 워크플로 호출의 이상에 대한 알림을 설정합니다. 가장 많이 투표된
  • C. EventBridge에서 이벤트를 모니터링하고 기록하여 처리된 이벤트에 대한 자세한 정보를 제공하는 AWS Lambda 로깅 ​​함수를 구성합니다.
  • D. 누락된 이벤트 호출과 연관이 있을 수 있는 실패 또는 시간 초과 패턴을 확인하기 위해 Step Functions 실행 기록을 검토합니다.
  • E. 규칙에 연결된 IAM 실행 역할에 충분한 권한이 있는지 확인하기 위해 EventBridge 실패한 호출에 대한 메트릭을 검토합니다. 가장 많이 투표된
  • F. EventBridge에서 Step Functions 워크플로를 호출하는 데 필요한 올바른 권한이 있는지 확인합니다.

B: Amazon CloudWatch를 구성하여 EventBridge와 Step Functions의 메트릭을 모니터링하고, 이상 탐지를 위한 알림을 설정합니다. 이를 통해 이벤트 패턴과 워크플로 호출의 이상을 실시간으로 감지할 수 있습니다.

D: Step Functions 실행 기록을 검토하여 실패 또는 시간 초과 패턴을 확인합니다. 이러한 기록을 통해 누락된 이벤트와 관련된 문제를 식별할 수 있습니다.

E: EventBridge 실패한 호출에 대한 메트릭을 검토하여, IAM 실행 역할에 충분한 권한이 있는지 확인합니다. 이는 이벤트가 Step Functions 워크플로를 호출할 수 있도록 필요한 권한이 설정되었는지 확인하는 데 중요합니다.

이 세 가지 방법은 누락된 이벤트 호출의 근본 원인을 파악하는 데 가장 효율적인 접근 방식입니다.

A, B, E도 유효한 선택일 수 있습니다. 아래에서 각 옵션에 대해 설명드리겠습니다.
A: IAM 역할 및 권한 문제를 확인하는 방법입니다. 만약 EventBridge와 Step Functions 간의 연결이 실패하는 이유가 권한 부족이라면, IAM 역할에서 필요한 권한이 부여되지 않았을 가능성이 있으므로 이 방법은 유효합니다. IAM 역할에 대한 세부 권한을 확인하고 누락된 권한을 추가하는 것이 중요합니다.
B: CloudWatch 메트릭 및 로그를 사용하여 EventBridge 및 Step Functions의 이상 징후를 모니터링하는 방법입니다. 이 방법은 실시간 모니터링 및 알림을 설정하여 문제를 빠르게 식별하고, 이상 패턴을 추적할 수 있습니다. 이는 디버깅 및 문제 해결에 유용한 접근법입니다.
E: EventBridge와 Step Functions 간의 IAM 권한을 검토하고, 필요한 권한이 누락되지 않았는지 확인하는 방법입니다. 권한 문제로 인해 EventBridge가 Step Functions를 호출하지 못하는 경우를 해결하는 데 중요한 부분입니다.

결론: A, B, E는 함께 사용하여 권한 문제를 해결하고, CloudWatch로 모니터링하여 이상 징후를 감지하는 방식으로 문제를 해결하는 데 도움이 될 수 있습니다.
이 세 가지 방법은 효과적인 문제 해결 및 모니터링을 위한 전략입니다.

Question #: 313
Topic #: 1

A security team sets up a workflow that invokes an AWS Step Functions workflow when Amazon EventBridge matches specific events. The events can be generated by several AWS services. AWS CloudTrail records user activities.

The security team notices that some important events do not invoke the workflow as expected. The CloudTrail logs do not indicate any direct errors related to the missing events.

Which combination of steps will identify the root cause of the missing event invocations? (Choose three.)

  • A. Enable EventBridge schema discovery on the event bus to determine whether the event patterns match the expected schema. Most Voted
  • B. Configure Amazon CloudWatch to monitor EventBridge metrics and Step Functions metrics. Set up alerts for anomalies in event patterns and workflow invocations. Most Voted
  • C. Configure an AWS Lambda logging function to monitor and log events from EventBridge to provide more details about the processed events.
  • D. Review the Step Functions execution history for patterns of failures or timeouts that could correlate to the missing event invocations.
  • E. Review metrics for the EventBridge failed invocations to ensure that the IAM execution role that is attached to the rule has sufficient permissions. Most Voted
  • F. Verify that the Step Functions workflow has the correct permissions to be invoked by EventBridge.
