Amazon의 AWS Certified DevOps Engineer - Professional DOP-C02 의 시험 문제
질문 #: 327
주제 #: 1

DevOps 엔지니어는 AWS CodePipeline에서 파이프라인을 사용합니다. 파이프라인에는 Amazon S3 버킷에 전달되는 단일 페이지 웹 애플리케이션에 대한 빌드 작업과 배포 작업이 있습니다. Amazon CloudFront가 웹 애플리케이션을 제공합니다. 빌드 작업은 웹 애플리케이션에 대한 아티팩트를 만듭니다.

DevOps 엔지니어는 S3 버킷을 정의하고 애플리케이션을 호스팅하도록 S3 버킷을 구성하는 AWS CloudFormation 템플릿을 만들었습니다. DevOps 엔지니어는 S3 작업 전에 CloudFormation 배포 작업을 구성했습니다. CloudFormation 배포 작업은 S3 버킷을 만듭니다. DevOps 엔지니어는 CloudFormation 템플릿의 S3 버킷을 사용하도록 S3 배포 작업을 구성해야 합니다.

이러한 요구 사항을 충족하는 단계 조합은 무엇입니까? (두 가지를 선택하십시오.)

  • A. CloudFormation 템플릿에 BucketName이라는 이름의 출력을 추가합니다. 출력 값을 CloudFormation 템플릿의 S3 버킷을 참조하도록 설정합니다. 출력 값을 Stackvariables라는 이름의 AWS::SSM::Parameter 리소스로 내보내도록 구성합니다.
  • B. CloudFormation 템플릿에 BucketName이라는 이름의 출력을 추가합니다. 출력의 값을 CloudFormation 템플릿의 S3 버킷을 참조하도록 설정합니다. 파이프라인에서 CloudFormation 작업의 네임스페이스를 StackVariables로 설정합니다. 가장 많이 투표된
  • C. 파이프라인에서 CloudFormation 작업의 출력 아티팩트를 StackVariables라는 이름의 AWS Systems Manager Parameter Store 매개변수로 구성합니다. 아티팩트 이름을 BucketName으로 지정합니다.
  • D. 빌드 작업의 빌드 아티팩트를 CodePipeline S3 배포 작업의 입력으로 구성합니다. StackVariables.BucketName 변수를 사용하여 배포 작업을 S3 버킷에 배포하도록 구성합니다. 가장 많이 투표된
  • E. 빌드 작업의 빌드 아티팩트와 AWS Systems Manager 매개변수를 배포 작업의 입력으로 구성합니다. StackVariables.BucketName 변수를 사용하여 S3 버킷에 배포하도록 배포 작업을 구성합니다.

제안된 답변: BD (OK)
B: CloudFormation 템플릿에서 BucketName이라는 출력을 추가하고, S3 버킷을 참조하는 값을 출력합니다. 이 출력 값을 사용하여 CloudFormation 작업의 네임스페이스를 StackVariables로 설정하면, 이후 작업에서 이 값을 사용할 수 있습니다.

D: 빌드 작업의 빌드 아티팩트를 S3 배포 작업의 입력으로 설정하고, StackVariables.BucketName 변수를 사용하여 배포 작업에서 동적으로 S3 버킷을 지정하는 방법입니다. 이를 통해 CloudFormation 템플릿에서 생성된 버킷을 사용하여 아티팩트를 배포할 수 있습니다.

이 두 방법은 AWS CodePipeline에서 CloudFormation과 S3 배포 작업을 연결하는 데 가장 적합한 해결책입니다.

Exam question from Amazon's AWS Certified DevOps Engineer - Professional DOP-C02
Question #: 327
Topic #: 1

A DevOps engineer uses a pipeline in AWS CodePipeline. The pipeline has a build action and a deploy action for a single-page web application that is delivered to an Amazon S3 bucket. Amazon CloudFront serves the web application. The build action creates an artifact for the web application.

The DevOps engineer has created an AWS CloudFormation template that defines the S3 bucket and configures the S3 bucket to host the application. The DevOps engineer has configured a CloudFormation deploy action before the S3 action. The CloudFormation deploy action creates the S3 bucket. The DevOps engineer needs to configure the S3 deploy action to use the S3 bucket from the CloudFormation template.

Which combination of steps will meet these requirements? (Choose two.)

  • A. Add an output named BucketName to the CloudFormation template. Set the output's value to refer to the S3 bucket from the CloudFormation template. Configure the output value to export to an AWS::SSM::Parameter resource named Stackvariables.
  • B. Add an output named BucketName to the CloudFormation template. Set the output's value to refer to the S3 bucket from the CloudFormation template. Set the CloudFormation action's namespace to StackVariables in the pipeline. Most Voted
  • C. Configure the output artifacts of the CloudFormation action in the pipeline to be an AWS Systems Manager Parameter Store parameter named StackVariables. Name the artifact BucketName.
  • D. Configure the build artifact from the build action as the input to the CodePipeline S3 deploy action. Configure the deploy action to deploy to the S3 bucket by using the StackVariables.BucketName variable. Most Voted
  • E. Configure the build artifact from the build action and the AWS Systems Manager parameter as the inputs to the deploy action. Configure the deploy action to deploy to the S3 bucket by using the StackVariables.BucketName variable.
