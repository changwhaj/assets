Amazon의 AWS Certified DevOps Engineer - Professional DOP-C02 의 시험 문제
질문 #: 351
주제 #: 1

한 회사가 AWS Organizations에서 조직을 사용하여 계층 구조에서 여러 AWS 계정을 관리합니다. 조직 루트와 연결된 SCP를 통해 IAM 사용자를 만들 수 있습니다.

DevOps 팀은 모든 수준의 권한을 가진 IAM 사용자를 만들 수 있어야 합니다. 개발자도 IAM 사용자를 만들 수 있어야 합니다. 그러나 개발자는 새 IAM 사용자에게 과도한 권한을 부여할 수 없어야 합니다. 개발자는 각 계정에서 CreateAndManageUsers 역할을 갖습니다. DevOps 팀은 다른 사용자가 IAM 사용자를 만드는 것을 방지할 수 있어야 합니다.

이러한 요구 사항을 충족하는 단계의 조합은 무엇입니까? (두 가지를 선택하십시오.)

  • A. 조직에서 SCP를 만들어 사용자가 IAM 사용자를 만들고 수정하는 기능을 거부합니다. SCP를 조직의 루트에 연결합니다. CreateAndManageUsers 역할을 개발자에게 연결합니다.
  • B. 조직에서 SCP를 만들어 DeveloperBoundary 정책이 첨부된 사용자에게 새 IAM 사용자를 만들고 IAM 사용자를 수정할 수 있는 권한을 부여합니다. 사용자가 모든 새 IAM 사용자에게 PermissionBoundaries 정책을 첨부하도록 SCP를 구성합니다. SCP를 조직의 루트에 첨부합니다.
  • C. 각 계정 내에 PermissionBoundaries라는 이름의 IAM 권한 정책을 만듭니다. PermissionBoundaries 정책을 구성하여 개발자가 새 IAM 사용자에게 부여할 수 있는 최대 권한을 지정합니다. 가장 많이 투표된
  • D. 각 계정 내에 PermissionBoundaries라는 이름의 IAM 권한 정책을 만듭니다. PermissionBoundaries 정책을 가진 사용자가 새 IAM 사용자를 만들 수 있도록 PermissionsBoundaries를 구성합니다.
  • E. 각 계정 내에 DeveloperBoundary라는 이름의 IAM 권한 정책을 만듭니다. DeveloperBoundary 정책을 구성하여 개발자가 IAM 사용자를 만들고 개발자가 권한 경계로 PermissionBoundaries 정책을 포함하는 경우에만 IAM 사용자에게 정책을 할당할 수 있도록 합니다. 각 계정 내에서 DeveloperBoundary 정책을 CreateAndManageUsers 역할에 연결합니다. 가장 많이 투표된

C는 각 계정 내에서 PermissionBoundaries라는 이름의 IAM 권한 정책을 생성하고, 이를 개발자가 새 IAM 사용자에게 부여할 수 있는 최대 권한을 정의하는 데 사용합니다. 이 정책은 개발자가 권한을 제한하는 방식으로, 과도한 권한 부여를 방지하는 데 유효합니다.

E는 DeveloperBoundary라는 이름의 IAM 권한 정책을 생성하고, 이를 사용하여 개발자가 새 IAM 사용자를 생성할 때 PermissionBoundaries를 권한 경계로 사용하도록 강제합니다. 이 방법은 개발자가 적절한 권한만을 부여할 수 있도록 하여, 과도한 권한을 방지할 수 있습니다.

이 두 가지 방법을 결합하여 개발자에게 IAM 사용자 생성 권한을 부여하면서도 권한의 범위를 제한할 수 있습니다.

Exam question from Amazon's AWS Certified DevOps Engineer - Professional DOP-C02
Question #: 351
Topic #: 1

A company uses an organization in AWS Organizations to manage multiple AWS accounts in a hierarchical structure. An SCP that is associated with the organization root allows IAM users to be created.

A DevOps team must be able to create IAM users with any level of permissions. Developers must also be able to create IAM users. However, developers must not be able to grant new IAM users excessive permissions. The developers have the CreateAndManageUsers role in each account. The DevOps team must be able to prevent other users from creating IAM users.

Which combination of steps will meet these requirements? (Choose two.)

  • A. Create an SCP in the organization to deny users the ability to create and modify IAM users. Attach the SCP to the root of the organization. Attach the CreateAndManageUsers role to developers.
  • B. Create an SCP in the organization to grant users that have the DeveloperBoundary policy attached the ability to create new IAM users and to modify IAM users. Configure the SCP to require users to attach the PermissionBoundaries policy to any new IAM user. Attach the SCP to the root of the organization.
  • C. Create an IAM permissions policy named PermissionBoundaries within each account. Configure the PermissionBoundaries policy to specify the maximum permissions that a developer can grant to a new IAM user. Most Voted
  • D. Create an IAM permissions policy named PermissionBoundaries within each account. Configure PermissionsBoundaries to allow users who have the PermissionBoundaries policy to create new IAM users.
  • E. Create an IAM permissions policy named DeveloperBoundary within each account. Configure the DeveloperBoundary policy to allow developers to create IAM users and to assign policies to IAM users of only if the developer includes the PermissionBoundaries policy as the permissions boundary. Attach the DeveloperBoundary policy to the CreateAndManageUsers role within each account. Most Voted

A would prevent anyone to create IAM users, so both DevOps teams and Developers cannot create IAM users. B would prevent DevOps team to create IAM users "with any level of permissions". C would create the permission boundary that defines the maximum permissions of a user created by the Developers. D does not work like that. E would give the Developers the permissions to create users, but would force them to also attach the permission boundary (created in C) to the new user, limiting their permissions correctly (even if the Developer would give that user too many permissions)
