Amazon의 AWS Certified DevOps Engineer - Professional DOP-C02 의 시험 문제
질문 #: 349
주제 #: 1

한 회사가 Amazon Elastic Kubernetes Service(Amazon EKS)에서 웹 애플리케이션을 실행합니다. 이 회사는 Amazon CloudFront를 사용하여 애플리케이션을 배포합니다. 이 회사는 최근 AWS WAF를 활성화했습니다. 이 회사는 Amazon CloudWatch Logs를 설정하여 aws-waf-logs 로그 그룹으로 로그를 전송합니다.

이 회사는 차단된 트래픽에 갑작스러운 변화가 있는 경우 DevOps 엔지니어가 알림을 받길 원합니다. 이 회사는 AWS WAF 로그 동작의 다른 변화에 대한 알림을 받고 싶어하지 않습니다. 이 회사는 시간이 지남에 따라 AWS WAF 규칙을 조정할 것입니다.

DevOps 엔지니어는 현재 환경에서 Amazon Simple Notification Service(Amazon SNS) 토픽을 구독하고 있습니다.

어떤 솔루션이 이러한 요구 사항을 충족할까요?

  • A. AWS WAF 로그 그룹에서 차단된 요청에 대한 CloudWatch Logs 메트릭 필터를 만들어 사용자 지정 메트릭을 만듭니다. CloudWatch 이상 탐지 및 게시된 사용자 지정 메트릭을 사용하여 CloudWatch 알람을 만듭니다. SNS 토픽에 DevOps 엔지니어에게 경고하도록 알람을 구성합니다. 가장 많이 투표된
  • B. 로그 그룹에 대한 CloudWatch 이상 감지기를 만듭니다. CloudWatch 이상 감지기가 게시하는 메트릭을 사용하여 CloudWatch 알람을 만듭니다. LogAnomalyPriority 메트릭에 대해 높은 설정을 사용합니다. 정적 임계값인 이상 하나가 감지되면 알람이 알람 상태로 전환되도록 구성합니다. DevOps 엔지니어에게 경고하도록 SNS 토픽에 알리도록 알람을 구성합니다.
  • C. AWS WAF 로그 그룹에서 계산된 요청에 대한 CloudWatch 메트릭 필터를 만들어 사용자 지정 메트릭을 만듭니다. 1시간 동안 사용자 지정 메트릭에서 차단된 요청의 합계가 1시간 동안 허용 가능한 차단된 요청 수에 대한 정적 추정치보다 클 때 활성화되는 CloudWatch 알람을 만듭니다. DevOps 엔지니어에게 경고하도록 SNS 토픽에 알리도록 알람을 구성합니다.
  • D. 로그 그룹에 대한 CloudWatch 이상 탐지기를 만듭니다. CloudWatch 이상 탐지기가 게시하는 메트릭을 사용하여 CloudWatch 알람을 만듭니다. LogAnomalyPriority 메트릭에 대해 medium 설정을 사용합니다. 1시간 동안 이상 탐지의 합이 예상 값보다 클 경우 알람이 알람 상태로 전환되도록 구성합니다. DevOps 엔지니어에게 경고하도록 SNS 토픽에 알리도록 알람을 구성합니다.
제안된 답변 보기 답변 숨기기
제안된 답변: A (OK)
A는 CloudWatch Logs 메트릭 필터를 사용하여 차단된 요청에 대해 사용자 지정 메트릭을 생성하고, 이를 CloudWatch 이상 탐지와 결합하여 차단된 트래픽의 갑작스러운 변화를 감지하는 방법입니다. 이 방법은 AWS WAF 로그 동작에서 예상치 못한 변화를 감지하고, 이를 SNS를 통해 DevOps 엔지니어에게 알림을 보낼 수 있게 해줍니다.

B는 이상 탐지를 사용하지만, 요구 사항에서 다른 WAF 로그 동작의 변화에는 알림을 원하지 않기 때문에 적합하지 않습니다.
C와 D는 정적 임계값을 사용하여 알림을 설정하지만, 변화 감지와 동적 알림에는 이상 탐지가 더 적합합니다.
따라서 A는 가장 적합한 해결책입니다.

Exam question from Amazon's AWS Certified DevOps Engineer - Professional DOP-C02
Question #: 349
Topic #: 1

A company runs a web application on Amazon Elastic Kubernetes Service (Amazon EKS). The company uses Amazon CloudFront to distribute the application. The company recently enabled AWS WAF. The company set up Amazon CloudWatch Logs to send logs to an aws-waf-logs log group.

The company wants a DevOps engineer to receive alerts if there are sudden changes in blocked traffic. The company does not want to receive alerts for other changes in AWS WAF log behavior. The company will tune AWS WAF rules over time.

The DevOps engineer is currently subscribed to an Amazon Simple Notification Service (Amazon SNS) topic in the environment.

Which solution will meet these requirements?

  • A. Create a CloudWatch Logs metrics filter for blocked requests on the AWS WAF log group to create a custom metric. Create a CloudWatch alarm by using CloudWatch anomaly detection and the published custom metric. Configure the alarm to notify the SNS topic to alert the DevOps engineer. Most Voted
  • B. Create a CloudWatch anomaly detector for the log group. Create a CloudWatch alarm by using metrics that the CloudWatch anomaly detector publishes. Use the high setting for the LogAnomalyPriority metric. Configure the alarm to go into alarm state if a static threshold of one anomaly is detected. Configure the alarm to notify the SNS topic to alert the DevOps engineer.
  • C. Create a CloudWatch metrics filter for counted requests on the AWS WAF log group to create a custom metric. Create a CloudWatch alarm that activates when the sum of blocked requests in the custom metric during a period of 1 hour is greater than a static estimate for the acceptable number of blocked requests in 1 hour. Configure the alarm to notify the SNS topic to alert the DevOps engineer.
  • D. Create a CloudWatch anomaly detector for the log group. Create a CloudWatch alarm by using metrics that the CloudWatch anomaly detector publishes. Use the medium setting for the LogAnomalyPriority metric. Configure the alarm to go into alarm state if a sum of anomalies over 1 hour is greater than an expected value. Configure the alarm to notify the SNS topic to alert the DevOps engineer.
