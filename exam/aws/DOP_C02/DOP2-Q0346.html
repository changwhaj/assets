Amazon의 AWS Certified DevOps Engineer - Professional DOP-C02 의 시험 문제
질문 #: 346
주제 #: 1

DevOps 팀은 Auto Scaling 그룹의 많은 수의 Amazon EC2 인스턴스에서 실행되는 애플리케이션을 지원합니다. DevOps 팀은 AWS CloudFormation을 사용하여 EC2 인스턴스를 배포합니다. 애플리케이션에서 최근에 문제가 발생했습니다. 단일 인스턴스에서 많은 비율의 요청에 오류가 반환되었습니다. EC2 인스턴스는 Amazon EC2와 Elastic Load Balancing 상태 검사 모두에서 정상으로 응답했습니다.

DevOps 팀은 임베디드 메트릭 형식을 사용하여 Amazon CloudWatch에서 애플리케이션 로그를 수집합니다. DevOps 팀은 모든 오류의 절반 이상을 담당하는 EC2 인스턴스가 있는 경우 알림을 받아야 합니다.

어떤 단계 조합이 이러한 요구 사항을 가장 적은 운영 오버헤드로 충족할 수 있을까요? (두 가지를 선택하세요.)

  • A. 인스턴스 ID와 오류를 기준으로 CloudWatch 애플리케이션 로그의 로그를 그룹화하는 CloudWatch Contributor Insights 규칙을 만듭니다. 가장 많이 투표된
  • B. AWS Resource Groups에서 리소스 그룹을 만듭니다. CloudFormation 스택을 사용하여 애플리케이션의 리소스를 그룹화합니다. 애플리케이션을 CloudWatch Application Insights에 추가합니다. 리소스 그룹을 사용하여 애플리케이션을 식별합니다.
  • C. 애플리케이션 로그에 대한 메트릭 필터를 만들어 "Error"라는 용어의 발생 횟수를 계산합니다. 오류가 발생했는지 여부를 확인하기 위해 METRIC_COUNT 함수를 사용하는 CloudWatch 알람을 만듭니다. DevOps 팀에 알리기 위해 Amazon Simple Notification Service(Amazon SNS) 토픽에 알림을 보내도록 CloudWatch 알람을 구성합니다.
  • D. INSIGHT_RULE_METRIC 함수를 사용하여 특정 인스턴스가 EC2 인스턴스에서 보고된 모든 오류의 절반 이상을 담당하는지 여부를 확인하는 CloudWatch 알람을 만듭니다. DevOps 팀에 알리기 위해 Amazon Simple Notification Service(Amazon SNS) 토픽에 알림을 보내도록 CloudWatch 알람을 구성합니다. 가장 많이 투표된
  • E. 오류를 필터링하고 AWS Lambda 함수를 호출하는 애플리케이션 로그에 대한 CloudWatch 구독 필터를 만듭니다. Lambda 함수를 구성하여 인스턴스 ID와 오류를 Amazon Simple Notification Service(Amazon SNS) 토픽에 알림으로 보내 DevOps 팀에 알립니다.

Exam question from Amazon's AWS Certified DevOps Engineer - Professional DOP-C02
Question #: 346
Topic #: 1

A DevOps team supports an application that runs on a large number of Amazon EC2 instances in an Auto Scaling group. The DevOps team uses AWS CloudFormation to deploy the EC2 instances. The application recently experienced an issue. A single instance returned errors to a large percentage of requests. The EC2 instance responded as healthy to both Amazon EC2 and Elastic Load Balancing health checks.

The DevOps team collects application logs in Amazon CloudWatch by using the embedded metric format. The DevOps team needs to receive an alert if any EC2 instance is responsible for more than half of all errors.

Which combination of steps will meet these requirements with the LEAST operational overhead? (Choose two.)

  • A. Create a CloudWatch Contributor Insights rule that groups logs from the CloudWatch application logs based on instance ID and errors. Most Voted
  • B. Create a resource group in AWS Resource Groups. Use the CloudFormation stack to group the resources for the application. Add the application to CloudWatch Application Insights. Use the resource group to identify the application.
  • C. Create a metric filter for the application logs to count the occurrence of the term "Error.'' Create a CloudWatch alarm that uses the METRIC_COUNT function to determine whether errors have occurred. Configure the CloudWatch alarm to send a notification to an Amazon Simple Notification Service (Amazon SNS) topic to notify the DevOps team.
  • D. Create a CloudWatch alarm that uses the INSIGHT_RULE_METRIC function to determine whether a specific instance is responsible for more than half of all errors reported by EC2 instances. Configure the CloudWatch alarm to send a notification to an Amazon Simple Notification Service (Amazon SNS) topic to notify the DevOps team. Most Voted
  • E. Create a CloudWatch subscription filter for the application logs that filters for errors and invokes an AWS Lambda function. Configure the Lambda function to send the instance ID and error and in a notification to an Amazon Simple Notification Service (Amazon SNS) topic to notify the DevOps team.
