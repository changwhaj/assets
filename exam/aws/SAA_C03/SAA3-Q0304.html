선택된 답변: A
• AWS DataSync는 NFS 파일 시스템 간에 대량의 데이터를 이동하도록 최적화된 데이터 전송 서비스입니다. 다른 AWS 리전에 있는 NFS 파일 시스템 간에 파일과 메타데이터를 자동으로 복사할 수 있습니다. • DataSync는 최소한의 설정과 관리가 필요합니다. 소스 및 대상 에이전트를 배포하고 소스 및 대상 위치를 제공하면 DataSync가 백그라운드에서 실제 데이터 전송을 효율적으로 처리합니다. • DataSync는 데이터 전송을 예약하고 모니터링하여 최소한의 오버헤드로 소스와 대상을 동기화 상태로 유지할 수 있습니다. 중단된 전송을 재개하고 데이터 무결성을 검증합니다. • DataSync는 AWS 네트워크 인프라에서 데이터 전송 성능을 최적화합니다. 운영에 미치는 영향을 최소화하면서 높은 처리량을 달성할 수 있습니다.

옵션 B - AWS Snowball은 데이터를 전송하기 위해 물리적 장치가 필요합니다. 이는 장치를 전송하고 데이터를 수동으로 로드/언로드하는 데 오버헤드를 발생시킵니다. 이는 온라인 데이터 전송 솔루션이 아닙니다. 옵션 C - SFTP 서버를 설정하고 관리하려면 EC2 인스턴스를 프로비저닝하고, 보안 그룹을 처리하고, 데이터 전송을 자동화하는 스크립트를 작성해야 합니다. 이 모든 작업은 DataSync보다 더 많은 오버헤드를 요구합니다. 옵션 D - AWS Database Migration Service는 일반 파일 시스템 데이터가 아닌 데이터베이스를 마이그레이션하도록 설계되었습니다. • AWS DataSync is a data transfer service optimized for moving large amounts of data between NFS file systems. It can automatically copy files and metadata between your NFS file systems in different AWS Regions. • DataSync requires minimal setup and management. You deploy a source and destination agent, provide the source and destination locations, and DataSync handles the actual data transfer efficiently in the background. • DataSync can schedule and monitor data transfers to keep source and destination in sync with minimal overhead. It resumes interrupted transfers and validates data integrity. • DataSync optimizes data transfer performance across AWS's network infrastructure. It can achieve high throughput with minimal impact to your operations.

Option B - AWS Snowball requires physical devices to transfer data. This incurs overhead to transport devices and manually load/unload data. It is not an online data transfer solution. Option C - Setting up and managing an SFTP server would require provisioning EC2 instances, handling security groups, and writing scripts to automate the data transfer - all of which demand more overhead than DataSync. Option D - AWS Database Migration Service is designed for migrating databases, not general file system data. It would require converting your NFS data into a database format, incurring additional overhead.
