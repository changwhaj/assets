주제 #: 1

한 회사가 Amazon Machine Images(AMI)를 관리하고자 합니다. 이 회사는 현재 AMI를 AMI가 생성된 동일한 AWS 리전에 복사합니다. 이 회사는 AWS API 호출을 캡처하고 회사 계정 내에서 Amazon EC2 CreateImage API 작업이 호출될 때마다 알림을 보내는 애플리케이션을 설계해야 합니다.

어떤 솔루션이 이러한 요구 사항을 최소한의 운영 오버헤드로 충족할까요?

  • A. AWS CloudTrail 로그를 쿼리하고 CreateImage API 호출이 감지되면 알림을 보내는 AWS Lambda 함수를 생성합니다.
  • B. 업데이트된 로그가 Amazon S3로 전송될 때 발생하는 Amazon Simple Notification Service(Amazon SNS) 알림으로 AWS CloudTrail을 구성합니다. Amazon Athena를 사용하여 새 테이블을 만들고 API 호출이 감지되면 CreateImage에서 쿼리합니다.
  • C. CreateImage API 호출에 대한 Amazon EventBridge(Amazon CloudWatch Events) 규칙을 만듭니다. CreateImage API 호출이 감지되면 알림을 보내도록 대상을 Amazon Simple Notification Service(Amazon SNS) 토픽으로 구성합니다. 가장 많이 투표된
  • D. AWS CloudTrail 로그의 대상으로 Amazon Simple Queue Service(Amazon SQS) FIFO 대기열을 구성합니다. CreateImage API 호출이 감지되면 Amazon Simple Notification Service(Amazon SNS) 토픽에 알림을 보내는 AWS Lambda 함수를 만듭니다.

Topic #: 1

A company wants to manage Amazon Machine Images (AMIs). The company currently copies AMIs to the same AWS Region where the AMIs were created. The company needs to design an application that captures AWS API calls and sends alerts whenever the Amazon EC2 CreateImage API operation is called within the company’s account.

Which solution will meet these requirements with the LEAST operational overhead?

  • A. Create an AWS Lambda function to query AWS CloudTrail logs and to send an alert when a CreateImage API call is detected.
  • B. Configure AWS CloudTrail with an Amazon Simple Notification Service (Amazon SNS) notification that occurs when updated logs are sent to Amazon S3. Use Amazon Athena to create a new table and to query on CreateImage when an API call is detected.
  • C. Create an Amazon EventBridge (Amazon CloudWatch Events) rule for the CreateImage API call. Configure the target as an Amazon Simple Notification Service (Amazon SNS) topic to send an alert when a CreateImage API call is detected. Most Voted
  • D. Configure an Amazon Simple Queue Service (Amazon SQS) FIFO queue as a target for AWS CloudTrail logs. Create an AWS Lambda function to send an alert to an Amazon Simple Notification Service (Amazon SNS) topic when a CreateImage API call is detected.

EventBridge(이전 명칭 CloudWatch Events)는 AWS 환경 내에서 이벤트를 모니터링하고 이에 대응할 수 있는 완전 관리형 이벤트 버스 서비스입니다. CreateImage API 호출에 대한 EventBridge 규칙을 만들면 이 이벤트를 쉽게 감지하고 캡처할 수 있습니다. 대상을 SNS 토픽으로 구성하면 CreateImage API 호출이 발생할 때마다 알림을 보낼 수 있습니다. EventBridge와 SNS는 완전 관리형 서비스이므로 이 솔루션은 최소한의 운영 오버헤드만 필요합니다. A. Lambda를 사용하여 CloudTrail 로그를 쿼리하고 알림을 보내면 원하는 결과를 얻을 수 있지만 EventBridge와 SNS를 직접 사용하는 것에 비해 추가적인 운영 오버헤드가 발생합니다. B. SNS 알림으로 CloudTrail을 구성하고 Athena를 사용하여 CreateImage API 호출을 쿼리하면 EventBridge와 SNS를 사용하는 것에 비해 더 많은 설정과 유지 관리가 필요합니다. D. CloudTrail 로그에 대한 대상으로 SQS FIFO 대기열을 구성하고 함수를 사용하여 SNS 토픽에 알림을 보내면 솔루션에 불필요한 복잡성이 추가되고 운영 오버헤드가 증가합니다. EventBridge와 SNS를 직접 사용하는 것이 더 간단하고 효율적인 방법입니다.
