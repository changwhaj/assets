As implied in the question, "The solution must be fully managed". Storage Gateway's SMB support is valid but not the best choice for this fully AWS-hosted and fully managed requirement. Storage Gateway requires the deployment of a gateway appliance (hardware or virtual). While AWS manages the back end (S3), setting up and maintaining the gateway appliance requires some operational effort, so it's not fully managed compared to FSx for Windows File Server. Storage Gateway is ideal for hybrid environments but adds unnecessary complexity if the workload is fully cloud-based. Not to mention that Storage Gateway File Gateway is focused on storing objects in S3, not providing a shared file system for direct collaboration or use by applications.
