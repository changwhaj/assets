Topic #: 1

A company uses 50 TB of data for reporting. The company wants to move this data from on premises to AWS. A custom application in the company’s data center runs a weekly data transformation job. The company plans to pause the application until the data transfer is complete and needs to begin the transfer process as soon as possible.
The data center does not have any available network bandwidth for additional workloads. A solutions architect must transfer the data and must configure the transformation job to continue to run in the AWS Cloud.
Which solution will meet these requirements with the LEAST operational overhead?

  • A. Use AWS DataSync to move the data. Create a custom transformation job by using AWS Glue.
  • B. Order an AWS Snowcone device to move the data. Deploy the transformation application to the device.
  • C. Order an AWS Snowball Edge Storage Optimized device. Copy the data to the device. Create a custom transformation job by using AWS Glue. Most Voted
  • D. Order an AWS Snowball Edge Storage Optimized device that includes Amazon EC2 compute. Copy the data to the device. Create a new EC2 instance on AWS to run the transformation application.

옵션 A(AWS Glue를 사용한 AWS DataSync)는 사용 가능한 네트워크 대역폭이 필요한 데이터 전송에 AWS DataSync를 사용하는 것을 포함합니다. 데이터 센터에는 추가 네트워크 대역폭이 없으므로 이 옵션은 적합하지 않습니다. 옵션 B(배포된 AWS Snowcone 디바이스)는 소규모 워크로드용으로 설계되었으며 50TB의 데이터를 전송하기에는 스토리지 용량이 충분하지 않을 수 있습니다. 또한 Snowcone 디바이스에 변환 애플리케이션을 배포하면 복잡성과 운영 오버헤드가 발생할 수 있습니다. 옵션 D(EC2 컴퓨팅이 포함된 AWS Snowball Edge)에는 Snowball Edge 디바이스를 사용하여 데이터를 전송한 다음 변환 애플리케이션을 실행하기 위해 AWS에서 새 EC2 인스턴스를 생성하는 작업이 포함됩니다. 이 옵션은 EC2 인스턴스 관리의 복잡성과 운영 오버헤드를 추가합니다. 이에 비해 옵션 C는 간단하고 효율적인 접근 방식을 제공합니다. Snowball Edge Storage Optimized 디바이스는 네트워크 대역폭에 의존하지 않고 대용량 데이터 전송을 처리할 수 있습니다. 데이터가 전송되면 AWS Glue를 사용하여 변환 작업을 생성하여 AWS 클라우드에서 애플리케이션 처리의 연속성을 보장할 수 있습니다.
