선택한 답변: A 기본적으로 ElastiCache의 Redis 노드에 있는 데이터는 메모리에만 상주하며 영구적이지 않습니다. 노드가 재부팅되거나 기본 물리적 서버에 하드웨어 오류가 발생하면 캐시의 데이터가 손실됩니다. 데이터 내구성이 필요한 경우 Redis 추가 전용 파일 기능(AOF)을 활성화할 수 있습니다. 이 기능을 활성화하면 노드가 캐시 데이터를 변경하는 모든 명령을 추가 전용 파일에 씁니다. 노드가 재부팅되고 캐시 엔진이 시작되면 AOF가 "재생"됩니다. 그 결과 모든 데이터가 그대로 유지된 따뜻한 Redis 캐시가 생성됩니다. AOF는 기본적으로 비활성화되어 있습니다. Redis를 실행하는 클러스터에 대해 AOF를 활성화하려면 appendonly 매개변수를 yes로 설정한 매개변수 그룹을 만들어야 합니다. 그런 다음 해당 매개변수 그룹을 클러스터에 할당합니다. appendfsync 매개변수를 수정하여 Redis가 AOF 파일에 쓰는 빈도를 제어할 수도 있습니다.

Selected Answer: A By default, the data in a Redis node on ElastiCache resides only in memory and isn't persistent. If a node is rebooted, or if the underlying physical server experiences a hardware failure, the data in the cache is lost. If you require data durability, you can enable the Redis append-only file feature (AOF). When this feature is enabled, the node writes all of the commands that change cache data to an append-only file. When a node is rebooted and the cache engine starts, the AOF is "replayed." The result is a warm Redis cache with all of the data intact. AOF is disabled by default. To enable AOF for a cluster running Redis, you must create a parameter group with the appendonly parameter set to yes. You then assign that parameter group to your cluster. You can also modify the appendfsync parameter to control how often Redis writes to the AOF file.
