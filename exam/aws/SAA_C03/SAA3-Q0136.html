A company is migrating its on-premises PostgreSQL database to Amazon Aurora PostgreSQL. The on-premises database must remain online and accessible during the migration. The Aurora database must remain synchronized with the on-premises database.
Which combination of actions must a solutions architect take to meet these requirements? (Choose two.)

  • A. Create an ongoing replication task. Most Voted
  • B. Create a database backup of the on-premises database.
  • C. Create an AWS Database Migration Service (AWS DMS) replication server. Most Voted
  • D. Convert the database schema by using the AWS Schema Conversion Tool (AWS SCT).
  • E. Create an Amazon EventBridge (Amazon CloudWatch Events) rule to monitor the database synchronization.

C. AWS Database Migration Service(AWS DMS) 복제 서버를 만듭니다. E. 데이터베이스 동기화를 모니터링하기 위한 Amazon EventBridge(Amazon CloudWatch Events) 규칙을 만듭니다. AWS DMS는 온프레미스 데이터베이스에서 Aurora PostgreSQL로 실시간으로 데이터를 복제할 수 있으므로 온프레미스 데이터베이스는 마이그레이션 중에도 온라인 상태를 유지하고 액세스할 수 있습니다. AWS DMS는 데이터베이스 스키마를 자동으로 변환할 수도 있으므로 AWS SCT를 사용할 필요가 없습니다. Amazon EventBridge 규칙을 사용하면 데이터베이스 동기화를 모니터링하고 오류가 발생하면 알림을 보낼 수 있습니다. 이는 솔루션 아키텍트가 마이그레이션 중에 발생할 수 있는 문제를 신속하게 식별하여 해결할 수 있기 때문에 중요합니다. AWS DMS가 실시간으로 데이터를 복제하므로 온프레미스 데이터베이스의 데이터베이스 백업은 필요하지 않습니다. 복제 서버가 생성되면 AWS DMS가 자동으로 지속적인 복제 작업을 생성하므로 지속적인 복제 작업을 생성할 필요가 없습니다.
