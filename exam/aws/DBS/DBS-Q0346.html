주제 #: 1

한 게임 회사가 출시 후 처음 2주 동안 활성 동시 사용자가 최대 25,000명에 달하는 모바일 게임을 구축하고 있습니다. 이 게임에는 지난 24시간 동안 가장 높은 점수를 얻은 플레이어 10명을 보여주는 리더보드가 있습니다. 순위표 계산은 AWS Lambda 함수에 의해 처리되며 약 10초가 소요됩니다. 회사에서는 리더보드의 데이터가 1분 이내의 데이터를 원합니다.

어떤 아키텍처가 운영상 가장 효율적인 방식으로 이러한 요구 사항을 충족합니까?

  • A. 플레이어 데이터를 Amazon Timestream 데이터베이스로 전달합니다. Redis 클러스터용 Amazon ElastiCache를 생성합니다. Redis에 결과를 저장하도록 Lambda 함수를 구성합니다. Amazon EventBridge로 예약된 이벤트를 생성하여 1분마다 한 번씩 Lambda 함수를 호출합니다. Redis 클러스터에서 순위표 데이터를 쿼리하도록 게임 서버를 재구성합니다.
  • B. 플레이어 데이터를 Amazon Timestream 데이터베이스로 전달합니다. Amazon DynamoDB 테이블을 생성합니다. DynamoD에 결과를 저장하도록 Lambda 함수를 구성합니다. Amazon EventBridge로 예약된 이벤트를 생성하여 1분마다 한 번씩 Lambda 함수를 호출합니다. DynamoDB 테이블에서 순위표 데이터를 쿼리하도록 게임 서버를 재구성합니다.
  • C. 플레이어 데이터를 Amazon Aurora MySQL 데이터베이스로 전달합니다. Amazon DynamoDB 테이블을 생성합니다. MySQL에 결과를 저장하도록 Lambda 함수를 구성합니다. Amazon EventBridge로 예약된 이벤트를 생성하여 1분마다 한 번씩 Lambda 함수를 호출합니다. DynamoDB 테이블에서 순위표 데이터를 쿼리하도록 게임 서버를 재구성합니다.
  • D. 플레이어 데이터를 Amazon Neptune 데이터베이스로 전달합니다. Redis 클러스터용 Amazon ElastiCache를 생성합니다. Redis에 결과를 저장하도록 Lambda 함수를 구성합니다. Amazon EventBridge로 예약된 이벤트를 생성하여 1분마다 한 번씩 Lambda 함수를 호출합니다. Redis 클러스터에서 순위표 데이터를 쿼리하도록 게임 서버를 재구성합니다.

The game has a leaderboard that shows the 10 highest scoring players over the last 24 hours. The leaderboard calculations are processed by an AWS Lambda function, which takes about 10 seconds. The company wants the data on the leaderboard to be no more than 1 minute old.

Which architecture will meet these requirements in the MOST operationally efficient way?

Create a scheduled event with Amazon EventBridge to invoke the Lambda function once every minute. Reconfigure the game server to query the Redis cluster for the leaderboard data.

Reconfigure the game server to query the DynamoDB table for the leaderboard data.

Create a scheduled event with Amazon EventBridge to invoke the Lambda function once every minute. Reconfigure the game server to query the DynamoDB table for the leaderboard data.
