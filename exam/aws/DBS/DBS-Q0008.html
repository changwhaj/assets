주제 #: 1

금융 서비스 회사는 회사 전체에서 다양한 애플리케이션을 지원하는 공유 데이터 서비스를 개발하고 있습니다. 데이터베이스 전문가는 클러스터 모드가 활성화된 Redis용 Amazon ElastiCache를 활용하여 성능과 확장성을 향상시키는 솔루션을 설계했습니다. 클러스터는 포트 6379를 수신하도록 구성되어 있습니다.
데이터베이스 전문가는 캐시 데이터를 보호하고 무단 액세스로부터 보호하기 위해 어떤 단계 조합을 수행해야 합니까? (3개를 선택하세요.)

  • A. ElastiCache 클러스터에서 전송 중 및 미사용 암호화를 활성화합니다. 최다 투표
  • B. Amazon CloudWatch 지표가 ElastiCache 클러스터에 구성되어 있는지 확인하십시오.
  • C. ElastiCache 클러스터의 보안 그룹이 자체로부터의 모든 인바운드 트래픽과 신뢰할 수 있는 클라이언트로부터의 TCP 포트 6379에서의 인바운드 트래픽만 허용하는지 확인하십시오. 최다 투표
  • D. 애플리케이션 서비스 역할이 모든 ElastiCache API 작업에 액세스할 수 있도록 IAM 정책을 생성합니다.
  • E. ElastiCache 클라이언트용 보안 그룹이 신뢰할 수 있는 ElastiCache 클러스터 보안 그룹의 인바운드 TCP 포트 6379 및 포트 22 트래픽을 승인하는지 확인합니다.
  • F. auth-token 매개변수를 사용하여 클러스터가 생성되었고 이 매개변수가 모든 후속 명령에 사용되는지 확인합니다. 최다 투표

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 8
Topic #: 1

A financial services company is developing a shared data service that supports different applications from throughout the company. A Database Specialist designed a solution to leverage Amazon ElastiCache for Redis with cluster mode enabled to enhance performance and scalability. The cluster is configured to listen on port 6379.
Which combination of steps should the Database Specialist take to secure the cache data and protect it from unauthorized access? (Choose three.)

  • A. Enable in-transit and at-rest encryption on the ElastiCache cluster. Most Voted
  • B. Ensure that Amazon CloudWatch metrics are configured in the ElastiCache cluster.
  • C. Ensure the security group for the ElastiCache cluster allows all inbound traffic from itself and inbound traffic on TCP port 6379 from trusted clients only. Most Voted
  • D. Create an IAM policy to allow the application service roles to access all ElastiCache API actions.
  • E. Ensure the security group for the ElastiCache clients authorize inbound TCP port 6379 and port 22 traffic from the trusted ElastiCache cluster's security group.
  • F. Ensure the cluster is created with the auth-token parameter and that the parameter is used in all subsequent commands. Most Voted
