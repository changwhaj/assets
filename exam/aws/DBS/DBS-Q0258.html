질문 #: 258
주제 #: 1

한 회사에서 블로그 플랫폼을 운영하고 있습니다. 보안 감사에서는 플랫폼에서 사용되는 Amazon RDS DB 인스턴스가 저장 데이터를 암호화하도록 구성되지 않았음을 확인합니다. 회사는 30일 이내에 DB 인스턴스를 암호화해야 합니다.
가동 중지 시간을 최소화하면서 이 요구 사항을 충족하려면 데이터베이스 전문가가 무엇을 해야 합니까?

  • A. DB 인스턴스의 읽기 전용 복제본을 생성하고 암호화를 활성화합니다. 읽기 전용 복제본을 사용할 수 있으면 읽기 전용 복제본을 승격하고 애플리케이션에서 사용하는 엔드포인트를 업데이트하세요. 암호화되지 않은 DB 인스턴스를 삭제합니다.
  • B. DB 인스턴스의 스냅샷을 찍습니다. 스냅샷의 암호화된 복사본을 만듭니다. 암호화된 스냅샷을 복원합니다. 새 DB 인스턴스를 사용할 수 있게 되면 애플리케이션에서 사용하는 엔드포인트를 업데이트하십시오. 암호화되지 않은 DB 인스턴스를 삭제합니다.
  • C. 새로운 암호화된 DB 인스턴스를 생성한다. 초기 데이터 로드를 수행하고 두 DB 인스턴스 간의 논리적 복제를 설정합니다. 새 DB 인스턴스가 원본 DB 인스턴스와 동기화되면 애플리케이션에서 사용하는 엔드포인트를 업데이트합니다. 암호화되지 않은 DB 인스턴스를 삭제합니다. 최다 투표
  • D. DB 인스턴스를 Amazon Aurora DB 클러스터로 변환하고 암호화를 활성화합니다. DB 클러스터를 사용할 수 있으면 애플리케이션에서 사용하는 엔드포인트를 클러스터 엔드포인트로 업데이트합니다. 암호화되지 않은 DB 인스턴스를 삭제합니다.

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 258
Topic #: 1

A company is running a blogging platform. A security audit determines that the Amazon RDS DB instance that is used by the platform is not configured to encrypt the data at rest. The company must encrypt the DB instance within 30 days.
What should a database specialist do to meet this requirement with the LEAST amount of downtime?

  • A. Create a read replica of the DB instance, and enable encryption. When the read replica is available, promote the read replica and update the endpoint that is used by the application. Delete the unencrypted DB instance.
  • B. Take a snapshot of the DB instance. Make an encrypted copy of the snapshot. Restore the encrypted snapshot. When the new DB instance is available, update the endpoint that is used by the application. Delete the unencrypted DB instance.
  • C. Create a new encrypted DB instance. Perform an initial data load, and set up logical replication between the two DB instances When the new DB instance is in sync with the source DB instance, update the endpoint that is used by the application. Delete the unencrypted DB instance. Most Voted
  • D. Convert the DB instance to an Amazon Aurora DB cluster, and enable encryption. When the DB cluster is available, update the endpoint that is used by the application to the cluster endpoint. Delete the unencrypted DB instance.
