Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 196
주제 #: 1

한 회사가 중요한 데이터를 Amazon Aurora MySQL 데이터베이스에 로드하고 있습니다. 규정 준수 요구 사항을 충족하려면 회사는 Aurora MySQL DB 클러스터에서 감사 로깅을 활성화하여 데이터베이스 활동을 감사해야 합니다. 이 로깅에는 연결, 연결 끊김, 쿼리 및 쿼리된 테이블과 같은 이벤트가 포함됩니다. 또한 회사는 실시간 데이터 분석을 수행하기 위해 Amazon CloudWatch에 DB 로그를 게시해야 합니다.
이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. 고급 감사를 활성화하려면 기본 옵션 그룹 매개 변수를 수정하십시오. 변경 사항을 적용하려면 데이터베이스를 다시 시작하십시오.
  • B. 사용자 지정 DB 클러스터 파라미터 그룹을 생성합니다. 고급 감사에 대한 매개변수를 수정합니다. 새 사용자 지정 DB 파라미터 그룹을 Aurora MySQL DB 클러스터와 연결하도록 클러스터를 수정합니다. 최다 투표
  • C. 데이터베이스의 스냅샷을 찍습니다. 새 DB 인스턴스를 생성하고 CloudWatch에 대한 사용자 지정 감사 및 로깅을 활성화합니다. 로깅이 없는 DB 인스턴스를 비활성화합니다.
  • D. DB 인스턴스에 대해 AWS CloudTrail을 활성화합니다. 연결, 연결 끊김, 쿼리 및 쿼리된 테이블만 제공하는 필터를 만듭니다.

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 196
Topic #: 1

A company is loading sensitive data into an Amazon Aurora MySQL database. To meet compliance requirements, the company needs to enable audit logging on the Aurora MySQL DB cluster to audit database activity. This logging will include events such as connections, disconnections, queries, and tables queried. The company also needs to publish the DB logs to Amazon CloudWatch to perform real-time data analysis.
Which solution meets these requirements?

  • A. Modify the default option group parameters to enable Advanced Auditing. Restart the database for the changes to take effect.
  • B. Create a custom DB cluster parameter group. Modify the parameters for Advanced Auditing. Modify the cluster to associate the new custom DB parameter group with the Aurora MySQL DB cluster. Most Voted
  • C. Take a snapshot of the database. Create a new DB instance, and enable custom auditing and logging to CloudWatch. Deactivate the DB instance that has no logging.
  • D. Enable AWS CloudTrail for the DB instance. Create a filter that provides only connections, disconnections, queries, and tables queried.
