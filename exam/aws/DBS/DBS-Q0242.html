주제 #: 1

한 회사에서 데이터 웨어하우스 솔루션으로 Amazon Redshift를 사용하고 있습니다. Redshift 클러스터는 다음 유형의 워크로드를 처리합니다.
✑ Amazon Kinesis Data Firehose를 통한 실시간 삽입
✑ Amazon S3의 COPY 명령을 통한 대량 삽입
✑ SQL 쿼리를 통한 분석
최근 클러스터에서 성능 문제가 발생하기 시작했습니다.
클러스터 성능을 향상시키기 위해 데이터베이스 전문가는 어떤 조치 조합을 취해야 합니까? (3개를 선택하세요.)

  • A. 버퍼 크기가 큰 Amazon S3로 데이터를 스트리밍하고 COPY 명령을 사용하여 Amazon Redshift로 데이터를 로드하도록 Kinesis Data Firehose 전송 스트림을 수정합니다.
  • B. 데이터를 영구 테이블에 로드하기 전에 실시간 데이터를 Redshift 임시 테이블로 스트리밍합니다. 최다 투표
  • C. 대량 삽입의 경우 Amazon S3의 입력 파일을 여러 파일로 분할하여 Amazon Redshift의 슬라이스 수와 일치시킵니다. 그런 다음 COPY 명령을 사용하여 Amazon Redshift에 데이터를 로드합니다. 최다 투표
  • D. 대량 삽입의 경우 COPY 명령에서 병렬 매개변수를 사용하여 멀티스레딩을 활성화합니다.
  • E. 정렬 키를 사용하도록 분석 SQL 쿼리를 최적화합니다. 최다 투표
  • F. 분석 SQL 쿼리에 임시 테이블을 사용하지 마십시오.

Question #: 242
Topic #: 1

A company is using Amazon Redshift as its data warehouse solution. The Redshift cluster handles the following types of workloads:
✑ Real-time inserts through Amazon Kinesis Data Firehose
✑ Bulk inserts through COPY commands from Amazon S3
✑ Analytics through SQL queries
Recently, the cluster has started to experience performance issues.
Which combination of actions should a database specialist take to improve the cluster's performance? (Choose three.)

  • A. Modify the Kinesis Data Firehose delivery stream to stream the data to Amazon S3 with a high buffer size and to load the data into Amazon Redshift by using the COPY command.
  • B. Stream real-time data into Redshift temporary tables before loading the data into permanent tables. Most Voted
  • C. For bulk inserts, split input files on Amazon S3 into multiple files to match the number of slices on Amazon Redshift. Then use the COPY command to load data into Amazon Redshift. Most Voted
  • D. For bulk inserts, use the parallel parameter in the COPY command to enable multi-threading.
  • E. Optimize analytics SQL queries to use sort keys. Most Voted
  • F. Avoid using temporary tables in analytics SQL queries.
