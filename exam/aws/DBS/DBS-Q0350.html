Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 350
주제 #: 1

회사에는 AWS Site-to-Site VPN 연결을 통해 VPC가 온프레미스 네트워크에 연결되는 하이브리드 환경이 있습니다. VPC에는 Amazon EC2 인스턴스에서 호스팅되는 애플리케이션이 포함되어 있습니다. EC2 인스턴스는 여러 퍼블릭 서브넷과 연결된 ALB(Application Load Balancer) 뒤의 프라이빗 서브넷에서 실행됩니다. EC2 인스턴스는 Amazon DynamoDB 테이블에 안전하게 액세스해야 합니다.

어떤 솔루션이 이러한 요구 사항을 충족합니까?

  • A. DynamoDB 테이블에 액세스하려면 VPC의 인터넷 게이트웨이를 사용하십시오. ALB를 사용하여 트래픽을 EC2 인스턴스로 라우팅합니다.
  • B. VPC의 퍼블릭 서브넷 중 하나에 NAT 게이트웨이를 추가합니다. NAT 게이트웨이를 통해 DynamoDB 테이블에 액세스하도록 EC2 인스턴스의 보안 그룹을 구성합니다.
  • C. Site-to-Site VPN 연결을 사용하여 온프레미스 네트워크 인프라를 통해 모든 DynamoDB 네트워크 트래픽을 라우팅하여 EC2 인스턴스에 액세스합니다.
  • D. DynamoDB용 VPC 엔드포인트를 생성합니다. EC2 인스턴스가 포함된 프라이빗 서브넷의 라우팅 테이블에 엔드포인트를 할당합니다.

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 350
Topic #: 1

A company has a hybrid environment in which a VPC connects to an on-premises network through an AWS Site-to-Site VPN connection. The VPC contains an application that is hosted on Amazon EC2 instances. The EC2 instances run in private subnets behind an Application Load Balancer (ALB) that is associated with multiple public subnets. The EC2 instances need to securely access an Amazon DynamoDB table.

Which solution will meet these requirements?

  • A. Use the internet gateway of the VPC to access the DynamoDB table. Use the ALB to route the traffic to the EC2 instances.
  • B. Add a NAT gateway in one of the public subnets of the VPC. Configure the security groups of the EC2 instances to access the DynamoDB table through the NAT gateway.
  • C. Use the Site-to-Site VPN connection to route all DynamoDB network traffic through the on-premises network infrastructure to access the EC2 instances.
  • D. Create a VPC endpoint for DynamoDB. Assign the endpoint to the route table of the private subnets that contain the EC2 instances.

A VPC endpoint for DynamoDB enables Amazon EC2 instances in your VPC to use their private IP addresses to access DynamoDB with no exposure to the public internet. The EC2 instances do not require public IP addresses, and internet gateway, NAT device, VPN connection, or AWS Direct Connect connection is not required to communicate with DynamoDB.
