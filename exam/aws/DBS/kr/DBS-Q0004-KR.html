Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 4
주제 #: 1

회사는 온프레미스 시스템에서 마이그레이션하여 Amazon Aurora에 솔루션을 배포하고 있습니다. IT 부서는 회사의 데이터 센터에서 AWS Direct Connect 링크를 설정했습니다. 회사의 데이터베이스 전문가는 일반 텍스트 데이터가 네트워크를 통해 설정되는 것을 방지하기 위해 연결에 SSL/TLS를 요구하는 옵션을 선택했습니다. 마이그레이션이 성공적으로 진행되고 있는 것으로 보이며 데스크톱 컴퓨터에서 데이터를 쿼리할 수 있습니다.
두 명의 데이터 분석가에게 새로운 Aurora DB 클러스터의 데이터를 쿼리하고 검증하도록 요청했습니다. 두 분석가 모두 Aurora에 연결할 수 없습니다. 사용자 이름과 암호가 유효한 것으로 확인되었으며 데이터베이스 전문가는 해당 계정을 사용하여 DB 클러스터에 연결할 수 있습니다. 또한 데이터베이스 전문가는 보안 그룹 구성이 모든 회사 IP 주소의 네트워크를 허용하는지 확인했습니다.
데이터 분석가가 연결할 수 없는 문제를 해결하기 위해 데이터베이스 전문가는 무엇을 해야 합니까?

  • A. SSL 변경 사항을 적용하기 위해 DB 클러스터를 재시작합니다.
  • B. 데이터 분석가에게 루트 인증서를 다운로드하고 연결 문자열에서 SSL 인증서를 사용하여 연결하도록 지시합니다. 최다 투표
  • C. 데이터 분석가의 IP 주소와 DB 클러스터에 할당된 보안 그룹의 인스턴스 간에 명시적 매핑을 추가합니다.
  • D. 데이터 분석가의 로컬 클라이언트 방화벽을 수정하여 AWS로의 네트워크 트래픽을 허용합니다.
