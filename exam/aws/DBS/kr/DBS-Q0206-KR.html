Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 206
주제 #: 1

회사에서 전체 로드 작업으로 AWS Database Migration Service(AWS DMS)를 사용하여 하나의 Amazon EC2 인스턴스에서 다른 EC2 인스턴스로 데이터베이스를 마이그레이션할 계획입니다. 회사는 마이그레이션하는 동안 데이터베이스가 비활성화되기를 원합니다. 회사는 dms.t3.medium 인스턴스를 사용하여 마이그레이션을 수행하고 마이그레이션에 대한 기본 설정을 사용합니다.
데이터 마이그레이션 성능을 가장 향상시키는 솔루션은 무엇입니까?

  • A. 병렬로 로드되는 테이블 수를 늘립니다.
  • B. 원본 테이블의 모든 인덱스를 삭제합니다.
  • C. 일괄 최적화 적용 옵션에서 트랜잭션 모드로 처리 모드를 변경합니다.
  • D. 전체 로드 작업이 진행되는 동안 대상 데이터베이스에서 다중 AZ를 활성화합니다.
추천 답변 보기 답변 숨기기
추천 답변: C 🗳️
변경 처리 최적화.
기본적으로 AWS DMS는 트랜잭션 무결성을 유지하는 트랜잭션 모드에서 변경 사항을 처리합니다. 트랜잭션 무결성의 일시적인 경과를 감당할 수 있는 경우 일괄 최적화 적용 옵션을 대신 사용할 수 있습니다. 이 옵션은 트랜잭션을 효율적으로 그룹화하고 효율성을 위해 배치로 적용합니다.
배치 최적화 적용 옵션을 사용하면 거의 항상 참조 무결성 제약 조건을 위반합니다. 따라서 마이그레이션 프로세스 중에 이러한 제약 조건을 해제하고 컷오버 프로세스의 일부로 다시 설정하는 것이 좋습니다.
참조:
https://docs.aws.amazon.com/dms/latest/userguide/CHAP_BestPractices.html
