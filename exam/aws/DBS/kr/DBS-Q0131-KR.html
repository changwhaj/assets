Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 131
주제 #: 1

소매 회사는 Amazon DynamoDB 테이블에 데이터를 저장하는 웹 애플리케이션을 관리합니다. 회사는 계정 통합 작업을 진행 중입니다. 데이터베이스 엔지니어는 DynamoDB 테이블을 현재 AWS 계정에서 새 AWS 계정으로 마이그레이션해야 합니다.
최소한의 관리 작업으로 이러한 요구 사항을 충족하는 전략은 무엇입니까?

  • A. AWS Glue를 사용하여 DynamoDB 테이블의 데이터를 크롤링합니다. 사용 가능한 청사진을 사용하여 작업을 생성하여 데이터를 Amazon S3로 내보냅니다. S3 파일에서 새 계정의 DynamoDB 테이블로 데이터를 가져옵니다.
  • B. 현재 계정의 DynamoDB 테이블 항목을 스캔하고 Amazon S3의 파일에 쓰는 AWS Lambda 함수를 생성합니다. S3 파일을 읽고 새 계정에서 DynamoDB 테이블의 항목을 복원하는 또 다른 Lambda 함수를 생성합니다.
  • C. 현재 계정에서 AWS Data Pipeline을 사용하여 DynamoDB 테이블의 데이터를 Amazon S3의 파일로 내보냅니다. Data Pipeline을 사용하여 S3 파일에서 새 계정의 DynamoDB 테이블로 데이터를 가져옵니다.
  • D. 현재 계정의 DynamoDB 테이블에 대한 Amazon DynamoDB 스트림을 구성합니다. 스트림에서 읽고 Amazon S3의 파일에 쓰는 AWS Lambda 함수를 생성합니다. S3 파일을 읽고 항목을 새 계정의 DynamoDB 테이블로 복원하는 또 다른 Lambda 함수를 생성합니다.

선택한 답변: D
현재 계정의 DynamoDB 테이블에 대한 Amazon DynamoDB 스트림을 구성합니다. 스트림에서 읽고 Amazon S3의 파일에 쓰는 AWS Lambda 함수를 생성합니다. S3 파일을 읽고 항목을 새 계정의 DynamoDB 테이블로 복원하는 또 다른 Lambda 함수를 생성합니다. 이 접근 방식은 생성, 업데이트 및 삭제를 포함하여 테이블에 대한 항목 수준 수정 사항을 캡처하는 DynamoDB Streams를 활용합니다. DynamoDB Streams 데이터는 서로 다른 AWS 계정 간에 거의 실시간으로 데이터를 복제하는 데 사용할 수 있습니다. 이 접근 방식을 사용하면 S3 및 DynamoDB 테이블을 각각 읽고 쓰는 데 두 개의 Lambda 함수만 생성하면 되므로 최소한의 관리 작업이 가능합니다. 이 접근 방식은 또한 마이그레이션 프로세스 중에 데이터 일관성이 유지되도록 합니다.
