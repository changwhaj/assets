질문 #: 169
주제 #: 1

회사에 온-프레미스 SQL Server 데이터베이스가 있습니다. 사용자는 Active Directory 인증을 사용하여 데이터베이스에 액세스합니다. 이 회사는 데이터베이스를 SQL Server용 Amazon RDS로 성공적으로 마이그레이션했습니다. 그러나 회사는 AWS Cloud 환경에서 사용자 인증에 대해 우려하고 있습니다.
데이터베이스 전문가는 사용자 인증을 위해 어떤 솔루션을 제공해야 합니까?

  • A. 온프레미스에 AD FS(Active Directory Federation Services)를 배포하고 온프레미스 Active Directory로 구성합니다. 온프레미스 AD FS와 AWS Security Token Service(AWS STS) 간에 위임을 설정하여 AmazonRDSDirectoryServiceAccess 관리형 IAM 정책을 사용하여 사용자 자격 증명을 역할에 매핑합니다.
  • B. 온프레미스 Active Directory와 Microsoft Active Directory용 AWS Directory Service 간에 포리스트 트러스트를 설정합니다. AWS SSO를 사용하여 SQL Server용 RDS의 데이터베이스에 액세스하도록 위임된 Active Directory 사용자를 구성합니다.
  • C. Active Directory Connector를 사용하여 클라우드에 정보를 캐싱하지 않고 디렉터리 요청을 회사의 온프레미스 Active Directory로 리디렉션합니다. RDS 마스터 사용자 자격 증명을 사용하여 DB 인스턴스에 연결하고 Active Directory 사용자 및 그룹에서 SQL Server 로그인 및 사용자를 구성합니다.
  • D. 온프레미스 Active Directory와 Microsoft Active Directory용 AWS Directory Service 간에 포리스트 트러스트를 설정합니다. SQL Server용 RDS가 혼합 모드 인증을 사용하는지 확인합니다. RDS 마스터 사용자 자격 증명을 사용하여 DB 인스턴스에 연결하고 Active Directory 사용자 및 그룹에서 SQL Server 로그인 및 사용자를 구성합니다. 최다 투표

Amazon RDS는 Windows 인증에 혼합 모드를 사용합니다. 이 접근 방식은 마스터 사용자(SQL Server DB 인스턴스를 생성하는 데 사용되는 이름과 암호)가 SQL 인증을 사용함을 의미합니다. 마스터 사용자 계정은 권한 있는 자격 증명이므로 이 계정에 대한 액세스를 제한해야 합니다. 온프레미스 또는 자체 호스팅 Microsoft Active Directory를 사용하여 Windows 인증을 받으려면 포리스트 트러스트를 만듭니다. 신뢰는 단방향 또는 양방향일 수 있습니다. AWS Directory Service를 사용하여 포리스트 트러스트를 설정하는 방법에 대한 자세한 내용은 AWS Directory Service Administration Guide의 트러스트 관계를 생성해야 하는 경우를 참조하십시오.
