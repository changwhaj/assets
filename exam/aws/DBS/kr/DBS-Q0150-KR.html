Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 150
주제 #: 1

금융 회사는 최근 90일 동안 MySQL 데이터베이스 백업을 사용할 수 있는지 확인해야 합니다. 모든 MySQL 데이터베이스는
Amazon RDS for MySQL DB 인스턴스에서 호스팅됩니다. 데이터베이스 전문가는 최소한의 개발 노력으로 백업 보존 요구 사항을 충족하는 솔루션을 구현해야 합니다.
데이터베이스 전문가는 어떤 접근 방식을 취해야 합니까?

  • A. AWS Backup을 사용하여 필요한 보존 기간에 대한 백업 계획을 세웁니다. 백업 계획에 DB 인스턴스를 할당합니다. 최다 투표
  • B. 자동 백업 옵션을 활성화하도록 DB 인스턴스를 수정합니다. 필요한 백업 보존 기간을 선택합니다.
  • C. Amazon EC2 인스턴스에서 일일 cron 작업을 자동화하여 MySQL 덤프를 생성하고 Amazon S3로 전송하고 S3 수명 주기 정책을 구현하여 보존 요구 사항을 충족합니다.
  • D. AWS Lambda를 사용하여 DB 인스턴스의 일일 수동 스냅샷을 예약합니다. 보존 요구 사항을 초과하는 스냅샷을 삭제합니다.

DB 인스턴스를 생성할 때 백업 보존 기간을 설정할 수 있습니다. 백업 보존 기간을 설정하지 않은 경우 Amazon RDS API 또는 AWS CLI를 사용하여 DB 인스턴스를 생성하면 기본 백업 보존 기간은 1일입니다. 콘솔을 사용하여 DB 인스턴스를 생성하는 경우 기본 백업 보존 기간은 7일입니다. DB 인스턴스를 생성한 후 백업 보존 기간을 수정할 수 있습니다. 백업 보존 기간은 0일에서 35일 사이로 설정할 수 있습니다. 백업 보존 기간을 0으로 설정하면 자동 백업이 비활성화됩니다. 수동 스냅샷 제한(지역당 100개)은 자동 백업에 적용되지 않습니다. 자동 백업의 최대 한도는 35일이며 수동 스냅샷은 https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/USER_WorkingWithAutomatedBackups.html#USER_WorkingWithAutomatedBackups 솔루션입니다.
