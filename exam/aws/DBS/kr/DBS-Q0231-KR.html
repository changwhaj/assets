Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 231
주제 #: 1

전자상거래 회사는 AWS DMS(AWS Database Migration Service)를 실행하여 온프레미스 Microsoft SQL Server 데이터베이스를 SQL Server용 Amazon RDS로 복제합니다. 이 회사는 온프레미스 데이터 센터에서 AWS로의 AWS Direct Connect 연결을 설정했습니다. 마이그레이션 중에 회사의 보안 팀은 마이그레이션과 관련된 알람을 받습니다. 보안 팀은 공용
IP 주소에서 DMS 복제 인스턴스에 액세스할 수 없도록 지정합니다.
이 요구 사항을 충족하려면 데이터베이스 전문가는 어떻게 해야 합니까?

  • A. Direct Connect 연결을 통한 트래픽을 암호화하도록 VPN 연결을 설정합니다.
  • B. 공개적으로 액세스 가능한 옵션을 비활성화하여 DMS 복제 인스턴스를 수정합니다.
  • C. DMS 복제 인스턴스를 삭제합니다. 공개적으로 액세스 가능한 옵션을 비활성화하여 DMS 복제 인스턴스를 다시 생성합니다. 최다 투표
  • D. 프라이빗 서브넷이 있는 새 복제 VPC 서브넷 그룹을 생성합니다. 새로 생성된 VPC 서브넷 그룹을 선택하여 DMS 복제 인스턴스를 수정합니다.

옵션 C가 맞습니다. 옵션 D에는 약간의 주의 사항이 있습니다. 프라이빗 서브넷이 있는 새 DMS 서브넷 그룹(DMS 콘솔 내)을 생성할 수 있지만 이 새 서브넷 그룹을 기존 DMS 인스턴스에 연결할 수는 없습니다. 이것은 지원되지 않으며 AWS 콘솔에서 테스트했습니다. 이 메모(https://repost.aws/knowledge-center/dms-disable-public-access)에서 설명하는 내용은 D 옵션을 해석하고 선택하는 일부 내용과 약간 다릅니다. 이 메모에 따라 공개를 제거할 수 있습니다. 서브넷 및 EXISTING SUBNET 그룹의 개인 서브넷만 선택하고 이것을 시도했습니다. 기존 DMS 인스턴스의 기존 서브넷 그룹을 수정했습니다. 모든 퍼블릭 서브넷을 제거하고 2개의 프라이빗 서브넷만 선택했습니다. 그러나 이것은 효과가 있었지만 옵션 D는 기존 서브넷 그룹 수정에 대해 말하는 것이 아닙니다. 새 서브넷 그룹을 만들고 위에서 이미 언급한 기존 DMS 인스턴스와 연결해야 한다는 것은 현재 지원되지 않는 옵션입니다. 따라서 옵션 D는 올바르지 않습니다.
