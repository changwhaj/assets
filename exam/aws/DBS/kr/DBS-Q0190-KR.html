응용 프로그램 개발자는 더 많은 사용자가 추가될수록 응용 프로그램 실행 속도가 느려진다고 보고했습니다. 애플리케이션 데이터베이스는
Aurora 복제본이 있는 Amazon Aurora DB 클러스터에서 실행 중입니다. 응용 프로그램은 판독기 끝점을 통해 읽기 확장을 활용하도록 작성되었습니다. 데이터베이스 전문가는 데이터베이스의 성능 지표를 살펴보고 새 사용자가 데이터베이스에 추가됨에 따라 Aurora 복제본 CPU 사용률이 일정하게 유지되는 동안 기본 인스턴스 CPU 사용률이 꾸준히 증가했음을 확인합니다.
데이터베이스 전문가는 다운타임을 최소화하면서 어떻게 데이터베이스 성능을 향상시킬 수 있습니까?

  • A. 전체 로드가 안정화될 때까지 더 많은 복제본을 추가하도록 Aurora DB 클러스터를 수정합니다. 그런 다음 애플리케이션이 서비스 수준 목표를 충족하면 복제본 수를 줄입니다.
  • B. 더 많은 CPU 용량을 제공하는 더 큰 인스턴스 크기로 기본 인스턴스를 수정합니다.
  • C. CPU 용량이 더 큰 더 큰 인스턴스 크기로 복제본을 수정합니다. 그런 다음 수정된 복제본을 승격합니다. 최다 투표
  • D. Aurora DB 클러스터를 CPU 용량이 더 많은 인스턴스 크기로 복원합니다. 그런 다음 이전 DB 클러스터와 새 DB 클러스터의 이름을 바꿉니다.
