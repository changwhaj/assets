질문 #: 34
주제 #: 1

회사는 고가용성 및 읽기 전용 워크로드 확장을 위해 xlarge 기본 인스턴스 마스터와 2개의 대형 Aurora 복제본이 있는 Amazon Aurora PostgreSQL DB 클러스터를 사용하고 있습니다. 장애 조치 이벤트가 발생하고 애플리케이션 성능이 몇 분 동안 저하됩니다. 이 시간 동안 모든 가용
영역의 애플리케이션 서버는 정상 상태이며 정상적으로 응답합니다.
이 애플리케이션 성능 문제를 해결하기 위해 회사는 무엇을 해야 합니까?

  • A. 두 Aurora 복제본을 기본 DB 인스턴스와 동일한 인스턴스 클래스로 구성합니다. DB 클러스터에서 캐시 일관성을 활성화하고 기본 DB 인스턴스 장애 조치 우선 순위를 Tier-0으로 설정하고 장애 조치 우선 순위 Tier-1을 복제본에 할당합니다.
  • B. 어떤 인스턴스가 실패했는지 확인하기 위해 DescribeDBInstances 작업을 호출하는 AWS Lambda 함수를 배포한 다음 PromoteReadReplica 작업을 사용하여 하나의 Aurora 복제본을 기본 DB 인스턴스로 승격합니다. Lambda 함수가 구독하는 Amazon SNS 주제로 알림을 보내도록 Amazon RDS 이벤트 구독을 구성합니다.
  • C. 기본 DB 인스턴스와 동일한 인스턴스 클래스를 갖도록 하나의 Aurora 복제본을 구성합니다. Aurora PostgreSQL DB 클러스터 캐시 관리를 구현합니다. 기본 DB 인스턴스와 인스턴스 클래스가 동일한 하나의 복제본에 대해 장애 조치 우선 순위를 Tier-0으로 설정합니다. 다른 복제본에 대해 장애 조치 우선 순위를 Tier-1로 설정합니다. 최다 투표
  • D. 기본 DB 인스턴스와 동일한 인스턴스 클래스를 갖도록 두 Aurora 복제본을 구성합니다. Aurora PostgreSQL DB 클러스터 캐시 관리를 구현합니다. 장애 조치 우선 순위를 기본 DB 인스턴스의 경우 Tier-0으로 설정하고 복제본의 경우 Tier-1로 설정합니다.
