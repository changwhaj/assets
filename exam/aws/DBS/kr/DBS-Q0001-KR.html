질문 #: 1
주제 #: 1

회사에서 새 AWS 계정에 전자 상거래 웹 애플리케이션을 배포했습니다. Amazon RDS for MySQL 다중 AZ DB 인스턴스는 포트 3306에서 수신 대기하는 database-1.xxxxxxxxxxxx.us-east-1.rds.amazonaws.com 엔드포인트가 있는 이 배포의 일부입니다. 회사의 데이터베이스 전문가는 다음에 로그인할 수 있습니다. 이러한 세부 정보를 사용하여 배스천 호스트에서 MySQL을 실행하고 쿼리를 실행합니다.
사용자가 AWS 계정에서 호스팅되는 애플리케이션을 활용하려고 하면 일반적인 오류 메시지가 표시됩니다. 애플리케이션 서버가 Amazon CloudWatch Logs에 '서버에 연결할 수 없음: 연결 시간 초과' 오류 메시지를 기록하고 있습니다.
이 오류의 원인은 무엇입니까?

  • A. 애플리케이션이 사용 중인 사용자 이름과 비밀번호가 올바르지 않습니다.
  • B. 애플리케이션 서버에 할당된 보안 그룹에는 DB 인스턴스로부터의 인바운드 연결을 허용하는 데 필요한 규칙이 없습니다.
  • C. DB 인스턴스에 할당된 보안 그룹에 애플리케이션 서버로부터의 인바운드 연결을 허용하는 데 필요한 규칙이 없습니다.
  • D. 사용자 이름과 암호는 정확하지만 DB 인스턴스를 사용할 수 있는 권한이 없는 사용자입니다.
