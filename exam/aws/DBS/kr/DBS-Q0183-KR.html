Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 183
주제 #: 1

한 회사가 한 AWS 리전에서 다른 AWS 리전으로 SQL Server DB 인스턴스용 Amazon RDS의 데이터베이스를 마이그레이션하고 있습니다. 회사는 마이그레이션 중에 데이터베이스 가동 중지 시간을 최소화하려고 합니다.
이 지역 간 마이그레이션을 위해 회사는 어떤 전략을 선택해야 합니까?

  • A. 동일한 리전의 Amazon S3 버킷에 기본 백업을 사용하여 원본 데이터베이스를 백업합니다. 그런 다음 대상 리전에서 백업을 복원합니다.
  • B. 동일한 리전의 Amazon S3 버킷에 기본 백업을 사용하여 소스 데이터베이스를 백업합니다. Amazon S3 교차 리전 복제를 사용하여 대상 리전의 S3 버킷에 백업을 복사합니다. 그런 다음 대상 리전에서 백업을 복원합니다.
  • C. 원본 데이터베이스와 대상 데이터베이스 간에 데이터를 복제하도록 AWS DMS(AWS Database Migration Service)를 구성합니다. 복제가 동기화되면 DMS 작업을 종료합니다. 최다 투표
  • D. 대상 리전에서 SQL Server 교차 리전 읽기 전용 복제본용 RDS를 추가합니다. 복제가 동기화되면 읽기 전용 복제본을 마스터로 승격합니다. 최다 투표
추천 답변 보기 답변 숨기기
제안된 답변: A 🗳️
Amazon RDS는 전체 백업 파일(.bak 파일)을 사용하여 Microsoft SQL Server 데이터베이스의 기본 백업 및 복원을 지원합니다. RDS를 사용하면 데이터베이스 서버의 로컬 파일 시스템을 사용하지 않고 Amazon S3에 저장된 파일에 액세스합니다.
참조:
https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/SQLServer.Procedural.Importing.html
