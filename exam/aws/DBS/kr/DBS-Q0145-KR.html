Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 145
주제 #: 1

승차 호출 애플리케이션은 Amazon RDS for MySQL DB 인스턴스를 예약을 위한 영구 스토리지로 사용합니다. 이 응용 프로그램은 매우 인기가 있으며 회사는 향후 몇 개월 동안 사용자 기반이 10배 증가할 것으로 예상합니다. 애플리케이션은 아침과 저녁 시간에 더 많은 트래픽을 경험합니다.
이 애플리케이션은 두 부분으로 구성됩니다.
✑ 사용자의 동시 요청에 직접 대응하는 온라인 예약을 수락하는 사내 예약 구성 요소.
✑ 고객 관리 담당자가 사용하는 타사 고객 관계 관리(CRM) 구성 요소입니다. CRM은 쿼리를 사용하여 예약 데이터에 액세스합니다.
데이터베이스 전문가는 이 워크로드를 처리하기 위해 비용 효율적인 데이터베이스 솔루션을 설계해야 합니다.
어떤 솔루션이 이러한 요구 사항을 충족합니까?

  • A. Redis용 Amazon ElastiCache를 사용하여 예약을 수락합니다. AWS Lambda 함수를 연결하여 변경 사항을 캡처하고 예약 데이터를 CRM에서 사용하는 RDS for MySQL DB 인스턴스로 푸시합니다.
  • B. Amazon DynamoDB를 사용하여 예약을 수락합니다. DynamoDB Streams를 활성화하고 AWS Lambda 함수를 연결하여 변경 사항을 캡처하고 예약 데이터를 Amazon SQS 대기열로 푸시합니다. 이렇게 하면 Amazon SQS에서 데이터를 가져와 CRM에서 사용하는 MySQL용 RDS DB 인스턴스에 쓰는 또 다른 Lambda 함수가 트리거됩니다. 최다 투표
  • C. Redis용 Amazon ElastiCache를 사용하여 예약을 수락합니다. AWS Lambda 함수를 연결하여 변경 사항을 캡처하고 예약 데이터를 CRM에서 사용하는 Amazon Redshift 데이터베이스로 푸시합니다.
  • D. Amazon DynamoDB를 사용하여 예약을 수락합니다. DynamoDB Streams를 활성화하고 AWS Lambda 함수를 연결하여 변경 사항을 캡처하고 예약 데이터를 CRM에서 사용하는 Amazon Athena로 푸시합니다.
