Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 72
주제 #: 1

회사는 사용자 요청이 Amazon Aurora MySQL DB 클러스터에 삽입을 수행하는 AWS Lambda 함수를 트리거하는 새로운 웹 플랫폼을 구축하고 있습니다. 새로운 플랫폼에서 10명 미만의 사용자를 대상으로 한 초기 테스트에서 성공적인 실행과 빠른 응답 시간이 나타났습니다. 그러나 3,000명의 동시 사용자를 실제 목표로 하는 보다 광범위한 테스트에서 Lambda 함수는 DB 클러스터에 연결할 수 없고 너무 많은 연결 오류를 수신합니다.
다음 중 이 문제를 해결하는 방법은 무엇입니까?

  • A. DB 클러스터의 my.cnf 파일을 편집하여 max_connections를 늘립니다.
  • B. DB 클러스터의 인스턴스 크기를 늘린다. 최다 투표
  • C. DB 클러스터를 다중 AZ로 변경
  • D. Aurora 복제본 수 늘리기

B. DB 클러스터의 인스턴스 크기를 늘리십시오. 인스턴스에 의존하는 더 많은 연결이 필요합니다. Max_connection은 RDS 매개변수 그룹의 공식입니다. ) https://docs.aws.amazon.com/AmazonRDS/latest/AuroraUserGuide/AuroraMySQL.Managing.Performance.html 인스턴스를 DB 인스턴스 클래스로 확장하여 Aurora MySQL DB 인스턴스에 대한 최대 연결 수를 늘릴 수 있습니다. 더 많은 메모리를 사용하거나 인스턴스의 DB 파라미터 그룹에서 max_connections 파라미터에 더 큰 값을 최대 16,000까지 설정합니다. my.cnf를 편집하지 말고 DB 파라미터 그룹에서 max_connections 파라미터에 대해 더 큰 값을 변경해야 합니다. 이는 MySQL을 호스팅하는 물리적 서버가 아닙니다.
