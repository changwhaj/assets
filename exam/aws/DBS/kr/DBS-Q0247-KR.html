소셜 미디어 회사는 Amazon DynamoDB를 사용하여 사용자 프로필 데이터와 사용자 활동 데이터를 저장하고 있습니다. 개발자가 데이터를 읽고 쓰기 때문에 테이블 크기가 크게 증가합니다. 개발자는 테이블의 성능 병목 현상에 직면하기 시작했습니다.
테이블에 대해 프로비저닝된 처리량을 모두 소모하지 않고 항목을 가장 빠르게 읽으려면 데이터베이스 전문가가 추천해야 하는 솔루션은 무엇입니까?

  • A. Scan API 작업을 여러 작업자와 병렬로 사용하여 모든 항목을 읽습니다. 쿼리 API 작업을 사용하여 특정 파티션 키와 정렬 키가 있는 여러 항목을 읽습니다. GetItem API 작업을 사용하여 단일 항목을 읽습니다.
  • B. 여러 항목을 읽을 수 있는 필터 식과 함께 Scan API 작업을 사용합니다. 쿼리 API 작업을 사용하여 특정 파티션 키와 정렬 키가 있는 여러 항목을 읽습니다. GetItem API 작업을 사용하여 단일 항목을 읽습니다.
  • C. 여러 항목을 읽을 수 있는 필터 식과 함께 Scan API 작업을 사용합니다. 쿼리 API 작업을 사용하여 특정 기본 키가 있는 단일 항목을 읽습니다. BatchGetItem API 작업을 사용하여 여러 항목을 읽습니다.
  • D. 많은 작업자와 병렬로 Scan API 작업을 사용하여 모든 항목을 읽습니다. Query API 작업을 사용하여 특정 기본 키가 있는 단일 항목을 읽습니다. BatchGetItem API 작업을 사용하여 여러 항목을 읽습니다.
