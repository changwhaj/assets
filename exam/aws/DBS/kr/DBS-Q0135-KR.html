Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 135
주제 #: 1

새로운 데이터 규정 준수 요구 사항을 충족하려면 회사는 중요한 데이터를 7년 동안 안정적으로 저장하고 쉽게 액세스할 수 있어야 합니다. 1년 이상 된 데이터는 아카이브 데이터로 간주되며 매주 자동으로 Amazon Aurora MySQL DB 클러스터 밖으로 이동해야 합니다. 평균적으로 매월 약 10GB의 새 데이터가 데이터베이스에 추가됩니다. 데이터베이스 전문가는 아카이브 데이터를
Amazon S3로 마이그레이션하기 위해 운영상 가장 효율적인 솔루션을 선택해야 합니다.
어떤 솔루션이 이러한 요구 사항을 충족합니까?

  • A. SQL 보기를 사용하여 DB 클러스터에서 Amazon S3로 아카이브 데이터를 내보낸 다음 DB 클러스터에서 아카이브 데이터를 삭제하는 사용자 지정 스크립트를 생성합니다. 매주 cron 작업으로 Amazon EC2 인스턴스를 시작하여 사용자 지정 스크립트를 실행합니다.
  • B. SELECT INTO OUTFILE S3 문을 사용하여 DB 클러스터에서 Amazon S3로 보관 데이터를 내보낸 다음 DB 클러스터에서 보관 데이터를 삭제하는 AWS Lambda 함수를 구성합니다. Amazon EventBridge(Amazon CloudWatch Events)를 사용하여 Lambda 함수가 매주 실행되도록 예약합니다.
  • C. 두 개의 AWS Lambda 함수를 구성합니다. 하나는 mysqldump 유틸리티를 사용하여 DB 클러스터에서 Amazon S3로 아카이브 데이터를 내보내고 다른 하나는 DB 클러스터에서 아카이브 데이터를 삭제합니다. Amazon EventBridge(Amazon CloudWatch Events)를 사용하여 매주 실행되도록 두 Lambda 함수를 예약합니다.
  • D. AWS Database Migration Service(AWS DMS)를 사용하여 아카이브 데이터를 DB 클러스터에서 Amazon S3로 지속적으로 내보냅니다. 사용자 지정 SQL 스크립트를 실행하여 DB 클러스터에서 아카이브 데이터를 삭제하는 AWS Data Pipeline 프로세스를 매주 실행하도록 구성합니다.
