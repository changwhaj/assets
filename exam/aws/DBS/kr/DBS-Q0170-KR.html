시험 AWS 공인 데이터베이스 - 전문 주제 1 질문 170 토론

Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 170
주제 #: 1

회사는 Amazon Redshift 클러스터를 사용하여 분석 워크로드를 실행합니다. 회사 정책에 따라 회사의 데이터는 고객 관리 키를 사용하여 유휴 상태에서 암호화되어야 합니다. 회사의 재해 복구 계획에 따라 정기적으로 클러스터 백업을 다른 AWS 리전에 복사해야 합니다.
데이터베이스 전문가는 이러한 정책에 따라 클러스터 데이터 백업 프로세스를 어떻게 자동화해야 합니까?

  • A. 소스 리전에서 대상 리전으로 AWS Key Management Service(AWS KMS) 고객 관리형 키를 복사합니다. 원본 리전에서 AWS Glue 작업을 설정하여 Amazon Redshift 클러스터의 최신 스냅샷을 원본 리전에서 대상 리전으로 복사합니다. AWS Glue에서 시간 기반 일정을 사용하여 매일 작업을 실행합니다.
  • B. 대상 리전에서 새 AWS Key Management Service(AWS KMS) 고객 관리형 키를 생성합니다. 새 키를 지정하여 대상 리전에서 스냅샷 복사 권한을 생성합니다. 소스 리전에서 대상 리전, 스냅샷 복사 권한 부여, 스냅샷 보관 기간을 지정하는 Amazon Redshift 클러스터에 대한 교차 리전 스냅샷을 구성합니다.
  • C. 소스 리전에서 대상 리전으로 AWS Key Management Service(AWS KMS) 고객 관리형 키를 복사합니다. 해당 리전의 키를 사용하여 각 리전에서 Amazon S3 버킷을 생성합니다. Amazon EventBridge(Amazon CloudWatch Events)를 사용하여 원본 리전에서 AWS Lambda 함수가 해당 리전의 S3 버킷에 최신 스냅샷을 복사하도록 예약합니다. 복제 구성에서 원본 및 대상 KMS 키 ID를 지정하여 스냅샷을 대상 리전에 복사하도록 S3 교차 리전 복제를 구성합니다.
  • D. 동일한 고객 제공 키 자료를 사용하여 대상 리전에서 동일한 프라이빗 키로 CMK를 생성합니다. 대상 리전을 대상으로 하는 원본 리전에서 교차 리전 스냅샷을 구성합니다. 대상 리전에서 해당 CMK를 지정하여 스냅샷을 암호화합니다.
