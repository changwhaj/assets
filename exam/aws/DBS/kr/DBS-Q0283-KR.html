Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 283
주제 #: 1

한 회사에서 200GB 온프레미스 PostgreSQL 데이터베이스를 Amazon Aurora PostgreSQL로 마이그레이션하고 있습니다. 원본 데이터베이스 열에는 NOT NULL 및 외래 키 제약 조건이 포함됩니다. 데이터베이스 관리자는 데이터베이스 마이그레이션 모범 사례를 따르면서 마이그레이션을 완료해야 합니다.

데이터베이스를 AWS로 마이그레이션하기 위해 이러한 요구 사항을 충족하는 옵션은 무엇입니까?

  • A. AWS Schema Conversion Tool(AWS SCT) 및 AWS Database Migration Service(AWS DMS)를 사용하여 데이터베이스를 Aurora PostgreSQL DB 클러스터로 마이그레이션합니다.
  • B. 소스 데이터베이스에 연결하고 대상 Aurora PostgreSQL DB 클러스터에 데이터를 로드하는 AWS Lambda 함수를 생성합니다.
  • C. PostgreSQL 도구 pg_dump 및 pg_restore를 사용하여 Aurora PostgreSQL DB 클러스터로 마이그레이션합니다.
  • D. Aurora PostgreSQL 읽기 전용 복제본을 생성하고 동기화되면 읽기 전용 복제본을 기본으로 승격합니다.

A. 자체 관리형 PostgreSQL 데이터베이스를 Amazon RDS for PostgreSQL 또는 Aurora PostgreSQL로 마이그레이션하려는 고객은 세 가지 주요 접근 방식 중 하나를 사용할 수 있습니다. 전체 로드 전용 마이그레이션의 경우 pg_dump 및 pg_restore와 같은 기본 또는 타사 데이터베이스 마이그레이션 방법을 사용하십시오. 전체 로드 및 지속적인 복제를 위해 AWS DMS(AWS Database Migration Service)와 같은 관리형 서비스를 사용합니다. 전체 로드에는 기본 도구를 사용하고 지속적인 복제에는 관리형 AWS DMS 서비스를 사용하십시오. 이 전략을 하이브리드 접근 방식이라고 합니다. https://docs.aws.amazon.com/dms/latest/sbs/chap-manageddatabases.postgresql-rds-postgresql.html
