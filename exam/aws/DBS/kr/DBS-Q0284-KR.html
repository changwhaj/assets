Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 284
주제 #: 1

한 회사에서 3,500개의 저장 프로시저가 있는 대규모 Oracle 데이터베이스 스키마를 Amazon Aurora PostgreSQL로 마이그레이션하는 작업을 하고 있습니다. 애플리케이션 개발자가 AWS Schema Conversion Tool(AWS SCT)을 사용하여 코드를 Oracle에서 Aurora PostgreSQL로 변환하고 있습니다. 그러나 코드 변환은 성능 문제로 인해 더 오랜 시간이 걸립니다. 애플리케이션 팀은 AWS SCT 변환 성능을 개선하기 위해 데이터베이스 전문가에게 연락했습니다.

데이터베이스 전문가는 성능 문제를 해결하기 위해 무엇을 해야 합니까?

  • A. AWS SCT에서 로컬 데스크톱의 최적 메모리 설정으로 메모리 소비 성능 균형 속도 옵션을 켭니다.
  • B. 더 높은 인스턴스 클래스로 대상 Aurora PostgreSQL 데이터베이스를 프로비저닝합니다. AWS SCT에서. 메모리 소비 성능 옵션으로 균형 속도를 켭니다.
  • C. AWS SCT에서 대용량 메모리 소비 성능 옵션으로 빠른 변환을 켜고 JavaOptions 섹션을 사용 가능한 최대 메모리로 설정합니다.
  • D. Aurora PostgreSQL 데이터베이스와 동일한 AWS 리전에서 더 많은 CPU 및 메모리 리소스로 클라이언트 Amazon EC2 시스템을 프로비저닝합니다.

선택한 답변: B
AWS SCT 변환의 성능 문제를 해결하려면 데이터베이스 전문가가 더 높은 인스턴스 클래스로 대상 Aurora PostgreSQL 데이터베이스를 프로비저닝하고 AWS SCT에서 메모리 소비 성능과 속도 균형 옵션을 켜야 합니다. 대상 Aurora PostgreSQL 데이터베이스의 인스턴스 클래스를 늘리면 처리 능력과 메모리가 향상되어 AWS SCT 변환을 보다 효율적으로 처리할 수 있기 때문에 옵션 B가 정답입니다. 또한 AWS SCT에서 메모리 소비 성능 옵션으로 균형 속도를 켜면 성능 및 메모리 사용에 대한 변환 프로세스가 최적화됩니다.
