Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 140
주제 #: 1

데이터베이스 전문가는 재해 복구 전략의 일환으로 미션 크리티컬 워크로드에서 Amazon DynamoDB 테이블의 야간 백업을 생성해야 합니다.
데이터베이스 전문가는 관리 오버헤드를 최소화하기 위해 어떤 백업 방법을 사용해야 합니까?

  • A. Amazon EC2 인스턴스에 AWS CLI를 설치합니다. DynamoDB 테이블의 백업을 생성하는 CLI 명령을 작성합니다. 밤마다 명령을 실행하는 예약된 작업 또는 작업을 만듭니다.
  • B. DynamoDB 테이블의 백업을 생성하는 AWS Lambda 함수를 생성합니다. 매일 밤 Lambda 함수를 실행하는 Amazon CloudWatch Events 규칙을 생성합니다.
  • C. AWS Backup을 사용하여 백업 계획을 만들고 24시간마다 백업 빈도를 지정하고 계획에 야간 백업 기간을 지정합니다. 최다 투표
  • D. 24시간마다 온디맨드 백업 빈도로 DynamoDB 백업 및 복원을 구성합니다.
추천 답변 보기 답변 숨기기
제안된 답변: D 🗳️
온디맨드 백업을 사용하면 데이터 보관을 위해 Amazon DynamoDB 테이블의 전체 백업을 생성할 수 있으므로 기업 및 정부 규제 요구 사항을 충족하는 데 도움이 됩니다. 프로덕션 애플리케이션의 성능 및 가용성에 영향을 주지 않고 몇 메가바이트에서 수백 테라바이트의 데이터 테이블을 백업할 수 있습니다. 백업은 테이블 크기에 관계없이 몇 초 만에 처리되므로 백업 일정이나 장기 실행 프로세스에 대해 걱정할 필요가 없습니다. 또한 모든 백업은 자동으로 암호화되고 카탈로그화되며 쉽게 검색할 수 있으며 명시적으로 삭제할 때까지 보관됩니다.
