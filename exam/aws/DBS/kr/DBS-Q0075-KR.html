한 회사에서 단일 MySQL 데이터베이스를 Amazon Aurora로 마이그레이션했습니다. 프로덕션 데이터는 VPC_PROD의 DB 클러스터에서 호스팅되고 12개의 테스트 환경은 동일한 AWS 계정을 사용하여 VPC_TEST에서 호스팅됩니다. 테스트 결과는 테스트 데이터에 대한 최소한의 변경입니다. 개발 팀은 각 테스트 데이터베이스가 매일 새로운 프로덕션 데이터를 포함하도록 각 환경을 매일 밤 새로 고치기를 원합니다.
구현하기에 가장 빠르고 비용 효율적인 마이그레이션 방법은 무엇입니까?

  • A. Amazon Aurora MySQL에서 마스터를 실행합니다. VPC_TEST에 12개의 복제본을 생성하고 매일 밤 삭제하고 다시 생성할 복제본을 스크립팅합니다. 최다 투표
  • B. Amazon Aurora MySQL에서 마스터를 실행합니다. 야간 스냅샷을 만들고 Aurora Serverless를 사용하여 VPC_TEST의 12개 데이터베이스로 복원합니다.
  • C. Amazon Aurora MySQL에서 마스터를 실행합니다. VPC_TEST에서 12개의 Aurora 복제본을 생성하고 매일 밤 삭제하고 다시 생성할 복제본을 스크립팅합니다.
  • D. Aurora Serverless를 사용하여 Amazon Aurora MySQL에서 마스터를 실행합니다. VPC_TEST에 12개의 복제본을 생성하고 매일 밤 삭제하고 다시 생성할 복제본을 스크립팅합니다. 최다 투표
