질문 #: 13
주제 #: 1

제조 회사의 웹 사이트는 Amazon Aurora PostgreSQL DB 클러스터를 사용합니다.
장애 조치 중에 애플리케이션 가동 중지 시간이 가장 적게 발생하는 구성은 무엇입니까? (3개를 선택하세요.)

  • A. 제공된 읽기 및 쓰기 Aurora 엔드포인트를 사용하여 Aurora DB 클러스터에 대한 연결을 설정합니다.
  • B. 기본 Aurora DB 클러스터에 연결할 수 없을 때 다른 가용 영역에서 복원을 트리거하는 Amazon CloudWatch 알림을 생성합니다.
  • C. 파라미터 그룹에서 Aurora DB 클러스터 캐시 관리를 편집하고 활성화합니다.
  • D. TCP keepalive 매개 변수를 높은 값으로 설정합니다.
  • E. JDBC 연결 문자열 시간 제한 변수를 낮은 값으로 설정합니다.
  • F. Java DNS 캐싱 시간 제한을 높은 값으로 설정합니다.

ACE 관련 C: Aurora PostgreSQL용 클러스터 캐시 관리를 통한 장애 조치 후 빠른 복구. 클러스터 캐시 관리를 통해 특정 리더 DB 인스턴스를 장애 조치 대상으로 설정합니다. 클러스터 캐시 관리는 지정된 리더 캐시의 데이터가 라이터 DB 인스턴스 캐시의 데이터와 동기화되도록 합니다. 값이 미리 채워진 지정된 판독기의 캐시를 웜 캐시라고 합니다. 장애 조치가 발생하면 지정된 리더는 새 라이터 DB 인스턴스로 승격되는 즉시 웜 캐시의 값을 사용합니다. 이 접근 방식은 애플리케이션에 훨씬 더 나은 복구 성능을 제공합니다. https://docs.aws.amazon.com/AmazonRDS/latest/AuroraUserGuide/AuroraPostgreSQL.cluster-cache-mgmt.html
