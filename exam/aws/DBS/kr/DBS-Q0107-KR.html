시험 AWS 공인 데이터베이스 - 전문 주제 1 질문 107 토론

Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 107
주제 #: 1

회사에서 미션 크리티컬 2TB Oracle 데이터베이스를 온프레미스에서 Amazon Aurora로 마이그레이션하고 있습니다. 데이터베이스 마이그레이션 비용은 최소한으로 유지되어야 하며 회사가 Aurora로 완전히 전환할 준비가 될 때까지 온프레미스 Oracle 데이터베이스와 Aurora DB 클러스터 모두 쓰기 트래픽을 위해 열려 있어야 합니다.
이 마이그레이션을 가능한 한 빨리 완료하기 위해 데이터베이스 전문가가 취해야 할 조치 조합은 무엇입니까? (두 가지를 선택하세요.)

  • A. AWS Schema Conversion Tool(AWS SCT)을 사용하여 원본 데이터베이스 스키마를 변환합니다. 그런 다음 변환된 스키마를 대상 Aurora DB 클러스터로 복원합니다. 최다 투표
  • B. Oracle의 Data Pump 도구를 사용하여 원본 데이터베이스 스키마의 복사본을 내보내고 텍스트 편집기에서 스키마를 수동으로 편집하여 Aurora와 호환되도록 합니다.
  • C. Oracle 데이터베이스에서 Aurora DB 클러스터로 데이터를 마이그레이션하는 AWS DMS 작업을 생성합니다. 회사에서 모든 사용자 트래픽을 Aurora DB 클러스터로 이동할 준비가 될 때까지 원본 및 대상 데이터베이스를 동기화 상태로 유지하기 위해 진행 중인 변경 사항을 복제할 마이그레이션 유형을 선택합니다. 최다 투표
  • D. Oracle 데이터베이스에서 Aurora DB 클러스터로 데이터를 마이그레이션하는 AWS DMS 작업을 생성합니다. 초기 로드가 완료되면 회사에서 모든 사용자 트래픽을 Aurora DB 클러스터로 이동할 준비가 될 때까지 AWS Kinesis Data Firehose 스트림을 생성하여 변경 데이터 캡처(CDC)를 수행합니다.
  • E. AWS Glue 작업 및 관련 리소스를 생성하여 Oracle 데이터베이스에서 Aurora DB 클러스터로 데이터를 마이그레이션합니다. 초기 로드가 완료되면 회사에서 모든 사용자 트래픽을 Aurora DB 클러스터로 이동할 준비가 될 때까지 변경 데이터 캡처(CDC)를 수행하는 AWS DMS 작업을 생성합니다.
