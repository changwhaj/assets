Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 84
주제 #: 1

회사에서 PostgreSQL용 Amazon RDS를 사용하고 있습니다. 보안 팀은 모든 데이터베이스 연결 요청이 기록되고 180일 동안 유지되기를 원합니다. RDS for PostgreSQL DB 인스턴스는 현재 기본 파라미터 그룹을 사용하고 있습니다. 데이터베이스 전문가는 log_connections 매개변수를 1로 설정하면 연결 로깅이 활성화됨을 확인했습니다.
로깅 및 보존 요구 사항을 충족하기 위해 데이터베이스 전문가가 수행해야 하는 단계 조합은 무엇입니까? (두 가지를 선택하세요.)

  • A. 기본 파라미터 그룹에서 log_connections 파라미터 업데이트
  • B. 사용자 지정 파라미터 그룹 생성, log_connections 파라미터 업데이트, 파라미터를 DB 인스턴스와 연결 최다 투표
  • C. Amazon CloudWatch Logs에 데이터베이스 엔진 로그 게시를 활성화하고 이벤트 만료를 180일로 설정 최다 투표
  • D. Amazon S3 버킷에 데이터베이스 엔진 로그 게시를 활성화하고 수명 주기 정책을 180일로 설정
  • E. RDS PostgreSQL 호스트에 연결하고 postgresql.conf 파일에서 log_connections 파라미터를 업데이트합니다.

B&C https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/USER_LogAccess.Concepts.PostgreSQL.html#USER_LogAccess.PostgreSQL.log_retention_period 시스템 로그의 보존 기간을 설정하려면 rds.log_retention_period 파라미터를 사용합니다. DB 인스턴스와 연결된 DB 파라미터 그룹에서 rds.log_retention_period를 찾을 수 있습니다. 이 매개변수의 단위는 분입니다. 예를 들어 1,440으로 설정하면 하루 동안 로그가 유지됩니다. 기본값은 4,320(3일)입니다. 최대값은 10,080(7일)입니다. 인스턴스에 보관된 로그 파일을 포함하기에 충분한 할당 스토리지가 있어야 합니다. 이전 로그를 유지하려면 Amazon CloudWatch Logs에 게시하십시오. 자세한 내용은 CloudWatch Logs에 PostgreSQL 로그 게시 단원을 참조하십시오. https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/USER_LogAccess.Concepts.PostgreSQL.html#USER_LogAccess.

B와 C. 잘못된 것: 파라미터 그룹 각 Amazon RDS PostgreSQL 인스턴스는 엔진별 구성을 포함하는 파라미터 그룹과 연결됩니다. 엔진 구성에는 PostgreSQL 로깅 동작을 제어하는 ​​여러 매개변수도 포함됩니다. AWS는 인스턴스에 사용할 기본 구성 설정을 파라미터 그룹에 제공합니다. 그러나 기본 설정을 변경하려면 기본 파라미터 그룹의 복제본을 생성하고 수정한 다음 인스턴스에 연결해야 합니다. DB 인스턴스에 대한 로깅 파라미터를 설정하려면 DB 파라미터 그룹에서 파라미터를 설정하고 해당 파라미터 그룹을 DB 인스턴스와 연결합니다. 자세한 내용은 DB 파라미터 그룹 작업 단원을 참조하십시오.
