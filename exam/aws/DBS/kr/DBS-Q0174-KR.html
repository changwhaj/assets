Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 174
주제 #: 1

전자상거래 회사는 핵심 애플리케이션 데이터베이스를 Amazon Aurora MySQL로 마이그레이션하고 있습니다. 이 회사는 현재 동시 데이터베이스 세션으로 온라인 트랜잭션 처리
(OLTP) 스트레스 테스트를 수행하고 있습니다. 첫 번째 테스트에서 데이터베이스 전문가는 일부 특정 쓰기 작업의 성능 저하를 발견했습니다.
Aurora DB 클러스터에 대한 Amazon CloudWatch 지표를 검토한 결과 CPU 사용률이 90%인 것으로 나타났습니다.
MOST가 높은 CPU 사용률과 느린 성능의 근본 원인을 효과적으로 식별하기 위해 데이터베이스 전문가가 수행해야 하는 단계는 무엇입니까? (두 가지를 선택하세요.)

  • A. 다음 테스트 라운드 전에 운영 체제 메트릭을 검토하려면 30초 미만의 세분성에서 Enhanced Monitoring을 활성화하십시오. 최다 투표
  • B. CloudWatch에서 VolumeBytesUsed 지표를 검토하여 쓰기 I/O가 급증하는지 확인합니다.
  • C. Amazon RDS 성능 개선 도우미를 검토하여 상위 SQL 문 및 대기 이벤트를 식별합니다. 최다 투표
  • D. AWS CloudTrail에서 Amazon RDS API 호출을 검토하여 장기 실행 쿼리를 식별합니다.
  • E. 다음 테스트 라운드 전에 Amazon CloudWatch에서 QUERY 이벤트를 기록하려면 Advance Auditing을 활성화하십시오.
