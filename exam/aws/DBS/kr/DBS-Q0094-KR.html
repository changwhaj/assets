Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 94
주제 #: 1

데이터베이스 전문가는 크기가 1GB 미만인 우편 번호 및 관련 지역 정보의 정적 공급업체 데이터 세트를 사용하는 시스템을 구축하고 있습니다. 데이터 세트는 시작 시 애플리케이션의 캐시에 로드됩니다. 회사는 낮은 애플리케이션 시작 시간으로 최저 비용을 제공하는 방식으로 이 데이터를 저장해야 합니다.
이러한 요구 사항을 충족하는 접근 방식은 무엇입니까?

  • A. Amazon RDS DB 인스턴스를 사용하십시오. 데이터를 읽은 후 인스턴스를 종료하십시오.
  • B. Amazon Aurora Serverless를 사용합니다. 필요에 따라 서비스가 리소스를 위아래로 회전하도록 허용합니다.
  • C. 온디맨드 용량 모드에서 Amazon DynamoDB를 사용합니다.
  • D. Amazon S3를 사용하고 플랫 파일에서 데이터를 로드합니다. 최다 투표

저는 아래 링크를 기반으로 D에 동의합니다. https://www.sumologic.com/insight/s3-cost-optimization/ 예를 들어, 1TB의 스토리지가 프로비저닝된 S3에 저장된 1GB 파일의 경우 1에 대한 요금이 청구됩니다. GB만 해당. Amazon EC2, Amazon Elastic Block Storage(Amazon EBS) 및 Amazon DynamoDB와 같은 다른 많은 서비스에서는 프로비저닝된 용량에 대해 비용을 지불합니다. 예를 들어 Amazon EBS 디스크의 경우 1GB 파일만 저장해도 1TB 디스크 크기에 해당하는 비용을 지불합니다. 따라서 Amazon EBS 및 Amazon EC2를 비롯한 다른 많은 서비스보다 S3 비용 관리가 더 쉽습니다. S3에서는 오버 프로비저닝의 위험이 없으며 디스크 사용률을 관리할 필요가 없습니다.
