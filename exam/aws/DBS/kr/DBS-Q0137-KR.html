시험 AWS 공인 데이터베이스 - 전문 주제 1 질문 137 토론

Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 137
주제 #: 1

한 회사가 사기 탐지 애플리케이션을 온프레미스에서 AWS 클라우드로 옮기고 데이터 스토리지에 Amazon Neptune을 사용하고 있습니다. 이 회사는 25TB의 사기 탐지 데이터를 온프레미스 데이터 센터에서 Neptune DB 인스턴스로 마이그레이션하기 위해 1Gbps AWS Direct Connect 연결을 설정했습니다. 회사에는 이미 Amazon S3 버킷과 S3 VPC 엔드포인트가 있으며 회사 네트워크 대역폭의 80%를 사용할 수 있습니다.
회사는 이 데이터 로드를 어떻게 수행해야 합니까?

  • A. 멀티파트 업로드와 함께 AWS SDK를 사용하여 온프레미스에서 S3 버킷으로 데이터를 전송합니다. Neptune에 대한 복사 명령을 사용하여 S3 버킷에서 Neptune DB 인스턴스로 데이터를 대량으로 이동합니다.
  • B. AWS DMS(AWS Database Migration Service)를 사용하여 온프레미스에서 S3 버킷으로 데이터를 전송합니다. Neptune용 Loader 명령을 사용하여 S3 버킷에서 Neptune DB 인스턴스로 데이터를 대량으로 이동합니다.
  • C. AWS DataSync를 사용하여 온프레미스에서 S3 버킷으로 데이터를 전송합니다. Neptune용 Loader 명령을 사용하여 S3 버킷에서 Neptune DB 인스턴스로 데이터를 대량으로 이동합니다.
  • D. AWS CLI를 사용하여 온프레미스에서 S3 버킷으로 데이터를 전송합니다. Neptune에 대한 복사 명령을 사용하여 S3 버킷에서 Neptune DB 인스턴스로 데이터를 대량으로 이동합니다.

DMS 대상은 S3일 수 없습니다. AWS DataSync는 온프레미스와 AWS 스토리지 서비스 간의 데이터 이동을 자동화하고 가속화하는 안전한 온라인 서비스입니다. C. AWS DataSync를 사용하여 온프레미스에서 S3 버킷으로 데이터를 전송합니다. Neptune용 Loader 명령을 사용하여 S3 버킷에서 Neptune DB 인스턴스로 데이터를 대량으로 이동합니다. aws DataSync는 온프레미스에서 S3로 데이터를 전송하는 방법이며 소스 db는 언급되지 않습니다. S3에서 AWS Neptune으로 데이터를 로드하는 유일한 방법은 https://docs.aws.amazon.com/neptune/latest/userguide/bulk-load.html 로더를 사용하는 것입니다.
