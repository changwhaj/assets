​​주제 #: 1

데이터베이스 전문가가 기존 프로덕션 MySQL 데이터베이스를 온프레미스에서 Amazon RDS for MySQL DB 인스턴스로 마이그레이션했습니다. 그러나 마이그레이션 후 데이터베이스는 AWS KMS를 사용하여 유휴 상태에서 암호화해야 했습니다. 데이터베이스 크기로 인해 데이터를 암호화된 데이터베이스로 다시 로드하는 데 너무 많은 시간이 소요되므로 선택 사항이 아닙니다.
데이터베이스 전문가는 이 새로운 요구 사항을 어떻게 충족해야 합니까?

  • A. 암호화되지 않은 RDS DB 인스턴스의 스냅샷을 생성합니다. 암호화되지 않은 스냅샷의 암호화된 복사본을 만듭니다. 암호화된 스냅샷 복사본을 복원합니다.
  • B. RDS DB 인스턴스를 수정합니다. AWS CLI를 활용하는 AWS KMS 암호화 옵션을 활성화합니다.
  • C. 암호화되지 않은 스냅샷을 암호화된 MySQL RDS DB 인스턴스로 복원합니다.
  • D. RDS DB 인스턴스의 암호화된 읽기 전용 복제본을 생성합니다. 그것을 주인으로 승격시키십시오.
