금융 서비스 회사는 TDE(투명한 데이터 암호화)와 함께 Amazon RDS for Oracle을 사용합니다. 회사는 미사용 데이터를 항상 암호화해야 합니다. 데이터를 해독하는 데 필요한 키는 가용성이 높아야 하며 키에 대한 액세스는 제한되어야 합니다. 규정 요구 사항에 따라 회사는 필요에 따라 암호화 키를 교체할 수 있어야 합니다. 회사는 잠재적인 보안 침해가 발견될 경우 키를 사용할 수 없게 만들 수 있어야 합니다. 회사는 또한 최소한의 오버헤드로 이러한 작업을 수행해야 합니다.
데이터베이스 관리자는 이러한 요구 사항을 충족하기 위해 암호화를 설정하기 위해 무엇을 사용해야 합니까?

  • B. AWS 관리형 키를 사용하는 AWS Key Management Service(AWS KMS)
  • C. 서버 측 암호화를 사용하는 AWS Key Management Service(AWS KMS)
  • D. 고객 제공 자료가 포함된 AWS Key Management Service(AWS KMS) CMK 최다 투표
