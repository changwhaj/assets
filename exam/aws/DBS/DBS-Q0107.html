Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 107
주제 #: 1

한 회사가 미션 크리티컬 2TB Oracle 데이터베이스를 온프레미스에서 Amazon Aurora로 마이그레이션하고 있습니다. 데이터베이스 마이그레이션 비용은 최소한으로 유지되어야 하며 회사가 Aurora로 완전히 전환할 준비가 될 때까지 온프레미스 Oracle 데이터베이스와 Aurora DB 클러스터는 모두 쓰기 트래픽을 위해 열려 있어야 합니다.
데이터베이스 전문가는 가능한 한 빨리 마이그레이션을 수행하기 위해 어떤 조치 조합을 취해야 합니까? (2개를 선택하세요.)

  • A. AWS SCT(AWS Schema Conversion Tool)를 사용하여 소스 데이터베이스 스키마를 변환합니다. 그런 다음 변환된 스키마를 대상 Aurora DB 클러스터로 복원합니다. 최다 투표
  • B. Oracle의 Data Pump 도구를 사용하여 소스 데이터베이스 스키마의 복사본을 내보내고 텍스트 편집기에서 스키마를 수동으로 편집하여 Aurora와 호환되도록 만듭니다.
  • C. Oracle 데이터베이스에서 Aurora DB 클러스터로 데이터를 마이그레이션하는 AWS DMS 작업을 생성합니다. 회사가 모든 사용자 트래픽을 Aurora DB 클러스터로 이동할 준비가 될 때까지 소스 및 대상 데이터베이스를 동기화 상태로 유지하기 위해 지속적인 변경 사항을 복제하려면 마이그레이션 유형을 선택하십시오. 최다 투표
  • D. Oracle 데이터베이스에서 Aurora DB 클러스터로 데이터를 마이그레이션하는 AWS DMS 작업을 생성합니다. 초기 로드가 완료되면 회사가 모든 사용자 트래픽을 Aurora DB 클러스터로 이동할 준비가 될 때까지 AWS Kinesis Data Firehose 스트림을 생성하여 변경 데이터 캡처(CDC)를 수행합니다.
  • E. Oracle 데이터베이스에서 Aurora DB 클러스터로 데이터를 마이그레이션하기 위해 AWS Glue 작업 및 관련 리소스를 생성합니다. 초기 로드가 완료되면 회사가 모든 사용자 트래픽을 Aurora DB 클러스터로 이동할 준비가 될 때까지 변경 데이터 캡처(CDC)를 수행하는 AWS DMS 작업을 생성합니다.

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 107
Topic #: 1

A company is migrating a mission-critical 2-TB Oracle database from on premises to Amazon Aurora. The cost for the database migration must be kept to a minimum, and both the on-premises Oracle database and the Aurora DB cluster must remain open for write traffic until the company is ready to completely cut over to Aurora.
Which combination of actions should a database specialist take to accomplish this migration as quickly as possible? (Choose two.)

  • A. Use the AWS Schema Conversion Tool (AWS SCT) to convert the source database schema. Then restore the converted schema to the target Aurora DB cluster. Most Voted
  • B. Use Oracle's Data Pump tool to export a copy of the source database schema and manually edit the schema in a text editor to make it compatible with Aurora.
  • C. Create an AWS DMS task to migrate data from the Oracle database to the Aurora DB cluster. Select the migration type to replicate ongoing changes to keep the source and target databases in sync until the company is ready to move all user traffic to the Aurora DB cluster. Most Voted
  • D. Create an AWS DMS task to migrate data from the Oracle database to the Aurora DB cluster. Once the initial load is complete, create an AWS Kinesis Data Firehose stream to perform change data capture (CDC) until the company is ready to move all user traffic to the Aurora DB cluster.
  • E. Create an AWS Glue job and related resources to migrate data from the Oracle database to the Aurora DB cluster. Once the initial load is complete, create an AWS DMS task to perform change data capture (CDC) until the company is ready to move all user traffic to the Aurora DB cluster.
