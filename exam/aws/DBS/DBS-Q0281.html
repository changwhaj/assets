주제 #: 1

한 회사는 온프레미스 Oracle 데이터베이스를 Oracle용 Amazon RDS로 마이그레이션했습니다. 데이터베이스 전문가는 데이터베이스의 대기 시간을 모니터링해야 합니다.

최소한의 운영 오버헤드로 이 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. RDS 성능 통찰력 지표를 Amazon CloudWatch에 게시합니다. AWS CloudTrail 필터를 추가하여 데이터베이스 성능 모니터링
  • B. Oracle Statspack을 설치합니다. 데이터베이스 성능을 모니터링하기 위해 성능 데이터를 수집, 저장 및 표시하려면 성능 통계 기능을 활성화하십시오.
  • C. RDS 성능 개선 도우미를 활성화하여 데이터베이스 로드를 시각화합니다. 다양한 스레드가 CPU를 어떻게 사용하는지 확인하려면 Enhanced Monitoring을 활성화하세요.
  • D. Al locatedStorage, DBInstanceClassMemory 및 DBInstanceVCPU 변수를 포함하는 새 DB 매개 변수 그룹을 생성합니다. RDS 성능 개선 도우미 활성화

  • B. Install Oracle Statspack. Enable the performance statistics feature to collect, store, and display performance data to monitor database performance.
  • C. Enable RDS Performance Insights to visualize the database load. Enable Enhanced Monitoring to view how different threads use the CPU
  • D. Create a new DB parameter group that includes the AllocatedStorage, DBInstanceClassMemory, and DBInstanceVCPU variables. Enable RDS Performance Insights
