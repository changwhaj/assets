주제 #: 1

데이터베이스 전문가는 기본 DB 파라미터 그룹을 사용하는 Amazon RDS DB 인스턴스 집합을 보유하고 있습니다. 데이터베이스 전문가는 사용자 지정 파라미터 그룹을 일부 DB 인스턴스와 연결해야 합니다.
데이터베이스 전문가가 이렇게 변경한 후 언제 인스턴스가 이 새 매개변수 그룹에 할당됩니까?

  • A. 파라미터 그룹 변경 후 즉시
  • B. 다음 예정된 DB 인스턴스 유지 관리 기간
  • C. DB 인스턴스를 수동으로 재부팅한 후 최다 투표
  • D. 파라미터 그룹 변경 후 24시간 이내
제안된 답변 표시 답변 숨기기
추천 답변: C 🗳️
해당 DB 인스턴스에 최신 파라미터 변경 사항을 적용하려면 DB 인스턴스를 수동으로 재부팅하세요.
참조:
https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/USER_WorkingWithParamGroups.html

Question #: 164
Topic #: 1

A database specialist has a fleet of Amazon RDS DB instances that use the default DB parameter group. The database specialist needs to associate a custom parameter group with some of the DB instances.
After the database specialist makes this change, when will the instances be assigned to this new parameter group?

  • A. Instantaneously after the change is made to the parameter group
  • B. In the next scheduled maintenance window of the DB instances
  • C. After the DB instances are manually rebooted Most Voted
  • D. Within 24 hours after the change is made to the parameter group
Show Suggested Answer Hide Answer
Suggested Answer: C 🗳️
To apply the latest parameter changes to that DB instance, manually reboot the DB instance.
Reference:
https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/USER_WorkingWithParamGroups.html

When you associate a new DB parameter group with a DB instance, the modified static and dynamic parameters are applied only after the DB instance is rebooted. However, if you modify dynamic parameters in the newly associated DB parameter group, these changes are applied immediately without a reboot.
