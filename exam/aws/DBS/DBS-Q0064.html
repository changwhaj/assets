질문 #: 64
주제 #: 1

금융 회사는 민감한 사용자 데이터를 Amazon Aurora PostgreSQL DB 클러스터에 저장하려고 합니다. 데이터베이스는 회사 전체의 여러 애플리케이션에서 액세스됩니다. 회사에서는 데이터베이스에 대한 모든 통신을 암호화하고 서버 ID를 검증하도록 규정했습니다. SSL 기반이 아닌 모든 연결은 데이터베이스에 대한 액세스를 허용하지 않아야 합니다.
이러한 요구 사항을 해결하는 솔루션은 무엇입니까?

  • A. DB 파라미터 그룹에서 rds.force_ssl=0 파라미터를 설정합니다. Amazon RDS 인증서 번들을 다운로드하여 사용하고 sslmode=allow로 PostgreSQL 연결 문자열을 구성합니다.
  • B. DB 파라미터 그룹에서 rds.force_ssl=1 파라미터를 설정합니다. Amazon RDS 인증서 번들을 다운로드하여 사용하고 sslmode=disable로 PostgreSQL 연결 문자열을 구성합니다.
  • C. DB 파라미터 그룹에서 rds.force_ssl=0 파라미터를 설정합니다. Amazon RDS 인증서 번들을 다운로드하여 사용하고 sslmode=verify-ca로 PostgreSQL 연결 문자열을 구성합니다.
  • D. DB 파라미터 그룹에서 rds.force_ssl=1 파라미터를 설정합니다. Amazon RDS 인증서 번들을 다운로드하여 사용하고 sslmode=verify-full로 PostgreSQL 연결 문자열을 구성합니다. 최다 투표

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 64
Topic #: 1

A financial company wants to store sensitive user data in an Amazon Aurora PostgreSQL DB cluster. The database will be accessed by multiple applications across the company. The company has mandated that all communications to the database be encrypted and the server identity must be validated. Any non-SSL- based connections should be disallowed access to the database.
Which solution addresses these requirements?

  • A. Set the rds.force_ssl=0 parameter in DB parameter groups. Download and use the Amazon RDS certificate bundle and configure the PostgreSQL connection string with sslmode=allow.
  • B. Set the rds.force_ssl=1 parameter in DB parameter groups. Download and use the Amazon RDS certificate bundle and configure the PostgreSQL connection string with sslmode=disable.
  • C. Set the rds.force_ssl=0 parameter in DB parameter groups. Download and use the Amazon RDS certificate bundle and configure the PostgreSQL connection string with sslmode=verify-ca.
  • D. Set the rds.force_ssl=1 parameter in DB parameter groups. Download and use the Amazon RDS certificate bundle and configure the PostgreSQL connection string with sslmode=verify-full. Most Voted
