Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 95
주제 #: 1

데이터베이스 전문가는 성능 문제가 있는 Amazon DynamoDB 테이블을 검토하고 최적화해야 합니다. 데이터베이스 전문가의 철저한 조사 결과 파티션 키로 인해 핫 파티션이 발생하여 새 파티션 키가 생성된 것으로 나타났습니다. 데이터베이스 전문가는 이 새로운 파티션 키를 모든 기존 데이터와 새 데이터에 효과적으로 적용해야 합니다.
이 솔루션을 어떻게 구현할 수 있나요?

  • A. Amazon EMR을 사용하여 현재 DynamoDB 테이블의 데이터를 Amazon S3로 내보냅니다. 그런 다음 Amazon EMR을 다시 사용하여 Amazon S3의 데이터를 새 파티션 키가 있는 새 DynamoDB 테이블로 가져옵니다. 최다 투표
  • B. AWS DMS를 사용하여 현재 DynamoDB 테이블의 데이터를 Amazon S3에 복사합니다. 그런 다음 DynamoDB 테이블을 가져와 새 파티션 키로 새 DynamoDB 테이블을 생성합니다.
  • C. AWS CLI를 사용하여 DynamoDB 테이블을 업데이트하고 파티션 키를 수정합니다.
  • D. AWS CLI를 사용하여 DynamoDB 테이블을 백업합니다. 그런 다음 Restore-table-from-backup 명령을 사용하고 파티션 키를 수정합니다.

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 95
Topic #: 1

A database specialist needs to review and optimize an Amazon DynamoDB table that is experiencing performance issues. A thorough investigation by the database specialist reveals that the partition key is causing hot partitions, so a new partition key is created. The database specialist must effectively apply this new partition key to all existing and new data.
How can this solution be implemented?

  • A. Use Amazon EMR to export the data from the current DynamoDB table to Amazon S3. Then use Amazon EMR again to import the data from Amazon S3 into a new DynamoDB table with the new partition key. Most Voted
  • B. Use AWS DMS to copy the data from the current DynamoDB table to Amazon S3. Then import the DynamoDB table to create a new DynamoDB table with the new partition key.
  • C. Use the AWS CLI to update the DynamoDB table and modify the partition key.
  • D. Use the AWS CLI to back up the DynamoDB table. Then use the restore-table-from-backup command and modify the partition key.
