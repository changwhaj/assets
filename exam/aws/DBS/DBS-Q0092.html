소규모 신생 기업에서 Amazon RDS for MySQL DB 인스턴스를 사용하여 4TB 온프레미스 MySQL 데이터베이스를 AWS로 마이그레이션하려고 합니다.
가동 중지 시간이 가장 적은 성공적인 마이그레이션을 허용하는 전략은 무엇입니까?

  • A. 새로운 RDS for MySQL DB 인스턴스를 배포하고 온프레미스 데이터 센터에서 액세스할 수 있도록 구성합니다. mysqldump 유틸리티를 사용하여 온프레미스 MySQL 서버에서 초기 스냅샷을 생성하고 Amazon S3 버킷에 복사합니다. Amazon EC2 인스턴스에서 실행되는 MySQL 유틸리티를 활용하여 스냅샷을 DB 인스턴스로 가져옵니다. 즉시 애플리케이션이 DB 인스턴스를 가리키도록 합니다.
  • B. 새 Amazon EC2 인스턴스를 배포하고 EC2 인스턴스에 MySQL 소프트웨어를 설치하고 온프레미스 데이터 센터에서 액세스할 수 있도록 네트워킹을 구성합니다. mysqldump 유틸리티를 사용하여 온프레미스 MySQL 서버의 스냅샷을 만듭니다. 스냅샷을 EC2 인스턴스에 복사하고 EC2 MySQL 인스턴스에 복원합니다. AWS DMS를 사용하여 MySQL DB 인스턴스용 새 RDS로 데이터를 마이그레이션합니다. 애플리케이션이 DB 인스턴스를 가리키도록 합니다.
  • C. 새 Amazon EC2 인스턴스를 배포하고 EC2 인스턴스에 MySQL 소프트웨어를 설치하고 온프레미스 데이터 센터에서 액세스할 수 있도록 네트워킹을 구성합니다. mysqldump 유틸리티를 사용하여 온프레미스 MySQL 서버의 스냅샷을 만듭니다. 스냅샷을 Amazon S3 버킷에 복사하고 EC2 인스턴스에서 실행되는 MySQL 유틸리티를 사용하여 MySQL DB 인스턴스용 새 RDS로 스냅샷을 가져옵니다. 애플리케이션이 DB 인스턴스를 가리키도록 합니다.
  • D. MySQL DB 인스턴스용 새 RDS를 배포하고 온프레미스 데이터 센터에서 액세스하도록 구성합니다. mysqldump 유틸리티를 사용하여 온프레미스 MySQL 서버에서 초기 스냅샷을 생성하고 Amazon S3 버킷에 복사합니다. Amazon EC2 인스턴스에서 실행되는 MySQL 유틸리티를 사용하여 스냅샷을 DB 인스턴스로 가져옵니다. MySQL 복제를 사용하여 새 DB 인스턴스로 복제를 설정합니다. 온프레미스 MySQL 서버에 대한 애플리케이션 액세스를 중지하고 나머지 트랜잭션이 복제되도록 합니다. 애플리케이션이 DB 인스턴스를 가리키도록 합니다. 최다 투표

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 92
Topic #: 1

A small startup company is looking to migrate a 4 TB on-premises MySQL database to AWS using an Amazon RDS for MySQL DB instance.
Which strategy would allow for a successful migration with the LEAST amount of downtime?

  • A. Deploy a new RDS for MySQL DB instance and configure it for access from the on-premises data center. Use the mysqldump utility to create an initial snapshot from the on-premises MySQL server, and copy it to an Amazon S3 bucket. Import the snapshot into the DB instance utilizing the MySQL utilities running on an Amazon EC2 instance. Immediately point the application to the DB instance.
  • B. Deploy a new Amazon EC2 instance, install the MySQL software on the EC2 instance, and configure networking for access from the on-premises data center. Use the mysqldump utility to create a snapshot of the on-premises MySQL server. Copy the snapshot into the EC2 instance and restore it into the EC2 MySQL instance. Use AWS DMS to migrate data into a new RDS for MySQL DB instance. Point the application to the DB instance.
  • C. Deploy a new Amazon EC2 instance, install the MySQL software on the EC2 instance, and configure networking for access from the on-premises data center. Use the mysqldump utility to create a snapshot of the on-premises MySQL server. Copy the snapshot into an Amazon S3 bucket and import the snapshot into a new RDS for MySQL DB instance using the MySQL utilities running on an EC2 instance. Point the application to the DB instance.
  • D. Deploy a new RDS for MySQL DB instance and configure it for access from the on-premises data center. Use the mysqldump utility to create an initial snapshot from the on-premises MySQL server, and copy it to an Amazon S3 bucket. Import the snapshot into the DB instance using the MySQL utilities running on an Amazon EC2 instance. Establish replication into the new DB instance using MySQL replication. Stop application access to the on-premises MySQL server and let the remaining transactions replicate over. Point the application to the DB instance. Most Voted
