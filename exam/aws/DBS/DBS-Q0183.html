질문 #: 183
주제 #: 1

한 회사가 SQL Server DB 인스턴스용 Amazon RDS의 데이터베이스를 한 AWS 리전에서 다른 AWS 리전으로 마이그레이션하고 있습니다. 회사는 마이그레이션 중에 데이터베이스 가동 중지 시간을 최소화하려고 합니다.
이 지역 간 마이그레이션을 위해 회사는 어떤 전략을 선택해야 합니까?

  • A. 동일한 지역의 Amazon S3 버킷에 대한 기본 백업을 사용하여 소스 데이터베이스를 백업하십시오. 그런 다음 대상 지역에서 백업을 복원합니다.
  • B. 동일한 지역의 Amazon S3 버킷에 대한 기본 백업을 사용하여 소스 데이터베이스를 백업합니다. Amazon S3 교차 리전 복제를 사용하여 대상 리전의 S3 버킷에 백업을 복사합니다. 그런 다음 대상 지역에서 백업을 복원합니다.
  • C. 소스 데이터베이스와 대상 데이터베이스 간에 데이터를 복제하도록 AWS Database Migration Service(AWS DMS)를 구성합니다. 복제가 동기화되면 DMS 작업을 종료합니다. 최다 투표
  • D. 대상 지역에 SQL Server 교차 지역 읽기 전용 복제본용 RDS를 추가합니다. 복제가 동기화되면 읽기 전용 복제본을 마스터로 승격합니다. 최다 투표
제안된 답변 표시 답변 숨기기
제안 답변: A 🗳️
Amazon RDS는 전체 백업 파일(.bak 파일)을 사용하여 Microsoft SQL Server 데이터베이스에 대한 기본 백업 및 복원을 지원합니다. RDS를 사용하면 데이터베이스 서버의 로컬 파일 시스템을 사용하는 대신 Amazon S3에 저장된 파일에 액세스합니다.
참조:
https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/SQLServer.Procedural.Importing.html

Question #: 183
Topic #: 1

A company is migrating a database in an Amazon RDS for SQL Server DB instance from one AWS Region to another. The company wants to minimize database downtime during the migration.
Which strategy should the company choose for this cross-Region migration?

  • A. Back up the source database using native backup to an Amazon S3 bucket in the same Region. Then restore the backup in the target Region.
  • B. Back up the source database using native backup to an Amazon S3 bucket in the same Region. Use Amazon S3 Cross-Region Replication to copy the backup to an S3 bucket in the target Region. Then restore the backup in the target Region.
  • C. Configure AWS Database Migration Service (AWS DMS) to replicate data between the source and the target databases. Once the replication is in sync, terminate the DMS task. Most Voted
  • D. Add an RDS for SQL Server cross-Region read replica in the target Region. Once the replication is in sync, promote the read replica to master. Most Voted
Show Suggested Answer Hide Answer
Suggested Answer: A 🗳️
Amazon RDS supports native backup and restore for Microsoft SQL Server databases using full backup files (.bak files). When you use RDS, you access files stored in Amazon S3 rather than using the local file system on the database server.
Reference:
https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/SQLServer.Procedural.Importing.html
