Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 99
주제 #: 1

회사는 Amazon EC2 인스턴스에서 실행되는 Oracle Database Standard Edition을 다중 AZ가 포함된 Oracle DB 인스턴스용 Amazon RDS로 마이그레이션해야 합니다.
데이터베이스는 지속적으로 실행되는 전자상거래 웹사이트를 지원합니다. 회사는 최대 5분의 유지 관리 기간만 제공할 수 있습니다.
어떤 솔루션이 이러한 요구 사항을 충족합니까?

  • A. EC2 인스턴스와 RDS DB 인스턴스에 Oracle Real Application Clusters(RAC)를 구성합니다. RAC 클러스터를 가리키도록 연결 문자열을 업데이트합니다. EC2 인스턴스와 RDS DB 인스턴스가 동기화되면 Amazon EC2에서 Amazon RDS로 장애 조치를 수행합니다.
  • B. Oracle Data Pump를 사용하여 EC2 인스턴스에서 Oracle 데이터베이스를 내보내고 Amazon RDS로 가져오기를 수행합니다. 전체 프로세스에 대해 응용 프로그램을 중지합니다. 가져오기가 완료되면 데이터베이스 연결 문자열을 변경한 다음 애플리케이션을 다시 시작합니다.
  • C. EC2 인스턴스를 소스로, RDS DB 인스턴스를 대상으로 AWS DMS를 구성합니다. 복제가 동기화되면 애플리케이션을 중지하고 데이터베이스 연결 문자열을 변경한 다음 애플리케이션을 다시 시작합니다. 최다 투표
  • D. EC2 인스턴스를 소스로, RDS DB 인스턴스를 대상으로 AWS DataSync를 구성합니다. 복제가 동기화되면 애플리케이션을 중지하고 데이터베이스 연결 문자열을 변경한 다음 애플리케이션을 다시 시작합니다.

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 99
Topic #: 1

A company needs to migrate Oracle Database Standard Edition running on an Amazon EC2 instance to an Amazon RDS for Oracle DB instance with Multi-AZ.
The database supports an ecommerce website that runs continuously. The company can only provide a maintenance window of up to 5 minutes.
Which solution will meet these requirements?

  • A. Configure Oracle Real Application Clusters (RAC) on the EC2 instance and the RDS DB instance. Update the connection string to point to the RAC cluster. Once the EC2 instance and RDS DB instance are in sync, fail over from Amazon EC2 to Amazon RDS.
  • B. Export the Oracle database from the EC2 instance using Oracle Data Pump and perform an import into Amazon RDS. Stop the application for the entire process. When the import is complete, change the database connection string and then restart the application.
  • C. Configure AWS DMS with the EC2 instance as the source and the RDS DB instance as the destination. Stop the application when the replication is in sync, change the database connection string, and then restart the application. Most Voted
  • D. Configure AWS DataSync with the EC2 instance as the source and the RDS DB instance as the destination. Stop the application when the replication is in sync, change the database connection string, and then restart the application.
