Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 55
주제 #: 1

데이터베이스 전문가는 온프레미스 Oracle 데이터베이스를 Amazon Aurora MySQL DB 클러스터로 마이그레이션하기 위한 데이터베이스 마이그레이션 전략을 정의해야 합니다. 회사에서는 데이터 마이그레이션을 위해 가동 중지 시간이 거의 0에 가깝습니다. 또한 솔루션은 비용 효율적이어야 합니다.
데이터베이스 전문가는 어떤 접근 방식을 취해야 합니까?

  • A. datapump(expdp)를 사용하여 Oracle 데이터베이스의 모든 테이블을 Amazon S3 버킷으로 덤프합니다. AWS Glue에서 데이터 변환을 실행합니다. S3 버킷의 데이터를 Aurora DB 클러스터로 로드합니다.
  • B. AWS Snowball 어플라이언스를 주문하고 Oracle 백업을 Snowball 어플라이언스에 복사합니다. Snowball 데이터가 Amazon S3에 전달되면 새 Aurora DB 클러스터를 생성합니다. Amazon S3에서 Amazon RDS로 데이터를 직접 마이그레이션하려면 S3 통합을 활성화합니다.
  • C. AWS SCT(AWS Schema Conversion Tool)를 사용하면 스키마 마이그레이션 중에 데이터베이스 객체를 MySQL에 다시 쓸 수 있습니다. AWS DMS를 사용하여 전체 로드 및 변경 데이터 캡처(CDC) 작업을 수행합니다. 최다 투표
  • D. AWS Server Migration Service(AWS SMS)를 사용하여 Oracle 가상 머신 이미지를 Amazon EC2 인스턴스로 가져옵니다. Oracle Logical Dump 유틸리티를 사용하여 Oracle 데이터를 Amazon EC2에서 Aurora DB 클러스터로 마이그레이션합니다.

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 55
Topic #: 1

A Database Specialist needs to define a database migration strategy to migrate an on-premises Oracle database to an Amazon Aurora MySQL DB cluster. The company requires near-zero downtime for the data migration. The solution must also be cost-effective.
Which approach should the Database Specialist take?

  • A. Dump all the tables from the Oracle database into an Amazon S3 bucket using datapump (expdp). Run data transformations in AWS Glue. Load the data from the S3 bucket to the Aurora DB cluster.
  • B. Order an AWS Snowball appliance and copy the Oracle backup to the Snowball appliance. Once the Snowball data is delivered to Amazon S3, create a new Aurora DB cluster. Enable the S3 integration to migrate the data directly from Amazon S3 to Amazon RDS.
  • C. Use the AWS Schema Conversion Tool (AWS SCT) to help rewrite database objects to MySQL during the schema migration. Use AWS DMS to perform the full load and change data capture (CDC) tasks. Most Voted
  • D. Use AWS Server Migration Service (AWS SMS) to import the Oracle virtual machine image as an Amazon EC2 instance. Use the Oracle Logical Dump utility to migrate the Oracle data from Amazon EC2 to an Aurora DB cluster.
