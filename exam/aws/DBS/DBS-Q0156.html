주제 #: 1

한 회사가 하나의 AWS 계정에서 2계층 전자상거래 애플리케이션을 실행하고 있습니다. 이 애플리케이션은 MySQL 다중 AZ DB 인스턴스용 Amazon RDS의 지원을 받습니다. 개발자가 프로덕션 환경에서 실수로 DB 인스턴스를 삭제했습니다. 회사는 데이터베이스를 복원하지만 이 사건으로 인해 몇 시간의 다운타임이 발생하고 수익 손실이 발생합니다.
앞으로 이러한 실수가 발생할 위험을 최소화할 수 있는 변경 사항 조합은 무엇입니까? (3개를 선택하세요.)

  • A. 그룹, IAM 사용자 및 역할에 최소한의 권한을 부여하십시오. 최다 투표
  • B. 모든 사용자가 백업에서 데이터베이스를 복원할 수 있도록 허용합니다.
  • C. 기존 프로덕션 DB 인스턴스에 대한 삭제 보호를 활성화합니다. 최다 투표
  • D. ACL 정책을 사용하여 사용자의 DB 인스턴스 삭제를 제한합니다. 최다 투표
  • E. AWS CloudTrail 로깅 및 향상된 모니터링을 활성화합니다.

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 156
Topic #: 1

A company is running a two-tier ecommerce application in one AWS account. The application is backed by an Amazon RDS for MySQL Multi-AZ DB instance. A developer mistakenly deleted the DB instance in the production environment. The company restores the database, but this event results in hours of downtime and lost revenue.
Which combination of changes would minimize the risk of this mistake occurring in the future? (Choose three.)

  • A. Grant least privilege to groups, IAM users, and roles. Most Voted
  • B. Allow all users to restore a database from a backup.
  • C. Enable deletion protection on existing production DB instances. Most Voted
  • D. Use an ACL policy to restrict users from DB instance deletion. Most Voted
  • E. Enable AWS CloudTrail logging and Enhanced Monitoring.
