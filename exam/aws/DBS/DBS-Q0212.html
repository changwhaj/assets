Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 212
주제 #: 1

한 회사에 MySQL용 Amazon RDS DB 인스턴스가 포함된 데이터베이스 집합이 있습니다. 감사 과정에서 회사는 DB 인스턴스에 저장된 데이터가 암호화되지 않은 것을 발견했습니다.
데이터베이스 전문가가 DB 인스턴스에 대한 암호화를 활성화해야 합니다. 데이터베이스 전문가는 DB 인스턴스에 대한 모든 연결도 암호화해야 합니다.
이러한 요구 사항을 충족하기 위해 데이터베이스 전문가는 어떤 조치 조합을 취해야 합니까? (3개를 선택하세요.)

  • A. RDS 콘솔에서 AWS Key Management Service(AWS KMS) 키를 사용하여 DB 인스턴스를 암호화하려면 '암호화 활성화'를 선택합니다.
  • B. AWS Key Management Service(AWS KMS) 키를 사용하여 암호화되지 않은 DB 인스턴스의 읽기 전용 복제본을 암호화합니다. 읽기 전용 복제본을 기본 DB 인스턴스로 장애 조치합니다.
  • C. 암호화되지 않은 DB 인스턴스의 스냅샷을 생성합니다. AWS Key Management Service(AWS KMS) 키를 사용하여 스냅샷을 암호화합니다. 암호화된 스냅샷에서 DB 인스턴스를 복원합니다. 원래 DB 인스턴스를 삭제합니다. 최다 투표
  • D. 해당 데이터베이스 사용자 계정에 SSL 연결이 필요합니다. 최다 투표
  • E. 애플리케이션에서 SSL/TLS를 사용하여 DB 인스턴스에 대한 연결을 암호화합니다. 최다 투표
  • F. DB 인스턴스에서 SSH 암호화를 활성화합니다.

A database specialist must enable encryption for the DB instance. The database specialist also must encrypt all connections to the DB instance.
Which combination of actions should the database specialist take to meet these requirements? (Choose three.)

  • A. In the RDS console, choose ג€Enable encryptionג€ to encrypt the DB instance by using an AWS Key Management Service (AWS KMS) key.
  • B. Encrypt the read replica of the unencrypted DB instance by using an AWS Key Management Service (AWS KMS) key. Fail over the read replica to the primary DB instance.
  • C. Create a snapshot of the unencrypted DB instance. Encrypt the snapshot by using an AWS Key Management Service (AWS KMS) key. Restore the DB instance from the encrypted snapshot. Delete the original DB instance. Most Voted
  • D. Require SSL connections for applicable database user accounts. Most Voted
  • E. Use SSL/TLS from the application to encrypt a connection to the DB instance. Most Voted
  • F. Enable SSH encryption on the DB instance.
