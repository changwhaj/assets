Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 158
주제 #: 1

한 회사가 SQL Server DB 인스턴스용으로 새로운 Amazon RDS를 설정하고 있습니다. 회사는 데이터베이스에서 SQL Server 감사를 활성화하려고 합니다.
이 요구 사항을 충족하려면 데이터베이스 전문가가 수행해야 하는 단계 조합은 무엇입니까? (2개를 선택하세요.)

  • A. Amazon RDS가 Amazon S3에 감사 로그를 저장할 수 있는 권한을 부여하는 Amazon RDS용 서비스 연결 역할을 생성합니다. 최다 투표
  • B. 감사 로그 저장을 위해 IAM 역할과 Amazon S3 버킷을 구성하기 위한 매개변수 그룹을 설정합니다. 파라미터 그룹을 DB 인스턴스와 연결합니다.
  • C. DB 인스턴스에서 다중 AZ를 비활성화한 다음 감사를 활성화합니다. 감사가 활성화된 후 다중 AZ를 활성화합니다.
  • D. DB 인스턴스에서 자동 백업을 비활성화한 다음 감사를 활성화합니다. 감사가 활성화된 후 자동 백업을 활성화합니다.
  • E. 감사 로그 저장을 위한 IAM 역할 및 Amazon S3 버킷을 구성하기 위한 옵션 그룹을 설정합니다. 옵션 그룹을 DB 인스턴스와 연결합니다. 최다 투표
제안된 답변 표시 답변 숨기기
제안 답변: AE 🗳️
이렇게 하려면 Amazon RDS 서비스가 Amazon S3 버킷을 사용할 수 있도록 IAM 역할을 생성하고 권한을 위임해야 합니다.
RDS는 제공된 IAM 역할을 사용하여 완료된 감사 로그를 S3 버킷에 업로드합니다. 보존을 활성화하면 RDS는 구성된 기간 동안 DB 인스턴스에 감사 로그를 보관합니다.
참조:
https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/Appendix.SQLServer.Options.Audit.html

Question #: 158
Topic #: 1

A company is setting up a new Amazon RDS for SQL Server DB instance. The company wants to enable SQL Server auditing on the database.
Which combination of steps should a database specialist take to meet this requirement? (Choose two.)

  • A. Create a service-linked role for Amazon RDS that grants permissions for Amazon RDS to store audit logs on Amazon S3. Most Voted
  • B. Set up a parameter group to configure an IAM role and an Amazon S3 bucket for audit log storage. Associate the parameter group with the DB instance.
  • C. Disable Multi-AZ on the DB instance, and then enable auditing. Enable Multi-AZ after auditing is enabled.
  • D. Disable automated backup on the DB instance, and then enable auditing. Enable automated backup after auditing is enabled.
  • E. Set up an options group to configure an IAM role and an Amazon S3 bucket for audit log storage. Associate the options group with the DB instance. Most Voted
Show Suggested Answer Hide Answer
Suggested Answer: AE 🗳️
To do this, you create an IAM role and delegate permissions so that the Amazon RDS service can use your Amazon S3 bucket.
RDS uploads the completed audit logs to your S3 bucket, using the IAM role that you provide. If you enable retention, RDS keeps your audit logs on your DB instance for the configured period of time.
Reference:
https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/Appendix.SQLServer.Options.Audit.html
