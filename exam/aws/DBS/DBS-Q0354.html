질문 #: 354
주제 #: 1

회사에는 AWS CloudFormation을 사용하는 CI/CD 프로세스의 일부로 Amazon Aurora PostgreSQL 데이터베이스를 배포하는 애플리케이션 환경이 있습니다. 회사의 데이터베이스 관리자는 결과 데이터베이스에서 성능 문제에 대한 보고를 받았지만 문제를 조사할 방법이 없습니다.

성능 데이터 수집을 자동화하기 위해 데이터베이스 관리자는 데이터베이스 배포에 어떤 변경 사항을 적용해야 합니까? (2개를 선택하세요.)

  • A. CloudFormation 템플릿에서 Aurora 데이터베이스 리소스에 대해 Amazon DevOps Guru를 활성화합니다.
  • B. 각 AWS 계정에서 AWS CloudTrail을 활성화합니다.
  • C. 모든 Aurora PostgreSQL 데이터베이스에 대해 AWS Config를 활성화하고 구성합니다.
  • D. Aurora PostgreSQL DB 인스턴스에서 Amazon CloudWatch 모니터링을 활성화하려면 CloudFormation 템플릿을 업데이트하십시오.
  • E. CloudFormation 템플릿을 업데이트하여 Aurora PostgreSQL에 대한 성능 개선 도우미를 활성화합니다.

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 354
Topic #: 1

A company has an application environment that deploys Amazon Aurora PostgreSQL databases as part of its CI/CD process that uses AWS CloudFormation. The company's database administrator has received reports of performance issues from the resulting database but has no way to investigate the issues.

Which combination of changes must the database administrator make to the database deployment to automate the collection of performance data? (Choose two.)

  • A. Turn on Amazon DevOps Guru for the Aurora database resources in the CloudFormation template.
  • B. Turn on AWS CloudTrail in each AWS account.
  • C. Turn on and configure AWS Config for all Aurora PostgreSQL databases.
  • D. Update the CloudFormation template to enable Amazon CloudWatch monitoring on the Aurora PostgreSQL DB instances.
  • E. Update the CloudFormation template to turn on Performance Insights for Aurora PostgreSQL.

D. Aurora PostgreSQL DB 인스턴스에서 Amazon CloudWatch 모니터링을 활성화하려면 CloudFormation 템플릿을 업데이트하십시오. Amazon CloudWatch는 AWS 리소스 및 애플리케이션에 대한 모니터링, 지표 수집 및 추적, 로그 파일 수집 및 모니터링, 경보 설정 기능을 제공합니다. 리소스 활용도, 애플리케이션 성능, 운영 상태에 대한 시스템 전반의 가시성을 확보하는 데 사용할 수 있습니다. E. CloudFormation 템플릿을 업데이트하여 Aurora PostgreSQL에 대한 성능 개선 도우미를 활성화합니다. 성능 개선 도우미는 AWS RDS(Aurora 포함)의 데이터베이스 성능 튜닝 및 모니터링 기능입니다. 이는 데이터베이스의 로드를 신속하게 평가하고 조치를 취할 시기와 장소를 결정하는 데 도움이 됩니다.

D. Update the CloudFormation template to enable Amazon CloudWatch monitoring on the Aurora PostgreSQL DB instances. Amazon CloudWatch provides monitoring for AWS resources and applications, collecting and tracking metrics, collecting and monitoring log files, and setting alarms. It can be used to gain system-wide visibility into resource utilization, application performance, and operational health. E. Update the CloudFormation template to turn on Performance Insights for Aurora PostgreSQL. Performance Insights is a database performance tuning and monitoring feature of AWS RDS (including Aurora). It helps you quickly assess the load on your database, and determine when and where to take action.
