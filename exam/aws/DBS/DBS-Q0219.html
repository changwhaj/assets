  • D. Configure an Amazon CloudWatch alarm for RDS metrics about changes to DB security groups.

정답은 다음과 같습니다. C. AWS CloudTrail을 켭니다. DB 보안 그룹에 대한 변경 사항 감지를 위한 알림을 구성합니다. 설명: AWS CloudTrail은 Amazon RDS 콘솔의 호출 및 Amazon RDS API에 대한 코드 호출을 포함하여 Amazon RDS에 대한 모든 API 호출을 이벤트로 캡처합니다. 추적을 생성하면 RDS용 이벤트를 포함하여 CloudTrail 이벤트를 Amazon S3 버킷으로 지속적으로 전달할 수 있습니다. 그런 다음 Amazon EventBridge(이전의 CloudWatch Events)를 사용하여 RDS 보안 그룹의 수정과 같은 AWS 리소스의 변경 사항을 감지하고 대응할 수 있습니다. 옵션 A, B, D가 올바르지 않음: A. RDS 이벤트 알림 구독 구성은 DB 보안 그룹 변경 사항을 다루지 않으며 DB 인스턴스 상태 변경, 장애 조치 등을 대상으로 합니다.

The correct answer is: C. Turn on AWS CloudTrail. Configure notifications for the detection of changes to DB security groups. Explanation: AWS CloudTrail captures all API calls for Amazon RDS as events, including calls from the Amazon RDS console and from code calls to the Amazon RDS APIs. If you create a trail, you can enable continuous delivery of CloudTrail events to an Amazon S3 bucket, including events for RDS. You can then use Amazon EventBridge (formerly known as CloudWatch Events) to detect and react to changes on your AWS resources like the modifications in your RDS Security Group. Options A, B, and D are incorrect: A. Configuring an RDS event notification subscription will not cover DB security group changes, it's more targeted towards DB instance state changes, failover etc.
