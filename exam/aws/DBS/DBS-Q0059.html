Amazon의 AWS 공인 데이터베이스 - 전문 분야 시험 문제
질문 #: 59
주제 #: 1

한 IT 컨설팅 회사에서 개발 환경 데이터베이스를 운영할 때 비용을 절감하려고 합니다. 회사의 워크플로는
각 개발 그룹에 대해 여러 Amazon Aurora MySQL DB 클러스터를 생성합니다. Aurora DB 클러스터는 하루 8시간 동안만 사용됩니다. 그런 다음 2주간 지속되는 개발 주기가 끝나면 DB 클러스터를 삭제할 수 있습니다.
다음 중 가장 비용 효율적인 솔루션을 제공하는 것은 무엇입니까?

  • A. AWS CloudFormation 템플릿을 사용하십시오. 각 개발 그룹에 DB 클러스터가 포함된 스택을 배포합니다. 개발 주기가 끝나면 스택을 삭제합니다.
  • B. Aurora DB 복제 기능을 사용하십시오. 단일 개발을 배포하고 Aurora DB 인스턴스를 테스트하고 개발 그룹을 위한 복제 인스턴스를 생성합니다. 개발 주기가 끝나면 클론을 삭제합니다. 최다 투표
  • C. Aurora 복제본을 사용하십시오. 마스터 자동 일시 중지 컴퓨팅 용량 옵션에서 각 개발 그룹에 대한 복제본을 생성하고 각 복제본을 마스터로 승격합니다. 개발 주기가 끝나면 복제본을 삭제합니다.
  • D. Aurora Serverless를 사용하십시오. 현재 Aurora 스냅샷을 복원하고 각 개발 그룹의 서버리스 클러스터에 배포합니다. 클러스터의 컴퓨팅 용량을 일시 중지하고 적절한 시간 제한을 설정하는 옵션을 활성화합니다. 최다 투표

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 59
Topic #: 1

An IT consulting company wants to reduce costs when operating its development environment databases. The company's workflow creates multiple Amazon
Aurora MySQL DB clusters for each development group. The Aurora DB clusters are only used for 8 hours a day. The DB clusters can then be deleted at the end of the development cycle, which lasts 2 weeks.
Which of the following provides the MOST cost-effective solution?

  • A. Use AWS CloudFormation templates. Deploy a stack with the DB cluster for each development group. Delete the stack at the end of the development cycle.
  • B. Use the Aurora DB cloning feature. Deploy a single development and test Aurora DB instance, and create clone instances for the development groups. Delete the clones at the end of the development cycle. Most Voted
  • C. Use Aurora Replicas. From the master automatic pause compute capacity option, create replicas for each development group, and promote each replica to master. Delete the replicas at the end of the development cycle.
  • D. Use Aurora Serverless. Restore current Aurora snapshot and deploy to a serverless cluster for each development group. Enable the option to pause the compute capacity on the cluster and set an appropriate timeout. Most Voted
Show Suggested Answer Hide Answer
