질문 #: 150
주제 #: 1

금융 회사는 최근 90일 동안의 MySQL 데이터베이스 백업이 사용 가능한지 확인해야 합니다. 모든 MySQL 데이터베이스는
MySQL DB 인스턴스용 Amazon RDS에서 호스팅됩니다. 데이터베이스 전문가는 최소한의 개발 노력으로 백업 보존 요구 사항을 충족하는 솔루션을 구현해야 합니다.
데이터베이스 전문가는 어떤 접근 방식을 취해야 합니까?

  • A. AWS Backup을 사용하여 필요한 보존 기간에 대한 백업 계획을 수립하십시오. DB 인스턴스를 백업 계획에 할당합니다. 최다 투표
  • B. 자동 백업 옵션을 활성화하도록 DB 인스턴스를 수정합니다. 필요한 백업 보존 기간을 선택합니다.
  • C. Amazon EC2 인스턴스에서 일일 크론 작업을 자동화하여 MySQL 덤프를 생성하고, Amazon S3로 전송하고, S3 수명 주기 정책을 구현하여 보존 요구 사항을 충족합니다.
  • D. AWS Lambda를 사용하여 DB 인스턴스의 일일 수동 스냅샷을 예약합니다. 보존 요구 사항을 초과하는 스냅샷을 삭제합니다.

Topic #: 1

A finance company needs to make sure that its MySQL database backups are available for the most recent 90 days. All of the MySQL databases are hosted on
Amazon RDS for MySQL DB instances. A database specialist must implement a solution that meets the backup retention requirement with the least possible development effort.
Which approach should the database specialist take?

  • A. Use AWS Backup to build a backup plan for the required retention period. Assign the DB instances to the backup plan. Most Voted
  • B. Modify the DB instances to enable the automated backup option. Select the required backup retention period.
  • C. Automate a daily cron job on an Amazon EC2 instance to create MySQL dumps, transfer to Amazon S3, and implement an S3 Lifecycle policy to meet the retention requirement.
  • D. Use AWS Lambda to schedule a daily manual snapshot of the DB instances. Delete snapshots that exceed the retention requirement.

You can set the backup retention period when you create a DB instance. If you don't set the backup retention period, the default backup retention period is one day if you create the DB instance using the Amazon RDS API or the AWS CLI.
