주제 #: 1

데이터베이스 관리자가 Amazon DynamoDB를 사용하는 애플리케이션 배포를 검토하고 있습니다. Amazon EC2 애플리케이션 인스턴스 집합이 데이터베이스에 액세스합니다.

데이터베이스 관리자는 EC2 인스턴스가 퍼블릭 IP 주소를 사용하여 데이터베이스에 액세스하고 있으며 데이터베이스를 인터넷에서 사용할 수 있음을 확인합니다. 회사 정책에 따르면 모든 회사 데이터는 비공개로 액세스해야 하며 인터넷을 통한 외부 액세스는 허용되지 않습니다.

DynamoDB 데이터베이스가 이러한 요구 사항을 충족하도록 보장하는 단계 조합은 무엇입니까? (2개를 선택하세요.)

  • A. 외부 액세스를 차단하도록 DynamoDB 보안 그룹과 네트워크 ACL을 구성합니다.
  • B. DynamoD용 AWS PrivateLink VPC 엔드포인트를 생성하고 VPC 라우팅 테이블을 업데이트합니다.
  • C. DynamoDB용 게이트웨이 VPC 엔드포인트를 생성합니다. VPC 라우팅 테이블을 업데이트합니다.
  • D. DynamoDB에 액세스하기 위해 NAT 게이트웨이를 프로비저닝합니다. VPC 라우팅 테이블을 업데이트합니다.
  • E. 테이블에 대한 액세스 권한을 제공하는 모든 IAM 역할에 대해 aws:sourceVpce 조건을 사용합니다.

Question #: 325
Topic #: 1

A database administrator is reviewing the deployment of an application that uses Amazon DynamoDB. A fleet of Amazon EC2 application instances accesses the database.

The database administrator notices that EC2 instances are using public IP addresses to access the database and that the database is available to the internet. Company policy requires that all corporate data must be accessed privately and that external access from the internet is not allowed.

Which combination of steps will ensure that the DynamoDB database meets these requirements? (Choose two.)

  • A. Configure the DynamoDB security group and network ACLs to block external access.
  • B. Create an AWS PrivateLink VPC endpoint for DynamoDUpdate the VPC route table.
  • C. Create a gateway VPC endpoint for DynamoDB. Update the VPC route table.
  • D. Provision a NAT gateway to access DynamoDB. Update the VPC route table.
  • E. Use the aws:sourceVpce condition for all the IAM roles that provision access to the table.

In Option A, configure the DynamoDB security group and network ACL to block external access. This restricts access to the database from the internet, ensuring that all company data is accessed privately. In Option B, create an AWS PrivateLink VPC endpoint for DynamoDB and update the VPC route table. Using AWS PrivateLink allows accessing DynamoDB through a private network, completely blocking access from the internet and keeping database access private.
