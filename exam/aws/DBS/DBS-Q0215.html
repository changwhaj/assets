질문 #: 215
주제 #: 1

회사에서 Microsoft SQL Server를 사용하여 비즈니스 크리티컬 애플리케이션을 온프레미스에서 실행하고 있습니다. 데이터베이스 전문가는 여러 데이터베이스가 있는 인스턴스를 AWS 클라우드로 마이그레이션할 계획입니다. 데이터베이스 전문가는 Amazon EC2 Windows 인스턴스에서 호스팅되는 SQL Server Standard 에디션을 사용합니다. 솔루션은 고가용성을 제공해야 하며 SQL Server 배포 아키텍처에서 단일 실패 지점을 방지해야 합니다.
이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. SQL Server 다중 AZ DB 인스턴스용 Amazon RDS를 생성합니다. Amazon S3를 공유 스토리지 옵션으로 사용하여 데이터베이스를 호스팅합니다.
  • B. Always On 장애 조치 클러스터 인스턴스를 단일 SQL Server 인스턴스로 설정합니다. 다중 AZ Amazon FSx for Windows File Server를 공유 스토리지 옵션으로 사용하여 데이터베이스를 호스팅합니다. 최다 투표
  • C. 여러 SQL Server 인스턴스에서 함께 장애 조치되는 하나 이상의 사용자 데이터베이스를 그룹화하도록 Always On 가용성 그룹을 설정합니다. 다중 AZ Amazon FSx for Windows File Server를 공유 스토리지 옵션으로 사용하여 데이터베이스를 호스팅합니다.
  • D. Application Load Balancer를 생성하여 여러 가용 영역의 여러 EC2 인스턴스에 데이터베이스 트래픽을 분산합니다. Amazon S3를 공유 스토리지 옵션으로 사용하여 데이터베이스를 호스팅합니다.

Exam question from Amazon's AWS Certified Database - Specialty
Question #: 215
Topic #: 1

A company is running a business-critical application on premises by using Microsoft SQL Server. A database specialist is planning to migrate the instance with several databases to the AWS Cloud. The database specialist will use SQL Server Standard edition hosted on Amazon EC2 Windows instances. The solution must provide high availability and must avoid a single point of failure in the SQL Server deployment architecture.
Which solution will meet these requirements?

  • A. Create Amazon RDS for SQL Server Multi-AZ DB instances. Use Amazon S3 as a shared storage option to host the databases.
  • B. Set up Always On Failover Cluster Instances as a single SQL Server instance. Use Multi-AZ Amazon FSx for Windows File Server as a shared storage option to host the databases. Most Voted
  • C. Set up Always On availability groups to group one or more user databases that fail over together across multiple SQL Server instances. Use Multi-AZ Amazon FSx for Windows File Server as a shared storage option to host the databases.
  • D. Create an Application Load Balancer to distribute database traffic across multiple EC2 instances in multiple Availability Zones. Use Amazon S3 as a shared storage option to host the databases.
Show Suggested Answer Hide Answer
Suggested Answer: C 🗳️
