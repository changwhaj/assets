  • A. Inbound data transfer from the internet
  • B. Outbound data transfer to the internet
  • C. Data transfer between AWS Regions
  • D. Data transfer between Availability Zones

선택한 답변: D
내 대답은 D이지만 질문은 오해의 소지가 있는 것 같습니다. 내부 데이터 전송: EC2, RDS, Redshift, DAX 및 ElastiCache와 같은 여러 서비스에 대해 동일한 리전의 동일한 가용 영역(AZ) 내 데이터 전송은 무료입니다. 그러나 이 접근 방식을 사용하려면 특정 영역 내에서 데이터와 처리를 유지해야 하며 이는 모든 사용 사례에 적합하지 않을 수 있습니다. AWS 외부로 데이터 전송: 놀랍게도 AWS에서 인터넷으로의 데이터 전송은 대부분의 서비스에서 무료입니다. 즉, 2TB를 AWS로 수집하면 요금이 부과될 수 있지만 다시 전송하면 요금이 부과되지 않습니다. 그러나 이는 데이터 백업 다운로드 또는 파일 공개 공유와 같은 특정 시나리오에만 적용됩니다.

선택한 답변: C
C. AWS 지역 간 데이터 전송 AWS 글로벌 네트워크를 통해 AWS 지역 간 데이터 전송에는 일반적으로 데이터 전송 비용이 부과되지 않습니다. AWS는 AWS 백본을 통해 리전 간 무료 데이터 전송을 제공합니다. 반면에: A. 인터넷에서 인바운드 데이터 전송: 인터넷에서 AWS로의 인바운드 데이터 전송은 일반적으로 무료이지만 AWS에서 인터넷으로의 아웃바운드 데이터 전송에는 관련 비용이 있습니다. B. 인터넷으로의 아웃바운드 데이터 전송: AWS에서 인터넷으로의 아웃바운드 데이터 전송에는 일반적으로 전송된 데이터 양에 따라 비용이 발생합니다. D. 가용 영역 간 데이터 전송: 동일한 지역 내의 가용 영역 간 데이터 전송은 일반적으로 무료이지만 특정 서비스나 기능을 사용하는 경우와 같은 일부 특정 시나리오에서는 가용 영역 간 데이터 전송과 관련된 비용이 발생할 수 있습니다. 가장 정확한 정보를 얻으려면 AWS 요금 설명서에서 구체적인 세부 정보를 확인하는 것이 중요합니다.

My answer would be D, however the question seems to be misleading Internal Data Transfers: Data transfer within the same Availability Zone (AZ) in the same Region for several services like EC2, RDS, Redshift, DAX, and ElastiCache is free. However, this approach requires keeping your data and processing within a specific zone, which might not be feasible for all use cases. Data Transfer Out of AWS: Surprisingly, data transfer out of AWS to the internet is free for most services. This means ingesting 2 TB into AWS might incur charges, but transferring it back out wouldn't. However, this only applies to specific scenarios, like downloading data backups or sharing files publicly.
