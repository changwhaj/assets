Perform client-side data encryption: This task is the responsibility of the customer to encrypt data on the client side before sending it to AWS. It ensures that data is secure during transit. Configure IAM credentials: Customers are responsible for configuring Identity and Access Management (IAM) credentials. This includes managing user accounts, permissions, and authentication mechanisms for accessing AWS resources. Establish the global infrastructure: This is typically a task managed by AWS. AWS is responsible for setting up and managing the global infrastructure, including data centers and the underlying network. Secure edge locations: AWS is responsible for securing its edge locations. This includes ensuring the physical security and proper operation of these locations. Patch Amazon RDS DB instances: This is typically a task managed by AWS. AWS is responsible for applying updates and security patches to Amazon RDS DB instances to keep them up to date and secure. Customers are responsible for their data and configurations within the RDS instances.
