선택한 답변: C
A. AWS Certificate Manager(ACM): HTTPS를 통한 보안 통신을 활성화하기 위해 Elastic Load Balancer(ELB) 또는 Amazon CloudFront와 같은 AWS 서비스와 함께 사용되는 SSL/TLS 인증서를 관리하는 데 주로 사용됩니다. Amazon EBS 볼륨 암호화와 직접적인 관련이 없습니다. B. AWS 시스템 관리자: AWS 리소스 전반에 걸쳐 운영 작업을 자동화할 수 있습니다. 시스템 구성, 패치 및 자동화 관리 기능을 제공하지만 Amazon EBS 볼륨 암호화를 위해 특별히 설계되지는 않았습니다. C. AWS Key Management Service(AWS KMS): 데이터를 암호화하는 데 사용되는 암호화 키를 쉽게 생성, 제어 및 관리할 수 있게 해주는 완전관리형 서비스입니다. 암호화 목적으로 Amazon EBS를 포함한 다른 AWS 서비스와 원활하게 통합됩니다. D. AWS Config: AWS 리소스의 구성을 평가, 감사 및 평가할 수 있습니다. Amazon EBS 볼륨에 대한 암호화를 직접 제공하지 않습니다.

Selected Answer: C
A. AWS Certificate Manager (ACM): Primarily used for managing SSL/TLS certificates used in conjunction with AWS services like Elastic Load Balancer (ELB) or Amazon CloudFront to enable secure communication over HTTPS. It is not directly related to encrypting Amazon EBS volumes. B. AWS Systems Manager: Allows you to automate operational tasks across your AWS resources. While it provides features for managing system configurations, patching, and automation, it is not specifically designed for encrypting Amazon EBS volumes. C. AWS Key Management Service (AWS KMS): A fully managed service that makes it easy for you to create, control, and manage encryption keys used to encrypt your data. It integrates seamlessly with other AWS services, including Amazon EBS, for encryption purposes. D. AWS Config: Enables you to assess, audit, and evaluate the configurations of your AWS resources. It does not directly provide encryption for Amazon EBS volumes.
