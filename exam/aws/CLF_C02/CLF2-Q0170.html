IAM supports identity federation through SAML 2.0, which allows users to access AWS resources and third-party applications using a single sign-on (SSO) experience ¹. With IAM, you can create a SAML identity provider (IdP) that represents your organization's identity management system, and configure it to establish trust between your organization and AWS ¹. You can then use IAM roles to grant permissions to your users to access AWS resources and third-party applications, and use SAML assertions to authenticate users and provide them with temporary security credentials ¹.
