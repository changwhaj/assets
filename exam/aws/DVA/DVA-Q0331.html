Amazon의 AWS 공인 개발자 어소시에이트 시험 문제
질문 #: 331
주제 #: 1

회사에서 AWS CodePipeline을 사용하여 애플리케이션 중 하나를 제공하고 있습니다. 전달 파이프라인은 AWS CodeCommit 리포지토리의 기본 분기 변경에 의해 트리거되며 AWS CodeBuild를 사용하여 프로세스의 테스트 및 빌드 단계를 구현하고 AWS CodeDeploy를 사용하여 애플리케이션을 배포합니다.

파이프라인은 몇 달 동안 성공적으로 운영되었으며 수정 사항이 없습니다. 최근 애플리케이션 소스 코드가 변경된 후 AWS CodeDeploy가 업데이트된 애플리케이션을 예상대로 배포하지 않았습니다.

가능한 원인은 무엇입니까? (두 가지를 선택하세요.)

  • A. AWS CodeCommit 리포지토리의 메인 브랜치에서 변경이 이루어지지 않았습니다. 최다 투표
  • B. 파이프라인의 이전 단계 중 하나가 실패하여 파이프라인이 종료되었습니다. 최다 투표
  • C. 회사의 AWS CodePipeline 클러스터에 있는 Amazon EC2 인스턴스 중 하나가 비활성 상태입니다.
  • D. AWS CodePipeline이 잘못 구성되어 AWS CodeDeploy를 호출하지 않습니다.
  • E. AWS CodePipeline은 AWS CodeCommit에 액세스할 수 있는 권한이 없습니다.

Exam question from Amazon's AWS Certified Developer Associate
Question #: 331
Topic #: 1

A company is using AWS CodePipeline to deliver one of its applications. The delivery pipeline is triggered by changes to the main branch of an AWS CodeCommit repository and uses AWS CodeBuild to implement the test and build stages of the process and AWS CodeDeploy to deploy the application.

The pipeline has been operating successfully for several months and there have been no modifications. Following a recent change to the application's source code, AWS CodeDeploy has not deployed the updated application as expected.

What are the possible causes? (Choose two.)

  • A. The change was not made in the main branch of the AWS CodeCommit repository. Most Voted
  • B. One of the earlier stages in the pipeline failed and the pipeline has terminated. Most Voted
  • C. One of the Amazon EC2 instances in the company's AWS CodePipeline cluster is inactive.
  • D. The AWS CodePipeline is incorrectly configured and is not invoking AWS CodeDeploy.
  • E. AWS CodePipeline does not have permissions to access AWS CodeCommit.
