The application will retrieve data from an Amazon DynamoDB table for anyone who visits the application without a login. Which AWS feature can the developer use to meet these requirements?

  • C. IAM identity-based policy
  • D. Amazon Cognito authorizer in Amazon API Gateway

IAM identity-based policies can be used to define permissions that grant read access to the DynamoDB table for unauthenticated or anonymous users. These policies can be attached to the IAM roles associated with the application's execution environment. By doing so, interactions with DynamoDB are secured, ensuring that only the necessary actions are allowed. It's important to note that the question does not mention any specific requirements for user authentication or authorization. Therefore, in this case, the most suitable AWS feature to meet the requirements is option A - IAM identity-based policy.

The developer can use IAM policies to control access to the S3 bucket and DynamoDB table, in order to ensure that only authorized users can access the resources. The developer can also use IAM policies to control access to other AWS services that the application may interact with, such as AWS Lambda or Amazon SNS.
