D of course is the right answer. The first assessment give us access to all the objects in the DOC-EXAMPLE-BUCKET. But since there is another policy with an explicity deny, it has the priority and the access won't be granted anymore for the objects in the same bucket that starts with "secrets". Combining the two policies: the user has access to all the objects in the DOC-EXAMPLE-BUCKET bucket only if they don't start with the word secrets*
