주제 #: 1

개발자가 새 애플리케이션의 일부로 Amazon Cognito를 배포하고 있습니다. Amazon Cognito 사용자 풀 도메인의 호스팅 UI는 애플리케이션의 배포된 도메인과 일치해야 합니다. DNS 레코드는 적절하게 업데이트되어야 합니다.

이러한 요구 사항을 충족하려면 개발자는 무엇을 해야 합니까?

  • A. 미리 정의된 도메인을 사용하도록 애플리케이션의 도메인 DNS 설정을 구성합니다.
  • B. 사용자 풀 앱 통합에서 Amazon Cognito 도메인의 접두사인 amazoncognito.com을 구성합니다. 이 새 접두사를 사용하도록 애플리케이션의 도메인 DNS 설정을 구성합니다.
  • C. us-east-1 리전에서 AWS Certificate Manager(ACM) 인증서를 생성합니다. 사용자 지정 도메인으로 사용자 풀의 앱 통합 도메인 이름을 구성합니다. 제공된 별칭 대상을 사용하여 애플리케이션의 도메인 DNS 설정을 구성합니다. 최다 투표
  • D. us-east-1 지역에서 AWS Certificate Manager(ACM) 인증서를 생성합니다. 사용자 지정 도메인을 사용하여 사용자 풀의 도메인 앱 클라이언트 설정을 구성합니다. 제공된 별칭 대상을 사용하여 애플리케이션의 도메인 DNS 설정을 구성합니다.

A developer is deploying Amazon Cognito as part of a new application. The Amazon Cognito user pool domain's hosted UI must match the application's deployed domain. DNS records must be updated appropriately.

What should the developer do to meet these requirements?

  • A. Configure the application's domain DNS setting to use the predefined domain.
  • B. Configure the prefix of the Amazon Cognito domain, amazoncognito.com, in the user pool app integration. Configure the application's domain DNS setting to use this new prefix.
  • C. Create an AWS Certificate Manager (ACM) certificate in the us-east-1 Region. Configure the user pool's app integration domain name with the custom domain. Configure the application's domain DNS setting with the supplied alias target. Most Voted
  • D. Create an AWS Certificate Manager (ACM) certificate in the us-east-1 Region. Configure the user pool's domain app client settings with the custom domain. Configure the application's domain DNS setting with the supplied alias target.
