주제 #: 1

개발자가 온프레미스 웹 애플리케이션을 AWS로 마이그레이션하고 있습니다. 웹 애플리케이션을 통해 사용자는 로그인하고 사용자 프로필을 수정할 수 있습니다. 웹 애플리케이션은 웹 서버와 관계형 데이터베이스로 구성됩니다. 개발자는 웹 서버를 제공하기 위해 Amazon EC2 인스턴스를 시작하는 데 사용할 Amazon Machine Image(AMI)를 생성했습니다. 개발자는 Amazon RDS에도 데이터베이스를 생성했습니다.

마이그레이션의 일환으로 개발자는 Amazon EC2 Auto Scaling 그룹을 생성하여 Application Load Balancer 뒤에서 웹 서버 인스턴스를 시작하려고 합니다. Auto Scaling 그룹의 시작 구성은 웹 서버용 AMI를 지정합니다.

웹 애플리케이션에 자동 확장 기능을 제공하려면 개발자가 추가로 수행해야 하는 단계는 무엇입니까?

  • A. 증가된 수요를 처리할 수 있을 만큼 충분히 큰 EC2 인스턴스 유형을 선택하십시오.
  • B. Amazon ElastiCache 클러스터를 생성합니다. 클러스터에 세션 상태를 저장하도록 웹 애플리케이션을 구성합니다. 최다 투표
  • C. EC2 인스턴스 중 하나에 데이터베이스를 설치합니다. 해당 EC2 인스턴스에서 AMI를 다시 생성합니다.
  • D. RDS 읽기 전용 복제본을 생성합니다. 모든 읽기 작업에 읽기 복제본을 사용하도록 웹 애플리케이션을 구성합니다. 최다 투표

Question #: 431
Topic #: 1

A developer is migrating an on-premises web application to AWS. The web application allows users to log in and modify their user profiles. The web application consists of a web server and a relational database. The developer has created an Amazon Machine Image (AMI) that will be used to launch Amazon EC2 instances to provide the web server. The developer also has created a database in Amazon RDS.

As part of the migration, the developer wants to create an Amazon EC2 Auto Scaling group to launch the web server instances behind an Application Load Balancer. The launch configuration for the Auto Scaling group will specify the AMI for the web server.

Which additional step should the developer take to give the web application the ability to automatically scale?

  • A. Select an EC2 instance type that is large enough to handle increased demand.
  • B. Create an Amazon ElastiCache cluster. Configure the web application to store session state in the cluster. Most Voted
  • C. Install the database on one of the EC2 instances. Recreate the AMI from that EC2 instance.
  • D. Create an RDS read replica. Configure the web application to use the read replica for all read operations. Most Voted
