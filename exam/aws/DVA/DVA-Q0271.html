한 회사가 AWS Lambda 기능과 Amazon API Gateway를 사용하는 기존 애플리케이션을 위한 서버리스 마이크로서비스를 구축하고 있습니다. 마이크로서비스는 OIDC(OpenID Connect) 표준을 지원하는 온프레미스 ID 공급자(IdP)를 자동화해야 합니다.

최소한의 운영 오버헤드로 API 인증 메커니즘을 구현하려면 회사는 무엇을 해야 합니까?

  • A. API Gateway HTTP API를 사용하여 Lambda 함수를 호출합니다. JWT(JSON 웹 토큰) 발급자를 온프레미스 IdP의 공개 OIDC 엔드포인트로 설정합니다.
  • B. API 게이트웨이 HTTP API를 사용하여 Lambda 함수를 호출합니다. Amazon Cognito 자격 증명 풀을 생성합니다. 자격 증명 풀과 온프레미스 IdP 간의 페더레이션을 생성합니다. 자격 증명 풀을 JWT(JSON Web Token) 발급자로 설정합니다.
  • C. API Gateway REST API를 사용하여 Lambda 함수를 호출합니다. 공용 인터넷을 통해 온프레미스 IdP에 연결하도록 JWT(JSON 웹 토큰) 발급자를 구성합니다.
  • D. API Gateway REST API를 사용하여 Lambda 함수를 호출합니다. Amazon Cognito 자격 증명 풀을 생성합니다. 자격 증명 풀과 온프레미스 IdP 간의 페더레이션을 생성합니다. Amazon Cognito를 승인자로 설정합니다. 최다 투표

Topic #: 1

A company is building a serverless microservice for an existing application that uses AWS Lambda functions and Amazon API Gateway. The microservice needs to automate an on-premises identity provider (IdP) that supports the OpenID Connect (OIDC) standard.

What should the company do to implement the API authorization mechanism with the LEAST operational overhead?

  • A. Use an API Gateway HTTP API to invoke the Lambda functions. Set the JSON Web Token (JWT) issuer as the public OIDC endpoint of the on-premises IdP.
  • B. Use an API Gateway HTTP API to invoke the Lambda functions. Create an Amazon Cognito identity pool. Create a federation between the identity pool and the on-premises IdP. Set the identity pool as the JSON Web Token (JWT) issuer.
  • C. Use an API Gateway REST API to invoke the Lambda functions. Configure the JSON Web Token (JWT) issuer to link to the on-premises IdP over the public internet.
  • D. Use an API Gateway REST API to invoke the Lambda functions. Create an Amazon Cognito identity pool. Create a federation between the identity pool and the on-premises IdP. Set Amazon Cognito as the authorizer. Most Voted
