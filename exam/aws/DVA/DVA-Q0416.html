개발자가 Amazon EC2에서 애플리케이션을 구축하고 있습니다. 개발자는 테스트하는 동안 AWS 서비스에 대한 일부 API 호출에서 "액세스 거부" 오류가 발생했습니다. 개발자는 인스턴스에 이미 부여된 권한을 수정해야 합니다.

변경을 최소화하고 가동 중지 시간을 최소화하면서 이러한 요구 사항을 어떻게 충족할 수 있습니까?

  • A. 필요한 권한이 있는 새 IAM 역할을 만듭니다. 인스턴스를 중지합니다. 새 IAM 역할을 인스턴스에 연결합니다. 인스턴스를 시작합니다.
  • B. 기존 IAM 역할을 삭제합니다. 필요한 권한이 있는 새 IAM 역할을 연결합니다.

  • D. 필요한 권한을 추가하여 연결된 IAM 역할을 업데이트합니다. 최다 투표
