주제 #: 1

한 물류 회사는 AWS Serverless Application Model(AWS SAM)을 사용하여 자산 추적 마이크로서비스를 구축했습니다. 마이크로서비스의 AWS Lambda 함수 중 하나는 함수가 요청을 처리하기 전에 Amazon S3에서 대규모 지리 공간 데이터 세트를 가져와야 합니다. 데이터 세트를 가져오려면 상당한 시간이 필요하며 함수 실행을 완료하는 데 너무 오랜 시간이 걸립니다. 그 결과 대기 시간과 비용이 증가합니다.

개발자는 최소한의 비용으로 더 빠르게 요청을 처리할 수 있도록 기능을 최적화해야 합니다.

어떤 솔루션이 이러한 요구 사항을 충족합니까?

  • A. Amazon S3 대신 Amazon Elastic File System(Amazon EFS)에 지리정보 데이터세트를 저장합니다. EFS 파일 시스템을 함수에 연결합니다. 모든 호출에 대해 파일 시스템에 액세스하여 데이터세트를 검색합니다.
  • B. Amazon S3 대신 Amazon Elastic File System(Amazon EFS)에 지리정보 데이터세트를 저장합니다. EFS 파일 시스템을 함수에 연결합니다. 모든 호출에 사용할 데이터를 /tmp 디렉터리에 캐시합니다. 최다 투표
  • C. 함수 핸들러 외부에서 Amazon S3 SDK 및 객체 요청을 선언합니다. 함수에 대해 프로비저닝된 동시성을 구성합니다. 최다 투표
  • D. 함수 핸들러 외부에서 Amazon S3 SDK 및 객체 요청을 선언합니다. 기능의 메모리 용량을 1,769MB로 늘립니다.

Exam question from Amazon's AWS Certified Developer Associate
Question #: 385
Topic #: 1

A logistics company built an asset-tracking microservice by using the AWS Serverless Application Model (AWS SAM). One of the microservice’s AWS Lambda functions needs to import a large geospatial dataset from Amazon S3 before the function can process the requests. The import of the dataset requires significant time and is causing the function to take too long to finish running. The results are increased latency and cost.

A developer needs to optimize the function to process requests faster with the least possible cost.

Which solution will meet these requirements?

  • A. Store the geospatial dataset on Amazon Elastic File System (Amazon EFS) instead of on Amazon S3. Attach the EFS file system to the function. Retrieve the dataset by accessing the file system for every invocation.
  • B. Store the geospatial dataset on Amazon Elastic File System (Amazon EFS) instead of on Amazon S3. Attach the EFS file system to the function. Cache the data in the /tmp directory for use with every invocation. Most Voted
  • C. Declare the Amazon S3 SDK and object request outside of the function handler. Configure provisioned concurrency for the function. Most Voted
  • D. Declare the Amazon S3 SDK and object request outside of the function handler. Increase memory capacity of the function to 1,769 MB.
