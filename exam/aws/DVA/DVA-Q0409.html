질문 #: 409
주제 #: 1

개발자는 PostgreSQL용 Amazon RDS 데이터베이스를 관리합니다. AWS Fargate를 사용하는 Amazon Elastic Container Service(Amazon ECS) 컨테이너는 야간 실행 중에 데이터베이스의 콘텐츠를 수정해야 합니다.

가장 안전한 방식으로 컨테이너에 대한 데이터베이스 액세스를 제공하기 위해 개발자는 어떤 단계 조합을 수행해야 합니까? (2개를 선택하세요.)

  • A. AWS Secrets Manager에서 데이터베이스 비밀을 정의합니다. 작업 정의를 사용하여 컨테이너에 자격 증명을 제공합니다. 최다 투표
  • B. AWS CloudFormation에서 비밀번호를 정의합니다. 컨테이너의 환경 변수에 비밀번호를 전달합니다.
  • C. 데이터베이스에 대한 IAM 인증을 활성화합니다. 컨테이너에 적절한 권한을 부여합니다. 최다 투표
  • D. Amazon Cognito 자격 증명 풀을 사용하여 컨테이너를 인증합니다. 제공된 응답 토큰을 사용하여 데이터베이스에 연결합니다.
  • E. 데이터베이스 콘텐츠를 암호화하는 AWS Key Management Service(AWS KMS) 키를 정의합니다. 컨테이너에 대한 암호 해독 권한을 부여합니다.

Exam question from Amazon's AWS Certified Developer Associate
Question #: 409
Topic #: 1

A developer manages an Amazon RDS for PostgreSQL database. An Amazon Elastic Container Service (Amazon ECS) container that uses AWS Fargate needs to modify the contents of the database during a nightly run.

Which combination of steps should the developer take to provide database access to the container in the MOST secure manner? (Choose two.)

  • A. Define a database secret in AWS Secrets Manager. Provide the credentials to the container by using the task definition. Most Voted
  • B. Define a password in AWS CloudFormation. Pass the password into the environment variables of the container.
  • C. Enable IAM authentication for the database. Grant the appropriate permissions to the container. Most Voted
  • D. Use an Amazon Cognito identity pool to authenticate the container. Use the provided response token to connect to the database.
  • E. Define an AWS Key Management Service (AWS KMS) key that encrypts the contents of the database. Grant permission to decrypt to the container.
