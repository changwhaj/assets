주제 #: 1

개발자는 서명된 요청(서명 버전 4)을 사용하여 다른 AWS 서비스를 호출하는 애플리케이션을 개발하고 있습니다. 개발자는 정식 요청을 생성하고, 서명할 문자열을 생성하고, 서명 정보를 계산했습니다.

개발자는 서명된 요청을 완료하기 위해 어떤 방법을 사용할 수 있습니까? (2개를 선택하세요.)

  • A. Authorization이라는 HTTP 헤더에 서명을 추가합니다. 최다 투표
  • B. 세션 쿠키에 서명 추가
  • C. 인증이라는 HTTP 헤더에 서명을 추가합니다.
  • D. X-Amz-Signature라는 쿼리 문자열 매개변수에 서명을 추가합니다. 최다 투표
  • E. WWW-Authenticate라는 HTTP 헤더에 서명을 추가합니다.

Exam question from Amazon's AWS Certified Developer Associate
Question #: 253
Topic #: 1

A developer is developing an application that uses signed requests (Signature Version 4) to call other AWS services. The developer has created a canonical request, has created the string to sign, and has calculated signing information.

Which methods could the developer use to complete a signed request? (Choose two.)

  • A. Add the signature to an HTTP header that is named Authorization Most Voted
  • B. Add the signature to a session cookie
  • C. Add the signature to an HTTP header that is named Authentication
  • D. Add the signature to a query string parameter that is named X-Amz-Signature Most Voted
  • E. Add the signature to an HTTP header that is named WWW-Authenticate

X-Amz-Signature라는 쿼리 문자열 매개 변수에 서명을 추가합니다. 서명 버전 4를 사용하는 경우 계산된 서명은 일반적으로 HTTP 요청의 Authorization 헤더에 추가됩니다. 이 헤더에는 AWS 서비스가 요청을 확인하는 데 필요한 인증 정보가 포함되어 있습니다. 일반적으로 AWS 서비스 간 통신에 사용됩니다. 또한 경우에 따라 계산된 서명을 X-Amz-Signature라는 이름의 쿼리 문자열 매개변수에 추가할 수 있습니다. 이를 통해 특정 AWS 서비스에 요청할 때 서명을 URL에 직접 포함할 수 있습니다. 따라서 옵션 A와 D는 서명 버전 4를 사용하여 서명된 요청을 완료하는 올바른 방법입니다.
