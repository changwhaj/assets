주제 #: 1

회사에서 Amazon Elastic Block Store(Amazon EBS) 지원 Amazon EC2 인스턴스의 프라이빗 키를 실수로 삭제했습니다. 보안 엔지니어가 인스턴스에 다시 액세스해야 합니다.
이 요구 사항을 충족하는 단계 조합은 무엇입니까? (2개를 선택하세요.)

  • A. 인스턴스를 중지합니다. 루트 볼륨을 분리합니다. 새 키 쌍을 생성합니다. 최다 투표
  • B. 인스턴스를 계속 실행하십시오. 루트 볼륨을 분리합니다. 새 키 쌍을 생성합니다.
  • C. 원본 인스턴스에서 볼륨이 분리되면 해당 볼륨을 다른 인스턴스에 데이터 볼륨으로 연결합니다. 새로운 공개 키를 사용하여 Authorized_keys 파일을 수정합니다. 볼륨을 원래 인스턴스로 다시 이동합니다. 인스턴스를 시작합니다. 최다 투표
  • D. 원본 인스턴스에서 볼륨이 분리되면 해당 볼륨을 다른 인스턴스에 데이터 볼륨으로 연결합니다. 새 개인 키를 사용하여 Authorized_keys 파일을 수정합니다. 볼륨을 원래 인스턴스로 다시 이동합니다. 인스턴스를 시작합니다.
  • E. 원본 인스턴스에서 볼륨이 분리되면 해당 볼륨을 다른 인스턴스에 데이터 볼륨으로 연결합니다. 새로운 공개 키를 사용하여 Authorized_keys 파일을 수정합니다. 실행 중인 원래 인스턴스로 볼륨을 다시 이동합니다.

Question #: 61
Topic #: 1

A company accidentally deleted the private key for an Amazon Elastic Block Store (Amazon EBS)-backed Amazon EC2 instance. A security engineer needs to regain access to the instance.
Which combination of steps will meet this requirement? (Choose two.)

  • A. Stop the instance. Detach the root volume. Generate a new key pair. Most Voted
  • B. Keep the instance running. Detach the root volume. Generate a new key pair.
  • C. When the volume is detached from the original instance, attach the volume to another instance as a data volume. Modify the authorized_keys file with a new public key. Move the volume back to the original instance. Start the instance. Most Voted
  • D. When the volume is detached from the original instance, attach the volume to another instance as a data volume. Modify the authorized_keys file with a new private key. Move the volume back to the original instance. Start the instance.
  • E. When the volume is detached from the original instance, attach the volume to another instance as a data volume. Modify the authorized_keys file with a new public key. Move the volume back to the original instance that is running.
