보안 엔지니어가 제품 팀과 협력하여 AWS에서 웹 애플리케이션을 구축하고 있습니다. 애플리케이션은 Amazon S3를 사용하여 정적 콘텐츠를 호스팅하고, Amazon API Gateway를 사용하여 RESTful 서비스를 제공하며, Amazon DynamoDB를 백엔드 데이터 저장소로 사용합니다. 사용자는 SAML ID 공급자를 통해 노출되는 디렉터리에 이미 존재합니다.

사용자가 웹 애플리케이션에 인증되고 API를 호출할 수 있도록 엔지니어는 다음 작업 중 어떤 조합을 수행해야 합니까? (3개를 선택하세요.)

  • A. AWS Lambda를 사용하여 사용자 지정 인증 서비스를 생성합니다.
  • B. Amazon Cognito 사용자 풀 속성에 속성을 매핑하도록 Amazon Cognito에서 SAML 자격 증명 공급자를 구성합니다. 최다 투표
  • C. Amazon Cognito 사용자 풀을 신뢰 당사자로 추가하도록 SAML 자격 증명 공급자를 구성합니다. 최다 투표
  • D. 소셜 로그인 공급자와 통합하도록 Amazon Cognito 자격 증명 풀을 구성합니다.
  • E. DynamoDB를 업데이트하여 사용자 이메일 주소와 비밀번호를 저장합니다.
  • F. COGNITO_USER_POOLS 권한 부여자를 사용하도록 API 게이트웨이를 업데이트합니다. 최다 투표

Exam question from Amazon's AWS Certified Security - Specialty SCS-C02
Question #: 87
Topic #: 1

A security engineer is working with a product team building a web application on AWS. The application uses Amazon S3 to host the static content, Amazon API Gateway to provide RESTful services, and Amazon DynamoDB as the backend data store. The users already exist in a directory that is exposed through a SAML identity provider.

Which combination of the following actions should the engineer take to allow users to be authenticated into the web application and call APIs? (Choose three.)

  • A. Create a custom authorization service using AWS Lambda.
  • B. Configure a SAML identity provider in Amazon Cognito to map attributes to the Amazon Cognito user pool attributes. Most Voted
  • C. Configure the SAML identity provider to add the Amazon Cognito user pool as a relying party. Most Voted
  • D. Configure an Amazon Cognito identity pool to integrate with social login providers.
  • E. Update DynamoDB to store the user email addresses and passwords.
  • F. Update API Gateway to use a COGNITO_USER_POOLS authorizer. Most Voted

E - Dynamo DB is irrelevant- never store store password in db without all the additional overheads required to keep them secure.
