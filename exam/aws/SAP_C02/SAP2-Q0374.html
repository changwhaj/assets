주제 #: 1

라이브 이벤트 회사가 AWS에서 티켓 애플리케이션을 위한 스케일링 솔루션을 설계하고 있습니다. 이 애플리케이션은 세일 이벤트 중에 사용률이 높습니다. 각 세일 이벤트는 예약된 일회성 이벤트입니다. 이 애플리케이션은 Auto Scaling 그룹에 있는 Amazon EC2 인스턴스에서 실행됩니다. 이 애플리케이션은 데이터베이스 계층에 PostgreSQL을 사용합니다.

이 회사는 세일 이벤트 중에 가용성을 극대화하기 위한 스케일링 솔루션이 필요합니다.

어떤 솔루션이 이러한 요구 사항을 충족할까요?

  • A. EC2 인스턴스에 예측적 스케일링 정책을 사용합니다. 읽기 복제본을 자동으로 스케일링하는 Amazon Aurora PostgreSQL Serverless v2 Multi-AZ DB 인스턴스에서 데이터베이스를 호스팅합니다. AWS Step Functions 상태 머신을 생성하여 병렬 AWS Lambda 함수를 실행하여 판매 이벤트 전에 데이터베이스를 사전 워밍합니다. Amazon EventBridge 규칙을 생성하여 상태 머신을 호출합니다.
  • B. EC2 인스턴스에 예약된 스케일링 정책을 사용합니다. 읽기 복제본을 자동으로 스케일링하는 Amazon RDS for PostgreSQL Mulli-AZ DB 인스턴스에서 데이터베이스를 호스팅합니다. 판매 이벤트 전에 더 큰 읽기 복제본을 생성하기 위해 AWS Lambda 함수를 호출하는 Amazon EventBridge 규칙을 만듭니다. 더 큰 읽기 복제본으로 장애 조치합니다. 판매 이벤트 후에 읽기 복제본을 축소하기 위해 다른 Lambda 함수를 호출하는 또 다른 EventBridge 규칙을 만듭니다.
  • C. EC2 인스턴스에 대한 예측적 확장 정책을 사용합니다. 읽기 복제본을 자동으로 확장하는 Amazon RDS for PostgreSQL MultiAZ DB 인스턴스에서 데이터베이스를 호스팅합니다. AWS Step Functions 상태 머신을 생성하여 병렬 AWS Lambda 함수를 실행하여 판매 이벤트 전에 데이터베이스를 사전 워밍합니다. Amazon EventBridge 규칙을 생성하여 상태 머신을 호출합니다.
  • D. EC2 인스턴스에 예약된 스케일링 정책을 사용합니다. Amazon Aurora PostgreSQL Multi-AZ DB 클러스터에서 데이터베이스를 호스팅합니다. 판매 이벤트 전에 더 큰 Aurora Replica를 생성하기 위해 AWS Lambda 함수를 호출하는 Amazon EventBridge 규칙을 만듭니다. 더 큰 Aurora Replica로 장애 조치합니다. 판매 이벤트 후에 Aurora Replica를 축소하기 위해 다른 Lambda 함수를 호출하는 또 다른 EventBridge 규칙을 만듭니다. 가장 많이 투표된

Exam question from Amazon's AWS Certified Solutions Architect - Professional SAP-C02
Question #: 374
Topic #: 1

A live-events company is designing a scaling solution for its ticket application on AWS. The application has high peaks of utilization during sale events. Each sale event is a one-time event that is scheduled. The application runs on Amazon EC2 instances that are in an Auto Scaling group. The application uses PostgreSQL for the database layer.

The company needs a scaling solution to maximize availability during the sale events.

Which solution will meet these requirements?

  • A. Use a predictive scaling policy for the EC2 instances. Host the database on an Amazon Aurora PostgreSQL Serverless v2 Multi-AZ DB instance with automatically scaling read replicas. Create an AWS Step Functions state machine to run parallel AWS Lambda functions to pre-warm the database before a sale event. Create an Amazon EventBridge rule to invoke the state machine.
  • B. Use a scheduled scaling policy for the EC2 instances. Host the database on an Amazon RDS for PostgreSQL Mulli-AZ DB instance with automatically scaling read replicas. Create an Amazon EventBridge rule that invokes an AWS Lambda function to create a larger read replica before a sale event. Fail over to the larger read replica. Create another EventBridge rule that invokes another Lambda function to scale down the read replica after the sale event.
  • C. Use a predictive scaling policy for the EC2 instances. Host the database on an Amazon RDS for PostgreSQL MultiAZ DB instance with automatically scaling read replicas. Create an AWS Step Functions state machine to run parallel AWS Lambda functions to pre-warm the database before a sale event. Create an Amazon EventBridge rule to invoke the state machine.
  • D. Use a scheduled scaling policy for the EC2 instances. Host the database on an Amazon Aurora PostgreSQL Multi-AZ DB cluster. Create an Amazon EventBridge rule that invokes an AWS Lambda function to create a larger Aurora Replica before a sale event. Fail over to the larger Aurora Replica. Create another EventBridge rule that invokes another Lambda function to scale down the Aurora Replica after the sale event. Most Voted
