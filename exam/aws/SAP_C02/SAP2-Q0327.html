주제 #: 1

어떤 회사가 온프레미스 애플리케이션을 AWS로 마이그레이션하려고 합니다. 애플리케이션의 데이터베이스는 구조화된 제품 데이터와 임시 사용자 세션 데이터를 저장합니다. 이 회사는 제품 데이터를 사용자 세션 데이터에서 분리해야 합니다. 또한 이 회사는 재해 복구를 위해 다른 AWS 리전에 복제를 구현해야 합니다.

어떤 솔루션이 이러한 요구 사항을 가장 높은 성능으로 충족할까요?

  • A. 제품 데이터와 사용자 세션 데이터를 호스팅하기 위해 별도의 스키마로 Amazon RDS DB 인스턴스를 만듭니다. 다른 지역의 DB 인스턴스에 대한 읽기 복제본을 구성합니다.
  • B. 제품 데이터를 호스팅하기 위해 Amazon RDS DB 인스턴스를 만듭니다. 다른 지역의 DB 인스턴스에 대한 읽기 복제본을 구성합니다. 사용자 세션 데이터를 호스팅하기 위해 Memcached용 Amazon ElastiCache에 글로벌 데이터스토어를 만듭니다.
  • C. 두 개의 Amazon DynamoDB 글로벌 테이블을 만듭니다. 하나의 글로벌 테이블을 사용하여 제품 데이터를 호스팅합니다. 다른 글로벌 테이블을 사용하여 사용자 세션 데이터를 호스팅합니다. 캐싱에는 DynamoDB Accelerator(DAX)를 사용합니다. 가장 많이 투표된
  • D. 제품 데이터를 호스팅할 Amazon RDS DB 인스턴스를 만듭니다. 다른 지역의 DB 인스턴스에 대한 읽기 복제본을 구성합니다. 사용자 세션 데이터를 호스팅할 Amazon DynamoDB 글로벌 테이블을 만듭니다. 가장 많이 투표된

Question #: 327
Topic #: 1

A company wants to migrate its on-premises application to AWS. The database for the application stores structured product data and temporary user session data. The company needs to decouple the product data from the user session data. The company also needs to implement replication in another AWS Region for disaster recovery.

Which solution will meet these requirements with the HIGHEST performance?

  • A. Create an Amazon RDS DB instance with separate schemas to host the product data and the user session data. Configure a read replica for the DB instance in another Region.
  • B. Create an Amazon RDS DB instance to host the product data. Configure a read replica for the DB instance in another Region. Create a global datastore in Amazon ElastiCache for Memcached to host the user session data.
  • C. Create two Amazon DynamoDB global tables. Use one global table to host the product data. Use the other global table to host the user session data. Use DynamoDB Accelerator (DAX) for caching. Most Voted
  • D. Create an Amazon RDS DB instance to host the product data. Configure a read replica for the DB instance in another Region. Create an Amazon DynamoDB global table to host the user session data. Most Voted

For those who voted D as the answer, your argument is that ElastiCache doesn’t support Multi-AZ and cross region deployment. But the question doesn’t ask for Multi-AZ and cross region session data replication.
