질문 #: 446
주제 #: 1

한 회사가 AWS Organizations를 사용하여 개발 환경을 관리합니다. 회사의 각 개발팀은 자체 AWS 계정을 가지고 있습니다. 각 계정에는 겹치지 않는 단일 VPC 및 CIDR 블록이 있습니다.

이 회사는 공유 서비스 계정에 Amazon Aurora DB 클러스터를 가지고 있습니다. 모든 개발팀은 DB 클러스터의 라이브 데이터로 작업해야 합니다.

어떤 솔루션이 최소한의 운영 오버헤드로 DB 클러스터에 필요한 연결을 제공할까요?

  • A. DB 클러스터에 대한 AWS Resource Access Manager(AWS RAM) 리소스 공유를 만듭니다. 모든 개발 계정과 DB 클러스터를 공유합니다. 가장 많이 투표된
  • B. 공유 서비스 계정에서 전송 게이트웨이를 만듭니다. 전송 게이트웨이에 대한 AWS Resource Access Manager(AWS RAM) 리소스 공유를 만듭니다. 전송 게이트웨이를 모든 개발 계정과 공유합니다. 개발자에게 리소스 공유를 수락하도록 지시합니다. 네트워킹을 구성합니다. 가장 많이 투표된
  • C. DB 클러스터의 IP 주소를 가리키는 Application Load Balancer(ALB)를 만듭니다. ALB를 사용하는 AWS PrivateLink 엔드포인트 서비스를 만듭니다. 각 개발 계정이 엔드포인트 서비스에 연결할 수 있도록 권한을 추가합니다.
  • D. 공유 서비스 계정에서 AWS Site-to-Site VPN 연결을 만듭니다. 네트워킹을 구성합니다. 각 개발 계정에서 AWS Marketplace VPN 소프트웨어를 사용하여 Site-to-Site VPN 연결에 연결합니다.

Exam question from Amazon's AWS Certified Solutions Architect - Professional SAP-C02
Question #: 446
Topic #: 1

A company uses AWS Organizations to manage its development environment. Each development team at the company has its own AWS account. Each account has a single VPC and CIDR blocks that do not overlap.

The company has an Amazon Aurora DB cluster in a shared services account. All the development teams need to work with live data from the DB cluster.

Which solution will provide the required connectivity to the DB cluster with the LEAST operational overhead?

  • A. Create an AWS Resource Access Manager (AWS RAM) resource share for the DB cluster. Share the DB cluster with all the development accounts. Most Voted
  • B. Create a transit gateway in the shared services account. Create an AWS Resource Access Manager (AWS RAM) resource share for the transit gateway. Share the transit gateway with all the development accounts. Instruct the developers to accept the resource share. Configure networking. Most Voted
  • C. Create an Application Load Balancer (ALB) that points to the IP address of the DB cluster. Create an AWS PrivateLink endpoint service that uses the ALB. Add permissions to allow each development account to connect to the endpoint service.
  • D. Create an AWS Site-to-Site VPN connection in the shared services account. Configure networking. Use AWS Marketplace VPN software in each development account to connect to the Site-to-Site VPN connection.
