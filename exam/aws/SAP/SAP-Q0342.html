주제 #: 1

사용자가 VPC에 퍼블릭 서브넷을 생성하고 그 안에서 EC2 인스턴스를 시작하려고 합니다. 사용자가 인스턴스를 시작하는 동안 퍼블릭 IP 주소를 할당하는 옵션을 선택하지 않았습니다.
이 시나리오와 관련하여 아래에 언급된 설명 중 올바른 것은 무엇입니까?

  • A. 인스턴스에는 기본적으로 항상 인스턴스에 연결된 퍼블릭 DNS가 있습니다.
  • B. 사용자는 서브넷의 라우팅 테이블에서 IGW에 대한 기본 경로를 생성한 다음 탄력적 IP를 인스턴스에 연결하여 인터넷에서 연결해야 합니다.
  • C. 사용자가 탄력적 IP를 인스턴스에 직접 연결할 수 있습니다.
  • D. 퍼블릭 IP가 할당되지 않으면 인스턴스가 시작되지 않습니다.
제안된 답변 표시 답변 숨기기
제안 답변: B 🗳️
Virtual Private Cloud(VPC)는 사용자의 AWS 계정 전용 가상 네트워크입니다. 사용자는 VPC가 있는 서브넷을 생성하고 해당 서브넷 내에서 인스턴스를 시작할 수 있습니다. 사용자가 인스턴스를 시작할 때 인스턴스에 퍼블릭 IP를 연결하는 옵션을 선택해야 합니다. 사용자가 공용 IP를 연결하는 옵션을 선택하지 않은 경우 시작 시 개인 IP만 갖게 됩니다. 사용자는 인터넷에서 인스턴스에 연결할 수 없습니다. 사용자가 인터넷에서 인스턴스에 연결하기 위해 탄력적 IP를 원하는 경우 인터넷 게이트웨이를 생성하고 인스턴스에 탄력적 IP를 할당해야 합니다.
참조:
http://docs.aws.amazon.com/AmazonVPC/latest/GettingStartedGuide/LaunchInstance.html

Question #: 342
Topic #: 1

A user wants to create a public subnet in VPC and launch an EC2 instance within it. The user has not selected the option to assign a public IP address while launching the instance.
Which of the below mentioned statements is true with respect to this scenario?

  • A. The instance will always have a public DNS attached to the instance by default
  • B. The user would need to create a default route to IGW in subnet's route table and then attach an elastic IP to the instance to connect from the internet
  • C. The user can directly attach an elastic IP to the instance
  • D. The instance will never launch if the public IP is not assigned
Show Suggested Answer Hide Answer
Suggested Answer: B 🗳️
A Virtual Private Cloud (VPC) is a virtual network dedicated to the user's AWS account. A user can create a subnet with VPC and launch instances inside that subnet. When the user is launching an instance he needs to select an option which attaches a public IP to the instance. If the user has not selected the option to attach the public IP, then it will only have a private IP when launched. The user cannot connect to the instance from the internet. If the user wants an elastic IP to connect to the instance from the internet, he should create an internet gateway and assign an elastic IP to instance.
Reference:
http://docs.aws.amazon.com/AmazonVPC/latest/GettingStartedGuide/LaunchInstance.html

The answer is B. Explanation: When you launch an EC2 instance in a public subnet, by default, the instance will not have a public IP address assigned to it. In order for the instance to be able to connect to the internet, you will need to create a default route to the Internet Gateway (IGW) in the subnet's route table. Once you have created the default route, you can then attach an Elastic IP (EIP) to the instance. The EIP will provide the instance with a public IP address that can be used to connect to the internet. Statement A is incorrect because the instance will not have a public DNS address assigned to it by default. Statement C is incorrect because you cannot directly attach an EIP to an instance that does not have a public IP address assigned to it. Statement D is incorrect because the instance will launch even if you do not assign a public IP address to it. However, the instance will not be able to connect to the internet until you create a default route to the IGW and attach an EIP to the instance.
