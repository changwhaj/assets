올바른 대안은 C와 D입니다. 옵션 C에는 모든 웹 서버를 Elastic Load Balancer(ELB) 뒤에 배치하는 작업이 포함됩니다. ELB는 들어오는 트래픽을 여러 웹 서버에 자동으로 분산하여 하나 이상의 웹 서버에 장애가 발생하더라도 애플리케이션을 계속 사용할 수 있도록 보장하므로 고가용성을 달성하기 위한 좋은 선택입니다. 그런 다음 ELB DNS 이름을 가리키도록 Route 53 CNAME 레코드를 구성할 수 있습니다. 옵션 D에는 모든 웹 서버에 탄력적 IP 주소(EIP)를 할당하는 것이 포함됩니다. 이를 통해 모든 EIP로 Route 53 레코드 세트를 구성하고 상태 확인 및 DNS 장애 조치를 설정할 수 있습니다. 이렇게 하면 하나 이상의 웹 서버에 장애가 발생하더라도 트래픽이 자동으로 나머지 정상 웹 서버로 라우팅되므로 애플리케이션을 계속 사용할 수 있습니다.

Selected Answer: CD
The correct alternatives are C and D. Option C involves placing all your web servers behind an Elastic Load Balancer (ELB). This is a good choice for achieving high availability because the ELB will automatically distribute incoming traffic across multiple web servers, ensuring that the application remains available even if one or more of the web servers fail. You can then configure a Route 53 CNAME record to point to the ELB DNS name. Option D involves assigning Elastic IP addresses (EIPs) to all your web servers. This allows you to configure a Route 53 record set with all the EIPs and set up health checks and DNS failover.
