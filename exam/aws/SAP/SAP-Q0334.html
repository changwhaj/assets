주제 #: 1

사용자가 마법사를 사용하여 CIDR 20.0.0.0/16이 있는 VPC를 생성했습니다.
사용자는 사용자의 데이터 센터에 연결하기 위해 VPN 게이트웨이(vgw-123456)와 함께 공용 서브넷 CIDR(20.0.0.0/24) 및 VPN 전용 서브넷 CIDR(20.0.1.0/24)을 생성했습니다 . 사용자의 데이터 센터에는 CIDR 172.28.0.0/12가 있습니다. 또한 사용자는 VPN 서브넷에서 인터넷으로의 트래픽을 허용하도록 NAT 인스턴스(i-123456)를 설정했습니다.
아래에 언급된 옵션 중 이 시나리오의 기본 경로 테이블에 대한 유효한 항목이 아닌 것은 무엇입니까?

  • A. 대상: 20.0.0.0/16 및 대상: 로컬
  • B. 대상: 0.0.0.0/0 및 대상: i-123456
  • C. 대상: 172.28.0.0/12 및 대상: vgw-123456
  • D. 대상: 20.0.1.0/24 및 대상: i-123456
제안된 답변 표시 답변 숨기기
제안 답변: D 🗳️
사용자는 VPC 내의 요구 사항에 따라 서브넷을 생성할 수 있습니다. 사용자가 자신의 데이터 센터에서 VPC를 연결하려는 경우 하드웨어 VPN 액세스를 사용하여 데이터 센터에 연결하는 공용 및 VPN 전용 서브넷을 설정할 수 있습니다. 사용자가 마법사를 사용하여 이 설정을 구성하면 VPN 서브넷의 모든 트래픽을 라우팅하는 가상 프라이빗 게이트웨이가 생성됩니다. 사용자가 모든 인터넷 요청을 라우팅하도록 NAT 인스턴스를 설정한 경우 인터넷에 대한 모든 요청은 NAT 인스턴스로 라우팅되어야 합니다. 조직의 DC에 대한 모든 요청은 VPN 게이트웨이로 라우팅됩니다. 이 시나리오에서 기본 라우팅 테이블에 대한 유효한 항목은 다음과 같습니다.
대상: 0.0.0.0/0 & 대상: i-123456(모든 인터넷 트래픽을 NAT 인스턴스로 라우팅하기 위해) 대상: 172.28.0.0/12 & 대상: vgw-123456(조직의 모든 데이터 센터 트래픽을 VPN 게이트웨이로 라우팅하기 위해) )
대상: 20.0.0.0/16 & 대상: 로컬(VPC에서 로컬 라우팅을 허용하기 위해)

Question #: 334
Topic #: 1

A user has created a VPC with CIDR 20.0.0.0/16 using the wizard. The user has created a public subnet CIDR (20.0.0.0/24) and VPN only subnets CIDR
(20.0.1.0/24) along with the VPN gateway (vgw-123456) to connect to the user's data center. The user's data center has CIDR 172.28.0.0/12. The user has also setup a NAT instance (i-123456) to allow traffic to the internet from the VPN subnet.
Which of the below mentioned options is not a valid entry for the main route table in this scenario?

  • A. Destination: 20.0.0.0/16 and Target: local
  • B. Destination: 0.0.0.0/0 and Target: i-123456
  • C. Destination: 172.28.0.0/12 and Target: vgw-123456
  • D. Destination: 20.0.1.0/24 and Target: i-123456
Show Suggested Answer Hide Answer
Suggested Answer: D 🗳️
The user can create subnets as per the requirement within a VPC. If the user wants to connect VPC from his own data centre, he can setup a public and VPN only subnet which uses hardware VPN access to connect with his data centre. When the user has configured this setup with Wizard, it will create a virtual private gateway to route all traffic of the VPN subnet. If the user has setup a NAT instance to route all the internet requests, then all requests to the internet should be routed to it. All requests to the organization's DC will be routed to the VPN gateway. Here are the valid entries for the main route table in this scenario:
Destination: 0.0.0.0/0 & Target: i-123456 (To route all internet traffic to the NAT Instance) Destination: 172.28.0.0/12 & Target: vgw-123456 (To route all the organization's data centre traffic to the VPN gateway)
Destination: 20.0.0.0/16 & Target: local (To allow local routing in VPC)

Selected Answer: D
The answer is D. Destination: 20.0.1.0/24 and Target: i-123456. The main route table is used to route traffic within the VPC. The destination CIDR in the route table entry specifies the range of IP addresses that the route applies to. The target specifies the gateway or instance that the traffic should be routed to. In this scenario, the destination CIDR for option D, 20.0.1.0/24, is the CIDR for the VPN-only subnet. The target for option D, i-123456, is the NAT instance. However, the NAT instance is not used to route traffic to the VPN-only subnet. Instead, traffic to the VPN-only subnet is routed to the VPN gateway.
