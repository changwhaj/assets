이 회사는 비즈니스 인텔리전스 분석가가 주간 비즈니스 보고서를 생성할 수 있도록 데이터 쿼리 플랫폼을 구축하고 있습니다 . 새 시스템은 임시 SQL 쿼리를 실행해야 합니다.
가장 비용 효율적인 솔루션은 무엇입니까?

  • A. 새로운 Amazon Redshift 클러스터를 생성하십시오. RDS 데이터베이스에서 Amazon Redshift 클러스터로 데이터를 복사하는 AWS Glue ETL 작업을 생성합니다. Amazon Redshift를 사용하여 쿼리를 실행합니다.
  • B. 코어 노드가 충분한 Amazon EMR 클러스터를 생성합니다. Apache Spark 작업을 실행하여 RDS 데이터베이스의 데이터를 HDFS(Hadoop 분산 파일 시스템)로 복사합니다. 로컬 Apache Hive 메타스토어를 사용하여 테이블 정의를 유지합니다. Spark SQL을 사용하여 쿼리를 실행합니다.
  • C. AWS Glue ETL 작업을 사용하여 모든 RDS 데이터베이스를 단일 Amazon Aurora PostgreSQL 데이터베이스에 복사합니다. Aurora PostgreSQL 데이터베이스에서 SQL 쿼리를 실행합니다.
  • D. AWS Glue 크롤러를 사용하여 모든 데이터베이스를 크롤링하고 AWS Glue 데이터 카탈로그에 테이블을 생성합니다. AWS Glue ETL 작업을 사용하여 RDS 데이터베이스의 데이터를 Amazon S3에 로드하고 Amazon Athena를 사용하여 쿼리를 실행합니다. 최다 투표

Topic #: 1

A company has several teams, and each team has their own Amazon RDS database that totals 100 TB. The company is building a data query platform for
Business Intelligence Analysts to generate a weekly business report. The new system must run ad-hoc SQL queries.
What is the MOST cost-effective solution?

  • A. Create a new Amazon Redshift cluster. Create an AWS Glue ETL job to copy data from the RDS databases to the Amazon Redshift cluster. Use Amazon Redshift to run the query.
  • B. Create an Amazon EMR cluster with enough core nodes. Run an Apache Spark job to copy data from the RDS databases to a Hadoop Distributed File System (HDFS). Use a local Apache Hive metastore to maintain the table definition. Use Spark SQL to run the query.
  • C. Use an AWS Glue ETL job to copy all the RDS databases to a single Amazon Aurora PostgreSQL database. Run SQL queries on the Aurora PostgreSQL database.
  • D. Use an AWS Glue crawler to crawl all the databases and create tables in the AWS Glue Data Catalog. Use an AWS Glue ETL job to load data from the RDS databases to Amazon S3, and use Amazon Athena to run the queries. Most Voted
