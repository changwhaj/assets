주제 #: 1

한 회사가 AWS 클라우드에서 이미지 처리 서비스를 호스팅합니다. 회사는 사용자 지정 온프레미스 애플리케이션을 사용하여 AWS Direct Connect 연결을 통해 온프레미스 시스템에서 서비스를 호출합니다. 애플리케이션은 Amazon EC2 인스턴스에 호스팅된 소프트웨어에 이미지 처리 요청을 보냅니다. 이미지 데이터는 요청 내에 포함됩니다. EC2 인스턴스는 Auto Scaling 그룹에 속하며 NLB(Network Load Balancer) 뒤에서 실행됩니다. Auto Scaling 그룹의 최소 용량은 인스턴스 2개입니다.

회사에서는 매일 동시에 수만 건의 이미지 처리 요청을 받고 있어 오류가 발생하고 있습니다. 요청 처리에는 시간이 중요하지 않습니다. 하루 종일 애플리케이션은 요청을 보내지 않습니다.

회사는 애플리케이션의 안정성을 향상하고 이미지 처리 요청 실패 횟수를 최소화하려고 합니다. 회사에서도 비용을 절감하고 싶어합니다.

최소한의 운영 오버헤드로 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. 목표 값이 90%인 메모리 사용률 지표를 기반으로 목표 추적 조정 정책을 추가합니다. 조정 이벤트가 발생할 때 추가 스팟 인스턴스를 사용하도록 Auto Scaling 그룹을 구성합니다. 최다 투표
  • B. 이미지를 Amazon S3에 업로드하도록 애플리케이션을 변경합니다. 객체 업로드에 대한 Amazon Simple Queue Service(Amazon SQS) 알림을 구성합니다. 메시지가 비동기적으로 처리되도록 객체 참조로 SQS 메시지를 수신하기 위해 EC2 인스턴스에서 실행되는 이미지 처리 코드를 다시 작성합니다. NLB를 제거합니다. 최다 투표
  • C. 이미지 처리 로직을 포함하는 AWS Lambda 함수를 생성합니다. EC2 인스턴스를 호스팅하는 VPC에 Lambda 함수를 배포합니다. NLB에서 대상 그룹을 업데이트하여 Lambda 함수를 호출합니다.
  • D. 이미지 처리 로직을 포함하는 AWS Lambda 함수를 생성합니다. Amazon S3에 이미지를 업로드하도록 애플리케이션을 변경합니다. Lambda 함수를 호출하는 재시도 정책을 사용하여 객체 업로드에 대한 Amazon Simple 알림 서비스(Amazon SNS) 알림을 구성합니다. NLB를 제거합니다.

Exam question from Amazon's AWS Certified Solutions Architect - Professional
Question #: 981
Topic #: 1

A company hosts an image-processing service in the AWS Cloud. The company uses a custom on-premises application to call the service from its on-premises systems over an AWS Direct Connect connection. The application sends image-processing requests to software that is hosted on Amazon EC2 instances. The image data is embedded within the requests. The EC2 instances are in an Auto Scaling group and run behind a Network Load Balancer (NLB). The Auto Scaling group has a minimum capacity of two instances.

The company is receiving tens of thousands of image-processing requests at the same time each day and is experiencing errors. The processing of the requests is not time critical. For much of the day, the application sends no requests.

The company wants to improve the application's reliability and minimize the number of failed image-processing requests. The company also wants to reduce cost.

Which solution will meet these requirements with the LEAST operational overhead?

  • A. Add a target tracking scaling policy based on a memory utilization metric that has a target value of 90%. Configure the Auto Scaling group to use additional Spot Instances when scaling events occur. Most Voted
  • B. Change the application to upload the images to Amazon S3. Configure Amazon Simple Queue Service (Amazon SQS) notifications for object uploads. Rewrite the image-processing code that runs on the EC2 instances to receive SQS messages with the object reference so that the messages are processed asynchronously. Remove the NLB. Most Voted
  • C. Create an AWS Lambda function that includes the image-processing logic. Deploy the Lambda function in the VPC that hosts the EC2 instances. Update the target group on the NLB to invoke the Lambda function.
  • D. Create an AWS Lambda function that includes the image-processing logic. Change the application to upload the images to Amazon S3. Configure Amazon Simple Notification Service (Amazon SNS) notifications for object uploads with a retry policy to invoke the Lambda function. Remove the NLB.
