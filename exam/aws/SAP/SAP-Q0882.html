주제 #: 1

일기 예보 회사는 PostgreSQL 데이터베이스에 온프레미스 데이터를 저장하는 애플리케이션을 마이그레이션하고 있습니다. 회사는 데이터베이스를 Amazon Aurora PostgreSQL로 마이그레이션하려고 합니다. 데이터베이스 크기는 매일 평균 5GB씩 증가하며 현재 50TB입니다. 데이터 센터에는
50Mbps의 사용 가능한 대역폭을 갖춘 인터넷 연결이 있습니다. AWS로의 마이그레이션은 앞으로 21일 이내에 최대한 빨리 완료되어야 합니다.
애플리케이션 가동 중지 시간을 최소화하면서 이러한 요구 사항을 충족하는 데이터 전송 전략은 무엇입니까?

  • A. 애플리케이션을 오프라인으로 전환하세요. 데이터베이스의 로컬 백업을 만듭니다. Amazon S3 버킷에 대한 기존 연결을 통해 데이터베이스 백업 파일을 전송합니다. 기본 데이터베이스 도구를 사용하여 백업을 새 데이터베이스로 복원하고 백업 이후 변경 사항을 캡처하도록 복제를 설정합니다. 데이터베이스 연결 문자열을 수정하고 애플리케이션을 온라인으로 전환합니다.
  • B. 로컬 데이터 센터에 서버 마이그레이션 커넥터 VM을 설치합니다. AWS Server Migration Service(AWS SMS) 콘솔을 사용하여 온프레미스 데이터베이스를 새 데이터베이스에 복제합니다. 새 데이터베이스를 가리키도록 DNS 레코드를 수정합니다.
  • C. 데이터베이스의 로컬 백업을 생성하고 해당 백업을 AWS Snowcone 장치에 복사합니다. 디바이스에서 AWS DataSync 에이전트를 활성화하고 백업 및 지속적인 변경 사항을 Amazon S3 버킷에 복사하도록 에이전트를 구성합니다. AWS Backup을 사용하여 백업을 새 데이터베이스로 복원하고 변경 사항을 적용합니다. 새 데이터베이스를 가리키도록 DNS 레코드를 수정합니다.
  • D. AWS Database Migration Service(AWS DMS)를 사용하여 연결된 VPC에서 복제 인스턴스를 시작합니다. AWS Schema Conversion Tool을 사용하여 로컬에서 데이터를 추출하고 데이터를 AWS Snowball Edge Storage Optimized 디바이스로 이동합니다. 디바이스를 AWS로 배송하고 AWS DMS 작업을 사용하여 대상 데이터베이스로의 전송을 완료합니다. 마이그레이션 유형의 경우 기존 데이터를 마이그레이션하고 지속적인 변경 사항을 복제하는 옵션을 선택합니다. 새 데이터베이스를 가리키도록 DNS 레코드를 수정합니다. 최다 투표

Question #: 882
Topic #: 1

A weather forecasting company is migrating an application that stores data on premises in a PostgreSQL database. The Company wants to migrate the database to Amazon Aurora PostgreSQL. The database size grows at an average rate of 5 GB daily and is currently 50 TB. The data center has an internet connection with
50 Mbps of available bandwidth. The migration to AWS must be completed as soon as possible within the next 21 days.
Which data transfer strategy meets these requirements with the LEAST amount of application downtime?

  • A. Take the application offline. Create a local backup of the database. Transmit the database backup file over the existing connection to an Amazon S3 bucket. Use native database tools to restore the backup onto the new database and to set up replication to capture any changes since the backup. Modify the database connection string, and bring the application online.
  • B. Install the Server Migration Connector VM in the local data center. Use the AWS Server Migration Service (AWS SMS) console to replicate the on-premises database to the new database. Modify DNS records to point to the new database.
  • C. Create a local backup of the database, and copy the backup onto an AWS Snowcone device. Activate the AWS DataSync agent on the device, and configure the agent to copy the backup and ongoing changes to an Amazon S3 bucket. Use AWS Backup to restore the backup onto the new database and to apply the changes. Modify DNS records to point to the new database.
  • D. Use AWS Database Migration Service (AWS DMS) to launch a replication instance in a connected VPC. Use the AWS Schema Conversion Tool to extract the data locally and to move the data to an AWS Snowball Edge Storage Optimized device. Ship the device to AWS, and use an AWS DMS task to complete the transfer to the target database. For the migration type, choose the option to migrate existing data and replicate ongoing changes. Modify DNS records to point to the new database. Most Voted
