주제 #: 1

회사에는 컨테이너에서 실행되는 PHP 애플리케이션이 있고 MySQL 데이터베이스도 있습니다. 회사는 애플리케이션과 데이터베이스를 온프레미스에서 AWS로 이동하려고 합니다. 회사에는 수백만 명의 사용자가 있으며 컨테이너 간에 공유된 애플리케이션 데이터를 관리해야 합니다.

회사에는 운영 오버헤드와 비용을 최소화할 수 있는 솔루션이 필요합니다. 또한 솔루션은 확장 가능하고 가용성이 높아야 합니다.

어떤 솔루션이 이러한 요구 사항을 충족합니까?

  • A. Amazon Elastic Container Service(Amazon ECS)에 컨테이너를 배포합니다. Amazon EC2 인스턴스에서 작업을 실행하도록 컨테이너를 설정합니다. 볼륨 유형으로 Amazon Elastic File System(Amazon EFS)을 선택합니다. 읽기 전용 복제본을 데이터베이스 솔루션으로 사용하여 Amazon RDS 다중 AZ DB 인스턴스를 배포합니다.
  • B. Amazon Elastic Container Service(Amazon ECS)에 컨테이너를 배포합니다. AWS Fargate에서 작업을 실행하도록 컨테이너를 설정합니다. 볼륨 유형으로 Amazon Elastic File System(Amazon EFS)을 선택합니다. Amazon Aurora Serverless를 데이터베이스 솔루션으로 배포합니다. 최다 투표
  • C. Amazon Elastic Kubernetes Service(Amazon EKS)에 컨테이너를 배포합니다. AWS Fargate에서 작업을 실행하도록 컨테이너를 설정합니다. 볼륨 유형으로 Amazon S3를 선택합니다. Amazon Aurora Serverless를 데이터베이스 솔루션으로 배포합니다.
  • D. Amazon Elastic Container Service(Amazon ECS)에 컨테이너를 배포합니다. AWS Fargate에서 작업을 실행하도록 컨테이너를 설정합니다. 볼륨 유형으로 Amazon Elastic Block Store(Amazon EBS)를 선택합니다. Amazon Aurora Serverless를 데이터베이스 솔루션으로 배포합니다.

Exam question from Amazon's AWS Certified Solutions Architect - Professional
Question #: 979
Topic #: 1

A company has PHP applications that run in containers and also has a MySQL database. The company wants to move the applications and the database from on premises to AWS. The company has millions of users and will need to manage shared application data between the containers.

The company needs a solution that will minimize operational overhead and cost. The solution also must be scalable and highly available.

Which solution will meet these requirements?

  • A. Deploy containers on Amazon Elastic Container Service (Amazon ECS). Set up the containers to run the tasks on Amazon EC2 instances. Choose Amazon Elastic File System (Amazon EFS) as the volume type. Deploy an Amazon RDS Multi-AZ DB instance with read replicas as the database solution.
  • B. Deploy containers on Amazon Elastic Container Service (Amazon ECS). Set up the containers to run the tasks on AWS Fargate. Choose Amazon Elastic File System (Amazon EFS) as the volume type. Deploy Amazon Aurora Serverless as the database solution. Most Voted
  • C. Deploy containers on Amazon Elastic Kubernetes Service (Amazon EKS). Set up the containers to run the tasks on AWS Fargate. Choose Amazon S3 as the volume type. Deploy Amazon Aurora Serverless as the database solution.
  • D. Deploy containers on Amazon Elastic Container Service (Amazon ECS). Set up the containers to run the tasks on AWS Fargate. Choose Amazon Elastic Block Store (Amazon EBS) as the volume type. Deploy Amazon Aurora Serverless as the database solution.
