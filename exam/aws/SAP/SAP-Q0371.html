사용자가 US-East-1a 지역에서 두 개의 EBS 지원 EC2 인스턴스를 시작했습니다. 사용자가 인스턴스 중 하나의 영역을 변경하려고 합니다. 사용자가 어떻게 변경할 수 있나요?

  • A. 인스턴스가 시작된 후에는 영역을 변경할 수 없습니다.
  • B. AWS EC2 콘솔에서 작업 - > 영역 변경을 선택하고 새 영역을 지정합니다.
  • C. 영역은 AWS CLI를 통해서만 수정할 수 있습니다.
  • D. 인스턴스 중 하나를 중지하고 가용 영역을 변경합니다.
제안된 답변 표시 답변 숨기기
제안 답변: A 🗳️
AWS EC2를 사용하면 사용자가 인스턴스를 시작할 때 시작 시 가용 영역(AZ)을 선택할 수 있습니다. 영역을 선택하지 않으면 AWS가 사용자를 대신하여 영역을 선택합니다. 인스턴스가 시작되면 사용자는 해당 인스턴스의 AMI를 생성하고 여기에서 새 인스턴스를 시작하지 않는 한 해당 인스턴스의 영역을 변경할 수 없습니다.
참조:
http://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-regions-availability-zones.html

Topic #: 1

A user has launched two EBS backed EC2 instances in the US-East-1a region. The user wants to change the zone of one of the instances. How can the user change it?

  • A. It is not possible to change the zone of an instance after it is launched
  • B. From the AWS EC2 console, select the Actions - > Change zones and specify the new zone
  • C. The zone can only be modified using the AWS CLI
  • D. Stop one of the instances and change the availability zone
Show Suggested Answer Hide Answer
Suggested Answer: A 🗳️
With AWS EC2, when a user is launching an instance he can select the availability zone (AZ) at the time of launch. If the zone is not selected, AWS selects it on behalf of the user. Once the instance is launched, the user cannot change the zone of that instance unless he creates an AMI of that instance and launches a new instance from it.
Reference:
http://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-regions-availability-zones.html

A. It is not possible to change the zone of an instance after it is launched. Once an EC2 instance is launched, its availability zone is fixed and cannot be changed. To deploy an instance in a different availability zone, the user would need to terminate the existing instance and launch a new one in the desired availability zone.

It's not possible to move an existing instance to another subnet, Availability Zone, or VPC. Instead, you can manually migrate the instance by creating a new Amazon Machine Image (AMI) from the source instance. Then, launch a new instance using the new AMI in the desired subnet, Availability Zone, or VPC. Finally, you can reassign any Elastic IP addresses from the source instance to the new instance.
