Amazon CloudWatch Logs are showing consistent 400 errors when attempting to update the table. The provisioned write capacity units are appropriately configured, and no throttling is occurring.
What is the LIKELY cause of the failure?

  • A. The ECS service was deleted.
  • B. The ECS configuration does not contain an Auto Scaling group.
  • C. The ECS instance task execution IAM role was modified.
  • D. The ECS task role was modified. Most Voted

For errors in Dynamodb showing 400 could be because of resource not in active state. To update the table in dynamo the resource should be in active state not in updating state. Now according the options, C is correct. ECS does the execution based on ECSTaskExecution Role. The error looks like " "InvalidParameterException: Unable to assume the service linked role. Please verify that the ECS service linked role exists.\n\tstatus code: 400, request id". So for this kind of errors we need to execute this command in your AWS CLI aws iam create-service-linked-role --aws-service-name ecs.amazonaws.com, then use the newly created role's Arn within the ExecutionRoleArn property of your AWS::ECS::TaskDefinition. So answer is C.
