주제 #: 1

한 대기업이 온프레미스 애플리케이션을 AWS 클라우드로 마이그레이션하고 있습니다. 회사의 모든 AWS 계정은 AWS Organizations의 조직에 속합니다.
각 애플리케이션은 별도의 AWS 계정에 있는 자체 VPC에 배포됩니다.
회사는 데이터베이스를 온프레미스에 유지하면서 프런트 엔드 웹 서비스를 마이그레이션하여 마이그레이션 프로세스를 시작하기로 결정했습니다. 데이터베이스는 온프레미스 환경에 특정한 로컬 도메인 이름으로 구성됩니다. 로컬 도메인 이름은 마이그레이션된 웹 서비스에서 확인할 수 있어야 합니다.
최소한의 운영 오버헤드로 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. 새 AWS 계정에 공유 서비스 VPC를 생성하십시오. Amazon Route 53 아웃바운드 해석기를 배포합니다. 관련 온프레미스 도메인의 경우 아웃바운드 확인자 설정을 사용하여 온프레미스 DNS 서버를 가리키는 전달 규칙을 만듭니다. AWS Resource Access Manager를 사용하여 이러한 규칙을 다른 AWS 계정과 공유하십시오. 최다 투표
  • B. 각 VPC에 다중 AZ Amazon Route 53 아웃바운드 확인자를 배포합니다. 각 VPC에 대한 로컬 아웃바운드 확인자에서 온프레미스 DNS 서버를 가리키는 전달 규칙을 만듭니다.
  • C. 새 AWS 계정에 공유 서비스 VPC를 생성합니다. 공유 서비스 VPC 내에서 조건부 전달자 역할을 하는 Amazon EC2 인스턴스를 배포합니다. 이러한 전달자를 DNS 서버로 가리키도록 각 VPC에 설정된 DHCP 옵션을 변경합니다. 이러한 전달자에서 관련 온프레미스 도메인에 대한 전달 규칙을 만듭니다.
  • D. 새 AWS 계정에 공유 서비스 VPC를 생성합니다. Amazon Route 53 인바운드 해석기를 배포합니다. 관련 온프레미스 도메인의 경우 온프레미스 DNS 서버를 가리키는 전달 규칙을 만듭니다. AWS Resource Access Manager를 사용하여 이러한 규칙을 다른 AWS 계정과 공유하십시오.

Exam question from Amazon's AWS Certified Solutions Architect - Professional
Question #: 870
Topic #: 1

A large company is migrating its on-premises applications to the AWS Cloud. All the company's AWS accounts belong to an organization in AWS Organizations.
Each application is deployed into its own VPC in separate AWS accounts.
The company decides to start the migration process by migrating the front-end web services while keeping the databases on premises. The databases are configured with local domain names that are specific to the on-premises environment. The local domain names must be resolvable from the migrated web services.
Which solution will meet these requirements with the LEAST operational overhead?

  • A. Create a shared services VPC in a new AWS account. Deploy Amazon Route 53 outbound resolvers. For relevant on-premises domains, use the outbound resolver settings to create forwarding rules that point to the on-premises DNS servers. Share these rules with the other AWS accounts by using AWS Resource Access Manager. Most Voted
  • B. Deploy Multi-AZ Amazon Route 53 outbound resolvers in each VPC. Create forwarding rules that point to on-premises DNS servers in local outbound resolvers for each VPC.
  • C. Create a shared services VPC in a new AWS account. Deploy Amazon EC2 instances that act conditional forwarders inside the shared services VPC. Change the DHCP options set in each VPC to point to these forwarders as DNS servers. Create forwarding rules for relevant on-premises domains in these forwarders.
  • D. Create a shared services VPC in a new AWS account. Deploy Amazon Route 53 inbound resolvers. For relevant on-premises domains, create forwarding rules that point to on-premises DNS servers. Share these rules with other AWS accounts by using AWS Resource Access Manager.
