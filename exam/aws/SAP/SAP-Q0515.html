회사는 최근 블로거가 자신의 게시물에 동영상을 추가할 수 있는 기능을 추가하여 이전 사용자 트래픽의 10배를 유치했습니다. 하루 중 사용량이 많은 시간대에 사용자는 사이트에 접속하거나 비디오를 시청하려고 할 때 버퍼링 및 시간 초과 문제를 보고합니다.
사용자의 문제를 해결하는 가장 비용 효율적이고 확장 가능한 배포는 무엇입니까?

  • B. 인스턴스 스토어 볼륨을 저장용으로 사용하도록 블로그 사이트를 업데이트합니다. 사이트 콘텐츠를 시작 시 볼륨에 복사하고 종료 시 Amazon S3에 복사합니다.
  • C. Amazon CloudFront 배포를 구성합니다. 배포를 S3 버킷으로 지정하고 비디오를 EFS에서 Amazon S3로 마이그레이션합니다.
  • D. 모든 사이트 콘텐츠에 대해 Amazon CloudFront 배포를 설정하고 ALB에서 배포를 지정합니다.

Topic #: 1

A company used Amazon EC2 instances to deploy a web fleet to host a blog site. The EC2 instances are behind an Application Load Balancer (ALB) and are configured in an Auto Scaling group. The web application stores all blog content on an Amazon EFS volume.
The company recently added a feature for bloggers to add video to their posts, attracting 10 times the previous user traffic. At peak times of day, users report buffering and timeout issues while attempting to reach the site or watch videos.
Which is the MOST cost-efficient and scalable deployment that will resolve the issues for users?

  • A. Reconfigure Amazon EFS to enable maximum I/O.
  • B. Update the blog site to use instance store volumes for storage. Copy the site contents to the volumes at launch and to Amazon S3 at shutdown.
  • C. Configure an Amazon CloudFront distribution. Point the distribution to an S3 bucket, and migrate the videos from EFS to Amazon S3.
  • D. Set up an Amazon CloudFront distribution for all site contents, and point the distribution at the ALB.

답변: D 참고: C는 너무 단순해 보입니다. 거의 흔들렸습니다. A - 부정확합니다. EFS I/O가 얼마나 높은지 달성되지 않습니다. - 파일 시스템 크기에서 작동하며 크기가 클수록 성능도 향상됩니다. B - 부정확함 - 확장 가능하지 않음, 그럴 수도 있고 아닐 수도 있음 성능을 향상시킵니다. C - 올바르지 않음 - 사용자가 블로그 사이트에 접속하기 전에도 웹사이트에 액세스하는 데 문제가 있다는 질문입니다. 비디오를 S3으로 이동해도 웹 사이트 응답 문제가 해결되지 않습니다. D - 올바른 - ALB를 모든 콘텐츠(웹 사이트 + 파일)의 원본으로 사용합니다. 이는 웹 사이트와 비디오의 버퍼링을 제거하는 데 도움이 될 수 있습니다. 참고: 웹 사이트에 액세스하는 사람 수의 10배 - 사람들이 비디오 파일을 업로드/다운로드하고 있기 때문에 웹사이트의 성능이 저하되었다고 가정할 수 없습니다. 일부는 시청할 새 파일이 있다는 것을 알고 탐색 중일 수도 있습니다. 파일을 S3로 오프로드하고 CloudFront에 캐싱하면 비디오 파일 액세스가 향상될 수 있지만 웹 사이트 자체에는 추가 사용자로 인해 여전히 병목 현상이 발생할 수 있습니다. 하루에 영상이 한개만 올라올지도 모르지만 기대감 때문에 이용자 수가 늘어나는군요. EC2 컴퓨팅을 늘리는 것도 도움이 되었을 수 있습니다.
