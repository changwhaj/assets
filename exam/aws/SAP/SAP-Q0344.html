When configuring your customer gateway to connect to your VPC, the________Association is established first between the virtual private gateway and customer gateway using the Pre-Shared Key as the authenticator.

  • A. IPsec
  • B. BGP
  • C. IKE Security

Suggested Answer: C 🗳️
When configuring your customer gateway to connect to your VPC, several steps need to be completed. The IKE Security Association is established first between the virtual private gateway and customer gateway using the Pre-Shared Key as the authenticator.

정답은 다음과 같습니다. C. IKE 보안 설명: Amazon Web Services(AWS)에서 Virtual Private Cloud(VPC)에 연결하도록 고객 게이트웨이를 구성할 때 가상 프라이빗 게이트웨이와 고객 게이트웨이 간의 초기 연결은 IKE를 사용하여 설정됩니다. (인터넷 키 교환) 보안. IKE는 보안 IPsec(인터넷 프로토콜 보안) 연결을 협상하고 설정하는 데 사용되는 프로토콜입니다. 고객 게이트웨이와 가상 프라이빗 게이트웨이 간의 IPsec 터널에 필요한 인증 및 암호화 설정을 처리합니다. 이 초기 연결 중에는 사전 공유 키(PSK)가 인증자로 사용됩니다. 이는 고객 게이트웨이와 가상 프라이빗 게이트웨이 간의 공유 암호입니다.

Selected Answer: C
The correct answer is: C. IKE Security Explanation: When configuring a customer gateway to connect to a Virtual Private Cloud (VPC) in Amazon Web Services (AWS), the initial association between the virtual private gateway and the customer gateway is established using IKE (Internet Key Exchange) Security. IKE is a protocol used to negotiate and establish secure IPsec (Internet Protocol Security) connections. It handles the authentication and encryption settings required for the IPsec tunnel between the customer gateway and the virtual private gateway. During this initial association, the Pre-Shared Key (PSK) is used as the authenticator, which is a shared secret between the customer gateway and the virtual private gateway.
