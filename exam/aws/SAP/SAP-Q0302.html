CloudFormation에서 Amazon Elastic Block Store를 Amazon EC2 인스턴스에 매핑하려는 경우 _________.

  • A. 논리적 ID를 참조하여 블록 스토어를 인스턴스와 연결합니다.
  • B. 리소스 유형과 함께 인스턴스의 물리적 ID를 참조합니다.
  • C. 리소스 속성과 함께 블록 스토어의 인스턴스 ID를 참조합니다.

In CloudFormation, if you want to map an Amazon Elastic Block Store to an Amazon EC2 instance, _________.

  • A. you reference the logical IDs to associate the block stores with the instance
  • B. you reference the physical IDs of the instance along with the resource type
  • C. you reference the instance IDs of the block store along with the resource properties
  • D. you reference the physical IDs of both the block stores and the instance
Show Suggested Answer Hide Answer
Suggested Answer: A 🗳️
In AWS CloudFormation, if you want to map an Amazon Elastic Block Store to an Amazon EC2 instance, you reference the logical IDs to associate the block stores with the instance.

Selected Answer: A
The correct answer is: A. you reference the logical IDs to associate the block stores with the instance. In CloudFormation, to map an Amazon Elastic Block Store (EBS) to an Amazon EC2 instance, you reference the logical IDs of the resources. Logical IDs are defined within the CloudFormation template and are used to associate resources with each other. By referencing the logical IDs, you can specify the relationship between the EC2 instance and the EBS volumes that should be attached to it. Option B is incorrect because physical IDs are not used in CloudFormation to associate resources. Option C is incorrect because instance IDs are not used to reference block stores in CloudFormation. Option D is incorrect because referencing the physical IDs of both the block stores and the instance is not the standard practice in CloudFormation for mapping resources.
