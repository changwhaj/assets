주제 #: 1

회사는 HTTP API를 제공하는 웹 애플리케이션을 실행합니다. API는 세션 지속성을 위해 MySQL 호환 SQL 데이터베이스를 사용합니다. 기존 워크로드는 Kubernetes 클러스터의 온프레미스 환경에서 실행됩니다. 이 회사는 온프레미스 서버 관리와 관련하여 상당한 운영 오버헤드를 안고 있으며 AWS로의 마이그레이션을 고려하고 있습니다.
회사는 가능한 한 많은 관리형 AWS 제품을 사용하여 서버나 인스턴스를 관리할 필요성을 없애기를 원합니다. 회사는 웹 애플리케이션이나 HTTP API에 큰 변화를 가져오는 것을 원하지 않습니다. 솔루션 설계자는 회사가 이러한 목표를 달성하는 데 사용할 수 있는 다양한 아키텍처 솔루션을 권장해야 합니다.
어떤 솔루션이 이러한 요구 사항을 충족합니까? (3개를 선택하세요.)

  • A. AWS에서 Kubernetes 클러스터를 생성합니다. Amazon Aurora 서버리스 클러스터를 배포합니다. 관리형 노드 그룹을 사용하여 Kubernetes에서 웹 애플리케이션을 실행합니다. Kubernetes에서 실행되는 컨테이너화된 애플리케이션을 사용하여 웹 애플리케이션을 노출하는 Application Load Balancer를 배포합니다.
  • B. 웹 애플리케이션용 Auto Scaling 그룹을 생성합니다. Amazon EC2 범용 인스턴스에서 웹 애플리케이션과 MySQL 데이터베이스를 실행합니다. ALB(Application Load Balancer)를 배포합니다. ALB를 Auto Scaling 그룹과 연결합니다.
  • C. 웹 애플리케이션을 위한 새로운 Amazon Elastic Container Service(Amazon ECS) 서비스를 생성합니다. Amazon RDS를 사용하여 MySQL을 배포합니다. 서버리스 컴퓨팅 엔진을 사용하여 Amazon ECS에서 웹 애플리케이션을 작업으로 실행합니다. 상태 확인을 구성합니다. AWS Cloud Map 및 DNS를 사용하여 서비스의 정상적인 작업으로 트래픽을 전달합니다. 최다 투표
  • D. AWS에서 Kubernetes 클러스터를 생성합니다. Amazon Aurora 서버리스 클러스터를 배포합니다. AWS Fargate 및 Amazon Elastic Kubernetes Service(Amazon EKS)를 사용하여 웹 애플리케이션을 실행합니다. Kubernetes에서 실행되는 컨테이너화된 애플리케이션을 사용하여 웹 애플리케이션을 노출하는 Application Load Balancer를 배포합니다. 최다 투표
  • E. 웹 애플리케이션을 위한 새로운 Amazon Elastic Container Service(Amazon ECS) 서비스를 생성합니다. MySQL 호환 Amazon Aurora 클러스터를 배포합니다. 외부 시작과 함께 Amazon ECS를 사용하여 웹 애플리케이션을 작업으로 실행합니다. ALB(Application Load Balancer)를 배포합니다. ALB를 ECS 서비스와 연결합니다.
  • F. 웹 애플리케이션용 Amazon Lightsail 컨테이너 배포를 생성합니다. MySQL 모드에서 고가용성 Lightsail 데이터베이스를 생성합니다. 컨테이너 배포를 위한 공용 끝점을 지정합니다. 퍼블릭 엔드포인트에 대한 사용자 지정 도메인을 추가합니다. 최다 투표

Exam question from Amazon's AWS Certified Solutions Architect - Professional
Question #: 887
Topic #: 1

A company runs a web application that provides an HTTP API. The API uses a MySQL-compatible SQL database for session persistence. The existing workload runs in an on-premises environment on a Kubernetes cluster. The company has significant operational overhead associated with managing servers on premises and is considering a migration to AWS.
The company wants to remove the need to manage servers or instances by using as many managed AWS offerings as possible. The company does not want to introduce significant changes to the web application or the HTTP API. A solutions architect must recommend different architecture solutions that the company can use to achieve these goals.
Which solutions will meet these requirements? (Choose three.)

  • A. Create a Kubernetes cluster on AWS. Deploy an Amazon Aurora Serverless cluster. Run the web application on Kubernetes by using a managed node group. Use a containerized application that runs on Kubernetes to deploy an Application Load Balancer that exposes the web application.
  • B. Create an Auto Scaling group for the web application. Run the web application and a MySQL database on Amazon EC2 general purpose instances. Deploy an Application Load Balancer (ALB). Associate the ALB with the Auto Scaling group.
  • C. Create a new Amazon Elastic Container Service (Amazon ECS) service for the web application. Deploy MySQL by using Amazon RDS. Run the web application as tasks in Amazon ECS with a serverless compute engine. Configure health checks. Use AWS Cloud Map and DNS to direct traffic to healthy tasks for the service. Most Voted
  • D. Create a Kubernetes cluster on AWS. Deploy an Amazon Aurora Serverless cluster. Run the web application by using AWS Fargate and Amazon Elastic Kubernetes Service (Amazon EKS). Use containerized application that runs on Kubernetes to deploy an Application Load Balancer that exposes the web application. Most Voted
  • E. Create a new Amazon Elastic Container Service (Amazon ECS) service for the web application. Deploy a MySQL-compatible Amazon Aurora cluster. Run the web application as tasks by using Amazon ECS with EXTERNAL launch. Deploy an Application Load Balancer (ALB). Associate the ALB with the ECS service.
  • F. Create an Amazon Lightsail containers deployment for the web application. Create a highly available Lightsail database in MySQL mode. Specify a public endpoint for the container deployment. Add a custom domain for the public endpoint. Most Voted
