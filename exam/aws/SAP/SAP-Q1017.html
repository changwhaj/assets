주제 #: 1

한 회사의 솔루션 설계자가 100만 명 이상의 학생을 지원하는 학습 플랫폼을 관리하고 있습니다. 회사의 비즈니스 보고 팀은 데이터베이스에서 대규모 데이터 세트를 추출하는 동안 성능이 저하되고 있습니다. 학습 애플리케이션은 PHP를 기반으로 하며 ALB(Application Load Balancer) 뒤의 Amazon EC2 Auto Scaling 그룹에 있는 Amazon EC2 인스턴스에서 실행됩니다. 애플리케이션 데이터는 Amazon S3 버킷과 Amazon RDS for MySOL 데이터베이스에 저장됩니다. ALB는 Amazon CloudFront 배포의 원본입니다.

솔루션 설계자는 SELECT 쿼리에 대한 느린 읽기 작업이 RDS for MySOL DB 인스턴스의 CPU 사용률에 영향을 미치는 것을 관찰했습니다. 솔루션 설계자는 가동 중지 시간이 거의 없이 느린 웹 사이트 성능을 개선할 수 있는 확장 가능한 솔루션을 찾아야 합니다. 또한 솔루션은 데이터 손실 없이 자동 장애 조치를 제공해야 합니다.

어떤 솔루션이 이러한 요구 사항을 충족합니까?

  • A. Percona XtraBackup을 사용하여 증분 데이터베이스 백업을 생성합니다. 백업 파일을 압축합니다. 백업 파일을 Amazon S3에 동기화합니다. Amazon S3의 백업 파일을 Amazon Aurora MySOL로 복원합니다. 애플리케이션 엔드포인트를 새 Aurora DB 인스턴스로 연결합니다.
  • B. DB 인스턴스를 다중 AZ 배포로 변환합니다. 데이터베이스의 query_cache_type 매개변수를 0으로 설정합니다. CloudFront 캐싱 TTL을 늘려 애플리케이션 서버 CPU 사용률을 줄이세요.
  • C. DB 인스턴스에서 Amazon Aurora 읽기 전용 복제본을 생성합니다. 읽기 전용 복제본이 원본 DB 인스턴스와 동기화될 때까지 기다립니다. 읽기 전용 복제본을 독립형 DB 클러스터로 승격합니다. 애플리케이션 엔드포인트를 새 Aurora DB 인스턴스로 연결합니다. 최다 투표
  • D. DB 인스턴스에 읽기 전용 복제본 클러스터를 생성합니다. 다중 AZ 배포를 사용합니다. 읽기 전용 복제본을 기본 DB 인스턴스와 동기화합니다. 읽기 전용 복제본을 기본 DB 인스턴스로 승격합니다.

Exam question from Amazon's AWS Certified Solutions Architect - Professional
Question #: 1017
Topic #: 1

A company's solutions architect is managing a learning platform that supports more than 1 million students. The company's business reporting team is experiencing slow performance while extracting large datasets from the database. The learning application is based on PHP and runs on Amazon EC2 instances that are in an Amazon EC2 Auto Scaling group behind an Application Load Balancer (ALB). Application data is stored in an Amazon S3 bucket and in an Amazon RDS for MySOL database. The ALB is the origin of an Amazon CloudFront distribution.

The solutions architect observes that slow read operations for SELECT queries are affecting the RDS for MySOL DB instance's CPU utilization. The solutions architect must find a scalable solution to improve the slow website performance with near-zero downtime. The solution also must provide automatic failover with no data loss.

Which solution will meet these requirements?

  • A. Create an incremental database backup by using Percona XtraBackup. Compress the backup files. Synchronize the backup files to Amazon S3. Restore the backup files from Amazon S3 to Amazon Aurora MySOL. Direct the application endpoint to the new Aurora DB instance.
  • B. Convert the DB instance to a Multi-AZ deployment. Set the query_cache_type parameter on the database to zero. Increase the CloudFront caching TTL to reduce application server CPU utilization.
  • C. Create an Amazon Aurora read replica from the DB instance. Wait until the read replica is synchronized with the source DB instance. Promote the read replica to a standalone DB cluster. Direct the application endpoint to the new Aurora DB instance. Most Voted
  • D. Create a read replica cluster on the DB instance. Use a Multi-AZ deployment. Synchronize the read replica with the primary DB instance. Promote the read replica as the primary DB instance.

Selected Answer C: the statement "RDS also supports automatic failover in 60 seconds" is not entirely accurate. While RDS does support automatic failover, the time it takes for the failover to complete can vary depending on the specific circumstances of the failure and the configuration of the RDS instance. Some of the search results suggest that the failover process can typically be completed within 60-120 seconds, while others suggest that it can take as little as 5 seconds or up to 30 seconds for Aurora.
