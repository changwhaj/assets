Amazon의 AWS 공인 데이터 분석 – 전문 분야 시험 문제
질문 #: 101
주제 #: 1

회사는 거의 실시간으로 여러 부서의 이벤트 데이터를 수집하고 처리하려고 합니다. Amazon S3에 데이터를 저장하기 전에 회사는 주소 및 타임스탬프 열의 형식을 표준화하여 데이터를 정리해야 합니다. 데이터의 크기는 각 특정 시점의 전체 로드에 따라 달라집니다. 단일 데이터 레코드의 크기는 100KB~10MB입니다.
데이터 분석 전문가는 데이터 수집을 위한 솔루션을 어떻게 설계해야 합니까?

  • A. Amazon Kinesis Data Streams를 사용하십시오. 원시 데이터에 대한 스트림을 구성합니다. Kinesis 에이전트를 사용하여 스트림에 데이터를 씁니다. 원시 스트림에서 데이터를 읽고 정리한 후 출력을 Amazon S3에 저장하는 Amazon Kinesis Data Analytics 애플리케이션을 생성합니다.
  • B. Amazon Kinesis Data Firehose를 사용하십시오. 데이터 정리를 위한 사전 처리 AWS Lambda 함수를 사용하여 Firehose 전송 스트림을 구성합니다. Kinesis 에이전트를 사용하여 전송 스트림에 데이터를 씁니다. Amazon S3에 데이터를 전송하도록 Kinesis Data Firehose를 구성합니다.
  • C. Apache Kafka에 Amazon Managed Streaming을 사용하십시오. 원시 데이터에 대한 주제를 구성합니다. Kafka 생산자를 사용하여 주제에 데이터를 씁니다. Apache Kafka 소비자 API를 사용하여 주제에서 데이터를 읽고, 데이터를 정리하고, Amazon S3에 쓰는 애플리케이션을 Amazon EC2에서 생성합니다. 최다 투표
  • D. Amazon Simple Queue Service(Amazon SQS)를 사용합니다. SQS 대기열에서 이벤트를 읽고 해당 이벤트를 Amazon S3에 업로드하도록 AWS Lambda 함수를 구성합니다.

Exam question from Amazon's AWS Certified Data Analytics - Specialty
Question #: 101
Topic #: 1

A company wants to collect and process events data from different departments in near-real time. Before storing the data in Amazon S3, the company needs to clean the data by standardizing the format of the address and timestamp columns. The data varies in size based on the overall load at each particular point in time. A single data record can be 100 KB-10 MB.
How should a data analytics specialist design the solution for data ingestion?

  • A. Use Amazon Kinesis Data Streams. Configure a stream for the raw data. Use a Kinesis Agent to write data to the stream. Create an Amazon Kinesis Data Analytics application that reads data from the raw stream, cleanses it, and stores the output to Amazon S3.
  • B. Use Amazon Kinesis Data Firehose. Configure a Firehose delivery stream with a preprocessing AWS Lambda function for data cleansing. Use a Kinesis Agent to write data to the delivery stream. Configure Kinesis Data Firehose to deliver the data to Amazon S3.
  • C. Use Amazon Managed Streaming for Apache Kafka. Configure a topic for the raw data. Use a Kafka producer to write data to the topic. Create an application on Amazon EC2 that reads data from the topic by using the Apache Kafka consumer API, cleanses the data, and writes to Amazon S3. Most Voted
  • D. Use Amazon Simple Queue Service (Amazon SQS). Configure an AWS Lambda function to read events from the SQS queue and upload the events to Amazon S3.
