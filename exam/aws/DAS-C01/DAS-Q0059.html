Amazon의 AWS 공인 데이터 분석 – 전문 분야 시험 문제
질문 #: 59
주제 #: 1

전 세계적으로 수백만 명의 사용자를 보유하고 있는 한 온라인 소매 회사는 전자상거래 분석 기능을 향상시키고 싶어합니다. 현재 클릭스트림 데이터는 압축 파일로 Amazon S3에 직접 업로드됩니다. Amazon EC2에서 실행되는 애플리케이션은 매일 여러 번 데이터를 처리하고 편집자와 마케팅 담당자가 시각화할 수 있도록 검색 옵션과 보고서를 제공합니다. 회사는 편집자와 마케팅 담당자가 웹사이트 클릭과 집계된 데이터를 몇 분 안에 사용할 수 있도록 하여 사용자와 보다 효과적으로 연결할 수 있도록 하려고 합니다.
가장 효율적인 방식으로 이러한 요구 사항을 충족하는 데 도움이 되는 옵션은 무엇입니까? (2개를 선택하세요.)

  • A. Amazon Kinesis Data Firehose를 사용하여 압축 및 일괄 처리된 클릭스트림 레코드를 Amazon OpenSearch Service(Amazon Elasticsearch Service)에 업로드하십시오. 최다 투표
  • B. 클릭스트림 레코드를 Amazon S3에 압축 파일로 업로드합니다. 그런 다음 AWS Lambda를 사용하여 Amazon S3에서 Amazon OpenSearch Service(Amazon Elasticsearch Service)로 데이터를 보냅니다.
  • C. Amazon EC2에 배포된 Amazon OpenSearch Service(Amazon Elasticsearch Service)를 사용하여 데이터를 집계, 필터링 및 처리합니다. 거의 실시간으로 콘텐츠 성능 대시보드를 새로 고칩니다.
  • D. OpenSearch 대시보드(Kibana)를 사용하여 Amazon OpenSearch Service(Amazon Elasticsearch Service)에 저장된 데이터를 집계, 필터링 및 시각화합니다. 거의 실시간으로 콘텐츠 성능 대시보드를 새로 고칩니다. 최다 투표
  • E. Amazon S3에서 Amazon Kinesis Data Streams로 클릭스트림 레코드를 업로드하고 Kinesis Data Streams 소비자를 사용하여 Amazon OpenSearch Service(Amazon Elasticsearch Service)로 레코드를 보냅니다.

Exam question from Amazon's AWS Certified Data Analytics - Specialty
Question #: 59
Topic #: 1

An online retail company with millions of users around the globe wants to improve its ecommerce analytics capabilities. Currently, clickstream data is uploaded directly to Amazon S3 as compressed files. Several times each day, an application running on Amazon EC2 processes the data and makes search options and reports available for visualization by editors and marketers. The company wants to make website clicks and aggregated data available to editors and marketers in minutes to enable them to connect with users more effectively.
Which options will help meet these requirements in the MOST efficient way? (Choose two.)

  • A. Use Amazon Kinesis Data Firehose to upload compressed and batched clickstream records to Amazon OpenSearch Service (Amazon Elasticsearch Service). Most Voted
  • B. Upload clickstream records to Amazon S3 as compressed files. Then use AWS Lambda to send data to Amazon OpenSearch Service (Amazon Elasticsearch Service) from Amazon S3.
  • C. Use Amazon OpenSearch Service (Amazon Elasticsearch Service) deployed on Amazon EC2 to aggregate, filter, and process the data. Refresh content performance dashboards in near-real time.
  • D. Use OpenSearch Dashboards (Kibana) to aggregate, filter, and visualize the data stored in Amazon OpenSearch Service (Amazon Elasticsearch Service). Refresh content performance dashboards in near-real time. Most Voted
  • E. Upload clickstream records from Amazon S3 to Amazon Kinesis Data Streams and use a Kinesis Data Streams consumer to send records to Amazon OpenSearch Service (Amazon Elasticsearch Service).
