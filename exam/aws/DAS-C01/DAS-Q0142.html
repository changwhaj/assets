Amazon의 AWS 공인 데이터 분석 – 전문 분야 시험 문제
질문 #: 142
주제 #: 1

회사는 온프레미스에서 Apache Flink 애플리케이션을 호스팅합니다. 애플리케이션은 여러 Apache Kafka 클러스터의 데이터를 처리합니다. 데이터는 웹 애플리케이션, 모바일 앱, 운영 데이터베이스 등 다양한 소스에서 발생합니다. 회사는 이러한 소스 중 일부를 AWS로 마이그레이션했으며 이제 Flink 애플리케이션을 마이그레이션하려고 합니다. 회사는 VPC 내의 데이터베이스에 있는 데이터가 인터넷을 통과하지 않도록 해야 합니다. 애플리케이션은 회사의 AWS 솔루션, 온프레미스 리소스 및 공용 인터넷에서 제공되는 모든 데이터를 처리할 수 있어야 합니다.
최소한의 운영 오버헤드로 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. 회사 VPC 내의 Amazon EC2에 Flink를 구현합니다. VPC에 Amazon Managed Streaming for Apache Kafka(Amazon MSK) 클러스터를 생성하여 VPC 내의 애플리케이션 및 데이터베이스에서 제공되는 데이터를 수집합니다. Amazon Kinesis Data Streams를 사용하여 공용 인터넷에서 제공되는 데이터를 수집합니다. AWS Client VPN 또는 AWS Direct Connect를 사용하여 Kinesis Data Streams Amazon MSK 및 온프레미스 Kafka 클러스터의 소스를 갖도록 Flink를 구성합니다.
  • B. 회사 VPC 내의 Amazon EC2에 Flink를 구현합니다. Amazon Kinesis Data Streams를 사용하여 VPC 및 퍼블릭 인터넷 내의 애플리케이션과 데이터베이스에서 나오는 데이터를 수집합니다. AWS Client VPN 또는 AWS Direct Connect를 사용하여 Kinesis Data Streams 및 온프레미스 Kafka 클러스터의 소스를 갖도록 Flink를 구성합니다.
  • C. 컴파일된 Flink .jar 파일을 업로드하여 Amazon Kinesis Data Analytics 애플리케이션을 생성합니다. Amazon Kinesis Data Streams를 사용하여 VPC 및 퍼블릭 인터넷 내의 애플리케이션과 데이터베이스에서 나오는 데이터를 수집합니다. AWS Client VPN 또는 AWS Direct Connect를 사용하여 Kinesis Data Streams 및 온프레미스 Kafka 클러스터의 소스를 갖도록 Kinesis Data Analytics 애플리케이션을 구성합니다. 최다 투표
  • D. 컴파일된 Flink .jar 파일을 업로드하여 Amazon Kinesis Data Analytics 애플리케이션을 생성합니다. 회사의 VPC에 Amazon Managed Streaming for Apache Kafka(Amazon MSK) 클러스터를 생성하여 VPC 내의 애플리케이션 및 데이터베이스에서 나오는 데이터를 수집합니다. Amazon Kinesis Data Streams를 사용하여 공용 인터넷에서 제공되는 데이터를 수집합니다. AWS Client VPN 또는 AWS Direct Connect를 사용하여 Kinesis Data Streams, Amazon MSK 및 온프레미스 Kafka 클러스터의 소스를 갖도록 Kinesis Data Analytics 애플리케이션을 구성합니다.

Exam question from Amazon's AWS Certified Data Analytics - Specialty
Question #: 142
Topic #: 1

A company hosts an Apache Flink application on premises. The application processes data from several Apache Kafka clusters. The data originates from a variety of sources, such as web applications, mobile apps, and operational databases. The company has migrated some of these sources to AWS and now wants to migrate the Flink application. The company must ensure that data that resides in databases within the VPC does not traverse the internet. The application must be able to process all the data that comes from the company's AWS solution, on-premises resources, and the public internet.
Which solution will meet these requirements with the LEAST operational overhead?

  • A. Implement Flink on Amazon EC2 within the company's VPC. Create Amazon Managed Streaming for Apache Kafka (Amazon MSK) clusters in the VPC to collect data that comes from applications and databases within the VPC. Use Amazon Kinesis Data Streams to collect data that comes from the public internet. Configure Flink to have sources from Kinesis Data Streams Amazon MSK, and any on-premises Kafka clusters by using AWS Client VPN or AWS Direct Connect.
  • B. Implement Flink on Amazon EC2 within the company's VPC. Use Amazon Kinesis Data Streams to collect data that comes from applications and databases within the VPC and the public internet. Configure Flink to have sources from Kinesis Data Streams and any on-premises Kafka clusters by using AWS Client VPN or AWS Direct Connect.
  • C. Create an Amazon Kinesis Data Analytics application by uploading the compiled Flink .jar file. Use Amazon Kinesis Data Streams to collect data that comes from applications and databases within the VPC and the public internet. Configure the Kinesis Data Analytics application to have sources from Kinesis Data Streams and any on-premises Kafka clusters by using AWS Client VPN or AWS Direct Connect. Most Voted
  • D. Create an Amazon Kinesis Data Analytics application by uploading the compiled Flink .jar file. Create Amazon Managed Streaming for Apache Kafka (Amazon MSK) clusters in the company's VPC to collect data that comes from applications and databases within the VPC. Use Amazon Kinesis Data Streams to collect data that comes from the public internet. Configure the Kinesis Data Analytics application to have sources from Kinesis Data Streams, Amazon MSK, and any on-premises Kafka clusters by using AWS Client VPN or AWS Direct Connect.
