질문 #: 37
주제 #: 1

한 기술 회사에서 시간에 민감한 데이터를 시각화하고 분석하는 대시보드를 만들고 있습니다. 데이터는 버터 간격이 60초로 설정된 Amazon Kinesis Data Firehose를 통해 수신됩니다. 대시보드는 거의 실시간 데이터를 지원해야 합니다.
이러한 요구 사항을 충족하는 시각화 솔루션은 무엇입니까?

  • A. Amazon OpenSearch Service(Amazon Elasticsearch Service)를 Kinesis Data Firehose의 엔드포인트로 선택합니다. 원하는 분석 및 시각화와 함께 Amazon OpenSearch Service(Amazon ES)의 데이터를 사용하여 OpenSearch 대시보드(Kibana)를 설정합니다. 최다 투표
  • B. Amazon S3를 Kinesis Data Firehose의 엔드포인트로 선택합니다. Amazon SageMaker Jupyter 노트북으로 데이터를 읽고 원하는 분석 및 시각화를 수행합니다.
  • C. Amazon Redshift를 Kinesis Data Firehose의 엔드포인트로 선택합니다. SPICE가 포함된 Amazon QuickSight를 Amazon Redshift에 연결하여 원하는 분석 및 시각화를 생성하세요.
  • D. Amazon S3를 Kinesis Data Firehose의 엔드포인트로 선택합니다. AWS Glue를 사용하여 데이터를 분류하고 Amazon Athena를 사용하여 쿼리합니다. SPICE가 포함된 Amazon QuickSight를 Athena에 연결하여 원하는 분석 및 시각화를 생성하세요.

Question #: 37
Topic #: 1

A technology company is creating a dashboard that will visualize and analyze time-sensitive data. The data will come in through Amazon Kinesis Data Firehose with the butter interval set to 60 seconds. The dashboard must support near-real-time data.
Which visualization solution will meet these requirements?

  • A. Select Amazon OpenSearch Service (Amazon Elasticsearch Service) as the endpoint for Kinesis Data Firehose. Set up an OpenSearch Dashboards (Kibana) using the data in Amazon OpenSearch Service (Amazon ES) with the desired analyses and visualizations. Most Voted
  • B. Select Amazon S3 as the endpoint for Kinesis Data Firehose. Read data into an Amazon SageMaker Jupyter notebook and carry out the desired analyses and visualizations.
  • C. Select Amazon Redshift as the endpoint for Kinesis Data Firehose. Connect Amazon QuickSight with SPICE to Amazon Redshift to create the desired analyses and visualizations.
  • D. Select Amazon S3 as the endpoint for Kinesis Data Firehose. Use AWS Glue to catalog the data and Amazon Athena to query it. Connect Amazon QuickSight with SPICE to Athena to create the desired analyses and visualizations.
