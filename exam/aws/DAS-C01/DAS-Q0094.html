Amazon의 AWS 공인 데이터 분석 – 전문 분야 시험 문제
질문 #: 94
주제 #: 1

한 소매 회사는 AWS Glue 데이터 카탈로그에 대한 임시 쿼리를 위해 Amazon Athena를 활용합니다. 데이터 분석 팀은 회사의 데이터 카탈로그와 데이터 액세스를 관리합니다. 데이터 분석 팀은 쿼리를 분리하고 다양한 워크로드 및 팀별로 해당 쿼리를 실행하는 데 드는 비용을 관리하려고 합니다.
이상적으로, 데이터 분석가는 팀 내의 다양한 사용자가 실행하는 쿼리를 그룹화하고, 쿼리 결과를 각 팀과 관련된 개별 Amazon S3 버킷에 저장하고, 데이터 카탈로그에 대해 실행되는 쿼리에 비용 제약을 적용하기를 원합니다.
이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. 회사 내 각 팀에 대한 IAM 그룹 및 리소스 태그를 생성합니다. Data Catalog 리소스에 대한 사용자 액세스 및 작업을 제어하는 ​​IAM 정책을 설정합니다.
  • B. 회사 내 각 팀에 대한 Athena 리소스 그룹을 생성하고 이 그룹에 사용자를 할당합니다. 리소스 그룹의 속성 목록에 S3 버킷 이름 및 기타 쿼리 구성을 추가합니다.
  • C. 회사 내 각 팀에 대해 Athena 작업 그룹을 만듭니다. 작업 그룹 리소스에 대한 사용자 액세스 및 작업을 제어하는 ​​IAM 작업 그룹 정책을 설정합니다.
  • D. 회사 내 각 팀에 대한 Athena 쿼리 그룹을 생성하고 해당 그룹에 사용자를 할당합니다.

Exam question from Amazon's AWS Certified Data Analytics - Specialty
Question #: 94
Topic #: 1

A retail company leverages Amazon Athena for ad-hoc queries against an AWS Glue Data Catalog. The data analytics team manages the data catalog and data access for the company. The data analytics team wants to separate queries and manage the cost of running those queries by different workloads and teams.
Ideally, the data analysts want to group the queries run by different users within a team, store the query results in individual Amazon S3 buckets specific to each team, and enforce cost constraints on the queries run against the Data Catalog.
Which solution meets these requirements?

  • A. Create IAM groups and resource tags for each team within the company. Set up IAM policies that control user access and actions on the Data Catalog resources.
  • B. Create Athena resource groups for each team within the company and assign users to these groups. Add S3 bucket names and other query configurations to the properties list for the resource groups.
  • C. Create Athena workgroups for each team within the company. Set up IAM workgroup policies that control user access and actions on the workgroup resources.
  • D. Create Athena query groups for each team within the company and assign users to the groups.

정답은 C입니다. Athena 작업 그룹은 팀, 애플리케이션 및 워크로드에 대한 쿼리를 격리하는 데 도움이 될 수 있습니다. 또한 결과를 별도의 S3 버킷에 저장하고 비용 제약을 적용하는 데 도움이 됩니다. 작업 그룹에 대한 액세스는 IAM을 사용하여 제어할 수 있습니다. AWS 문서에 따르면 작업 그룹을 사용하여 팀, 애플리케이션 또는 다양한 워크로드에 대한 쿼리를 격리하는 것이 좋습니다. 예를 들어 조직의 서로 다른 두 팀에 대해 별도의 작업 그룹을 만들 수 있습니다. 워크로드를 분리할 수도 있습니다. 예를 들어 두 개의 독립적인 작업 그룹을 생성할 수 있습니다. 하나는 보고서 생성과 같은 자동화된 예약 애플리케이션용이고 다른 하나는 분석가의 임시 사용용입니다. 작업그룹 간에 전환할 수 있습니다. 작업 그룹에 대한 액세스를 제어하려면 리소스 수준 IAM 권한 또는 자격 증명 기반 IAM 정책을 사용하세요. https://docs.aws.amazon.com/athena/latest/ug/workgroups.html

Correct answer is C as Athena workgroups can help isolate queries for teams, applications, and workloads. It also helps store the results in a separate S3 bucket and enforce cost constraints. Access to workgroups can be controlled using IAM. AWS doc says : We recommend using workgroups to isolate queries for teams, applications, or different workloads. For example, you may create separate workgroups for two different teams in your organization. You can also separate workloads. For example, you can create two independent workgroups, one for automated scheduled applications, such as report generation, and another for ad-hoc usage by analysts. You can switch between workgroups. To control access to workgroups, use resource-level IAM permissions or identity-based IAM policies. https://docs.aws.amazon.com/athena/latest/ug/workgroups.html
