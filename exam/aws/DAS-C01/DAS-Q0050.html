Amazon의 AWS 공인 데이터 분석 – 전문 분야 시험 문제
질문 #: 50
주제 #: 1

한 회사에서는 추천 엔진에 더 많은 기능을 추가하여 스마트 홈 시스템에 대한 사용자 만족도를 높이고자 합니다. 각 센서는 Java의 Kinesis Producer Library(KPL)를 사용하여 중첩된 JSON 데이터를 Amazon Kinesis Data Streams에 비동기식으로 푸시합니다. 실패한 센서 세트의 통계에 따르면 센서가 오작동할 때 기록된 데이터가 항상 클라우드로 전송되지 않는 것으로 나타났습니다.
회사에는 가장 최근에 업데이트된 센서의 데이터에 대해 거의 실시간 분석을 제공하는 솔루션이 필요합니다.
회사에서 이러한 요구 사항을 충족할 수 있는 솔루션은 무엇입니까?

  • A. KPL의 RecordMaxBufferedTime 속성을 "גˆ'1"로 설정하여 센서 측 버퍼링을 비활성화합니다. Kinesis Data Analytics를 사용하면 회사에서 개발한 이상 탐지 SQL 스크립트를 기반으로 데이터를 강화할 수 있습니다. 강화된 데이터를 Kinesis 데이터 스트림 집합에 푸시하고 데이터 변환 기능을 활성화하여 JSON 파일을 평면화합니다. 고밀도 스토리지 Amazon Redshift 클러스터를 인스턴스화하고 이를 Kinesis Data Firehose 전송 스트림의 대상으로 사용합니다.
  • B. Java용 AWS SDK를 통해 Kinesis Data Streams API의 PutRecord/PutRecords 호출을 사용하도록 센서 코드를 업데이트합니다. Kinesis Data Analytics를 사용하면 회사에서 개발한 이상 탐지 SQL 스크립트를 기반으로 데이터를 강화할 수 있습니다. KDA 애플리케이션의 출력을 Kinesis Data Firehose 전송 스트림으로 보내고, 데이터 변환 기능을 활성화하여 JSON 파일을 평면화하고, Kinesis Data Firehose 대상을 Amazon OpenSearch Service(Amazon Elasticsearch Service) 클러스터로 설정합니다. 최다 투표
  • C. KPL의 RecordMaxBufferedTime 속성을 "0"으로 설정하여 센서 측 버퍼링을 비활성화합니다. 각 스트림에 대해 전용 Kinesis Data Firehose 전송 스트림을 연결하고 데이터 변환 기능을 활성화하여 JSON 파일을 Amazon S3 버킷으로 보내기 전에 평면화합니다. S3 데이터를 Amazon Redshift 클러스터에 로드합니다.
  • D. Java용 AWS SDK를 통해 Kinesis Data Streams API의 PutRecord/PutRecords 호출을 사용하도록 센서 코드를 업데이트합니다. AWS Glue를 사용하여 Kinesis 클라이언트 라이브러리(KCL)를 사용하여 스트림에서 데이터를 가져오고 처리합니다. Amazon Elasticsearch Service 클러스터를 인스턴스화하고 AWS Lambda를 사용하여 데이터를 직접 푸시합니다.

Exam question from Amazon's AWS Certified Data Analytics - Specialty
Question #: 50
Topic #: 1

A company wants to improve user satisfaction for its smart home system by adding more features to its recommendation engine. Each sensor asynchronously pushes its nested JSON data into Amazon Kinesis Data Streams using the Kinesis Producer Library (KPL) in Java. Statistics from a set of failed sensors showed that, when a sensor is malfunctioning, its recorded data is not always sent to the cloud.
The company needs a solution that offers near-real-time analytics on the data from the most updated sensors.
Which solution enables the company to meet these requirements?

  • A. Set the RecordMaxBufferedTime property of the KPL to "גˆ’1" to disable the buffering on the sensor side. Use Kinesis Data Analytics to enrich the data based on a company-developed anomaly detection SQL script. Push the enriched data to a fleet of Kinesis data streams and enable the data transformation feature to flatten the JSON file. Instantiate a dense storage Amazon Redshift cluster and use it as the destination for the Kinesis Data Firehose delivery stream.
  • B. Update the sensors code to use the PutRecord/PutRecords call from the Kinesis Data Streams API with the AWS SDK for Java. Use Kinesis Data Analytics to enrich the data based on a company-developed anomaly detection SQL script. Direct the output of KDA application to a Kinesis Data Firehose delivery stream, enable the data transformation feature to flatten the JSON file, and set the Kinesis Data Firehose destination to an Amazon OpenSearch Service (Amazon Elasticsearch Service) cluster. Most Voted
  • C. Set the RecordMaxBufferedTime property of the KPL to "0" to disable the buffering on the sensor side. Connect for each stream a dedicated Kinesis Data Firehose delivery stream and enable the data transformation feature to flatten the JSON file before sending it to an Amazon S3 bucket. Load the S3 data into an Amazon Redshift cluster.
  • D. Update the sensors code to use the PutRecord/PutRecords call from the Kinesis Data Streams API with the AWS SDK for Java. Use AWS Glue to fetch and process data from the stream using the Kinesis Client Library (KCL). Instantiate an Amazon Elasticsearch Service cluster and use AWS Lambda to directly push data into it.
