항공사에는 AWS Glue 데이터 카탈로그와 함께 Amazon S3에 .csv 형식의 데이터가 저장되어 있습니다.
데이터 분석가는 Dally 배치 프로세스의 일부로 Amazon Redshift 에 저장된 콜 센터 데이터와 이 데이터를 결합하려고 합니다 . Amazon Redshift 클러스터는 이미 과부하 상태입니다. 솔루션은 관리형이고 서버리스이며 제대로 작동해야 하며 기존 Amazon Redshift 클러스터의 로드를 최소화해야 합니다. 또한 솔루션에는 최소한의 노력과 개발 활동이 필요합니다.
이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. AWS Lambda 함수를 사용하여 Amazon Redshift에서 Amazon S3로 콜센터 데이터를 언로드합니다. AWS Glue ETL 스크립트를 사용하여 조인을 수행합니다. 최다 투표
  • B. AWS Glue의 Python 셸을 사용하여 Amazon Redshift에서 콜센터 데이터를 내보냅니다. AWS Glue ETL 스크립트를 사용하여 조인을 수행합니다.
  • C. 콜센터 데이터용으로 Amazon Redshift Spectrum을 사용하여 외부 테이블을 생성하고 Amazon Redshift와 조인을 수행합니다. 최다 투표
  • D. Apache Sqoop을 사용하여 Amazon Redshift에서 Amazon EMR로 콜센터 데이터를 내보냅니다. Apache Hive를 사용하여 조인을 수행합니다.

Topic #: 1

An airline has .csv-formatted data stored in Amazon S3 with an AWS Glue Data Catalog. Data analysts want to join this data with call center data stored in
Amazon Redshift as part of a dally batch process. The Amazon Redshift cluster is already under a heavy load. The solution must be managed, serverless, well- functioning, and minimize the load on the existing Amazon Redshift cluster. The solution should also require minimal effort and development activity.
Which solution meets these requirements?

  • A. Unload the call center data from Amazon Redshift to Amazon S3 using an AWS Lambda function. Perform the join with AWS Glue ETL scripts. Most Voted
  • B. Export the call center data from Amazon Redshift using a Python shell in AWS Glue. Perform the join with AWS Glue ETL scripts.
  • C. Create an external table using Amazon Redshift Spectrum for the call center data and perform the join with Amazon Redshift. Most Voted
  • D. Export the call center data from Amazon Redshift to Amazon EMR using Apache Sqoop. Perform the join with Apache Hive.

I think is c correct. https://aws.amazon.com/pt/blogs/big-data/10-best-practices-for-amazon-redshift-spectrum/ Amazon Redshift Spectrum is a sophisticated serverless compute service. The native Amazon Redshift cluster makes the invocation to Amazon Redshift Spectrum when the SQL query requests data from an external table stored in Amazon S3. You can handle multiple requests in parallel by using Amazon Redshift Spectrum on external tables to scan, filter, aggregate, and return rows from Amazon S3 into the Amazon Redshift cluster. All these operations are performed outside of Amazon Redshift, which reduces the computational load on the Amazon Redshift cluster and improves concurrency. In addition, Amazon Redshift Spectrum scales intelligently. Based on the demands of your queries, Amazon Redshift Spectrum can potentially use thousands of instances to take advantage of massively parallel processing (MPP).
