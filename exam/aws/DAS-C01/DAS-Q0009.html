Amazon의 AWS 공인 데이터 분석 – 전문 분야 시험 문제
질문 #: 9
주제 #: 1

한 제조 회사는 1년 동안 공장 현장의 장치에서 IoT 센서 데이터를 수집해 왔으며 매일 분석을 위해 Amazon Redshift에 데이터를 저장하고 있습니다. 데이터 분석가는 하루 약 2TB의 예상 수집 속도에서 4개월 이내에 클러스터 크기가 축소될 것이라고 판단했습니다. 장기적인 해결책이 필요합니다. 데이터 분석가는 대부분의 쿼리가 가장 최근 13개월의 데이터만 참조하지만 지난 7년 동안 생성된 모든 데이터를 쿼리해야 하는 분기별 보고서도 있다고 밝혔습니다. CTO(최고 기술 책임자)는 장기적인 솔루션의 비용, 관리 노력 및 성능에 대해 우려하고 있습니다.
데이터 분석가는 이러한 요구 사항을 충족하기 위해 어떤 솔루션을 사용해야 합니까?

  • A. AWS Glue에서 일일 작업을 생성하여 13개월이 지난 레코드를 Amazon S3에 UNLOAD하고 Amazon Redshift에서 해당 레코드를 삭제합니다. Amazon Redshift에서 S3 위치를 가리키는 외부 테이블을 생성합니다. Amazon Redshift Spectrum을 사용하여 13개월이 지난 데이터에 조인합니다. 최다 투표
  • B. Amazon Redshift 클러스터의 스냅샷을 찍습니다. 추가 스토리지 용량이 있는 고밀도 스토리지 노드를 사용하여 클러스터를 새 클러스터로 복원합니다.
  • C. CREATE TABLE AS SELECT(CTAS) 문을 실행하여 13개월이 지난 레코드를 Amazon S3가 지원하는 Amazon Redshift Spectrum의 분기별로 분할된 데이터로 이동합니다.
  • D. S3 Intelligent-Tiering을 사용하여 Amazon Redshift의 모든 테이블을 Amazon S3 버킷으로 언로드합니다. AWS Glue를 사용하여 S3 버킷 위치를 크롤링하여 AWS Glue 데이터 카탈로그에 외부 테이블을 생성합니다. 일일 분석 요구 사항에 대해 Auto Scaling을 사용하여 Amazon EMR 클러스터를 생성하고, 분기별 보고서에 Amazon Athena를 사용합니다. 두 가지 모두 동일한 AWS Glue 데이터 카탈로그를 사용합니다.

Exam question from Amazon's AWS Certified Data Analytics - Specialty
Question #: 9
Topic #: 1

A manufacturing company has been collecting IoT sensor data from devices on its factory floor for a year and is storing the data in Amazon Redshift for daily analysis. A data analyst has determined that, at an expected ingestion rate of about 2 TB per day, the cluster will be undersized in less than 4 months. A long-term solution is needed. The data analyst has indicated that most queries only reference the most recent 13 months of data, yet there are also quarterly reports that need to query all the data generated from the past 7 years. The chief technology officer (CTO) is concerned about the costs, administrative effort, and performance of a long-term solution.
Which solution should the data analyst use to meet these requirements?

  • A. Create a daily job in AWS Glue to UNLOAD records older than 13 months to Amazon S3 and delete those records from Amazon Redshift. Create an external table in Amazon Redshift to point to the S3 location. Use Amazon Redshift Spectrum to join to data that is older than 13 months. Most Voted
  • B. Take a snapshot of the Amazon Redshift cluster. Restore the cluster to a new cluster using dense storage nodes with additional storage capacity.
  • C. Execute a CREATE TABLE AS SELECT (CTAS) statement to move records that are older than 13 months to quarterly partitioned data in Amazon Redshift Spectrum backed by Amazon S3.
  • D. Unload all the tables in Amazon Redshift to an Amazon S3 bucket using S3 Intelligent-Tiering. Use AWS Glue to crawl the S3 bucket location to create external tables in an AWS Glue Data Catalog. Create an Amazon EMR cluster using Auto Scaling for any daily analytics needs, and use Amazon Athena for the quarterly reports, with both using the same AWS Glue Data Catalog.
