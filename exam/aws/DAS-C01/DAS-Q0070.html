Amazon의 AWS 공인 데이터 분석 – 전문 분야 시험 문제
질문 #: 70
주제 #: 1

한 회사는 거의 실시간으로 애플리케이션 로그를 보강하고 추가 분석을 위해 보강된 데이터 세트를 사용하려고 합니다. 애플리케이션은 여러 가용 영역의 Amazon EC2 인스턴스에서 실행되고 Amazon CloudWatch Logs를 사용하여 로그를 저장합니다. 강화 소스는 Amazon DynamoDB 테이블에 저장됩니다.
이벤트 수집 및 강화에 대한 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. CloudWatch Logs 구독을 사용하여 Amazon Kinesis Data Firehose로 데이터를 보냅니다. AWS Lambda를 사용하여 Kinesis Data Firehose 전송 스트림의 데이터를 변환하고 DynamoDB 테이블의 데이터로 이를 강화합니다. Amazon S3를 Kinesis Data Firehose 전송 대상으로 구성합니다. 최다 투표
  • B. AWS CLI를 사용하여 매시간 원시 로그를 Amazon S3로 내보냅니다. AWS Glue 크롤러를 사용하여 로그를 분류합니다. DynamoDB 테이블에 대한 AWS Glue 연결을 설정하고 AWS Glue ETL 작업을 설정하여 데이터를 보강합니다. 강화된 데이터를 Amazon S3에 저장합니다.
  • C. 로그를 로컬로 작성하고 Amazon Kinesis 에이전트를 사용하여 데이터를 Amazon Kinesis Data Streams로 보내도록 애플리케이션을 구성합니다. Kinesis 데이터 스트림을 소스로 사용하여 Kinesis Data Analytics SQL 애플리케이션을 구성합니다. SQL 애플리케이션 입력 스트림을 DynamoDB 레코드와 조인한 다음 Amazon Kinesis Data Firehose를 사용하여 강화된 출력 스트림을 Amazon S3에 저장합니다.
  • D. AWS CLI를 사용하여 시간 단위로 원시 로그를 Amazon S3로 내보냅니다. Amazon EMR에서 Apache Spark SQL을 사용하여 Amazon S3에서 로그를 읽고 DynamoDB의 데이터로 레코드를 보강합니다. 강화된 데이터를 Amazon S3에 저장합니다.

Exam question from Amazon's AWS Certified Data Analytics - Specialty
Question #: 70
Topic #: 1

A company wants to enrich application logs in near-real-time and use the enriched dataset for further analysis. The application is running on Amazon EC2 instances across multiple Availability Zones and storing its logs using Amazon CloudWatch Logs. The enrichment source is stored in an Amazon DynamoDB table.
Which solution meets the requirements for the event collection and enrichment?

  • A. Use a CloudWatch Logs subscription to send the data to Amazon Kinesis Data Firehose. Use AWS Lambda to transform the data in the Kinesis Data Firehose delivery stream and enrich it with the data in the DynamoDB table. Configure Amazon S3 as the Kinesis Data Firehose delivery destination. Most Voted
  • B. Export the raw logs to Amazon S3 on an hourly basis using the AWS CLI. Use AWS Glue crawlers to catalog the logs. Set up an AWS Glue connection for the DynamoDB table and set up an AWS Glue ETL job to enrich the data. Store the enriched data in Amazon S3.
  • C. Configure the application to write the logs locally and use Amazon Kinesis Agent to send the data to Amazon Kinesis Data Streams. Configure a Kinesis Data Analytics SQL application with the Kinesis data stream as the source. Join the SQL application input stream with DynamoDB records, and then store the enriched output stream in Amazon S3 using Amazon Kinesis Data Firehose.
  • D. Export the raw logs to Amazon S3 on an hourly basis using the AWS CLI. Use Apache Spark SQL on Amazon EMR to read the logs from Amazon S3 and enrich the records with the data from DynamoDB. Store the enriched data in Amazon S3.
