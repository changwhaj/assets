주제 #: 1

한 소매 회사는 Amazon S3 버킷을 사용하여 전자상거래 데이터 레이크를 호스팅하고 있습니다. 회사는 AWS Lake Formation을 사용하여 데이터 레이크를 관리하고 있습니다.
데이터 분석 전문가는 새로운 비즈니스 분석가 팀에 대한 액세스를 제공해야 합니다. 팀은 AWS Management Console에서 Amazon Athena를 사용하여 전자상거래 데이터베이스의 기존 web_sales 및 고객 테이블에서 데이터를 쿼리합니다. 팀에는 읽기 전용 액세스 권한과 성과 이름을 사용하여 고객을 고유하게 식별할 수 있는 기능이 필요합니다. 그러나 팀은 다른 개인 식별 데이터를 볼 수 없어야 합니다. 테이블 구조는 다음과 같습니다.

데이터 분석 전문가가 최소 권한 원칙을 사용하여 필요한 권한을 제공하기 위해 수행해야 하는 단계 조합은 무엇입니까? (3개를 선택하세요.)

  • A. AWS Lake Formation에서 business_analyst 그룹에 web_sales 테이블에 대한 SELECT 및 ALTER 권한을 부여하십시오.
  • B. AWS Lake Formation에서 business_analyst 그룹에 web_sales 테이블에 대한 SELECT 권한을 부여합니다. 최다 투표
  • C. AWS Lake Formation에서 business_analyst 그룹에 고객 테이블에 대한 SELECT 권한을 부여합니다. 열 아래에서 fisrt_name, last_name 및 customer_id 열이 있는 필터 유형 "열 포함"을 선택합니다. 최다 투표
  • D. AWS Lake Formation에서 business_analyst 그룹에 고객 테이블에 대한 SELECT 및 ALTER 권한을 부여합니다. 열 아래에서 fisrt_name 및 last_name 열과 함께 필터 유형 "열 포함"을 선택합니다.
  • E. business_analyst IAM 그룹 아래에 사용자를 생성합니다. Lakeformation:GetDataAccess 작업, athena:* 작업 및 Glue:Get* 작업을 허용하는 정책을 생성합니다.
  • F. business_analyst IAM 그룹 아래에 사용자를 생성합니다. Lakeformation:GetDataAccess 작업, athena:* 작업 및 Glue:Get* 작업을 허용하는 정책을 생성합니다. 또한 Athena 쿼리 결과 S3 버킷에 대해 s3:GetObject 작업, s3:PutObject 작업 및 s3:GetBucketLocation 작업을 허용합니다. 최다 투표

Selected Answer: BCF
A - WRONG, ALTER grants write access to raw data. B - CORRECT, SELECT grants read access to the transaction data, where no customer information is present. C - CORRECT, Lake Formation allows specifying which columns are accessible. customer_id is needed in order to join with web_sales table. D - WRONG, ALTER grants write access to raw data. E - WRONG, s3:GetObject is needed to see Athena results ( https://docs.aws.amazon.com/athena/latest/ug/querying.html ). F - CORRECT, even though it's Athena that puts the objects in the bucket and not the IAM user itself, granting s3:PutObject on the Athena bucket doesn't provide for the original S3 bucket where raw data resides.
