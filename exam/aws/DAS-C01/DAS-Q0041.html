이 작업을 효율적이고 안전하게 수행하기 위해 데이터 분석가는 어떤 단계를 수행해야 합니까?

  • A. DynamoDB 스트림을 처리하는 AWS Lambda 함수를 생성합니다. 동일한 KMS 키를 사용하여 중요한 데이터의 암호를 해독합니다. 재무팀을 위해 제한된 S3 버킷에 출력을 저장합니다. 재무팀만 액세스할 수 있는 Amazon Redshift 재무 테이블을 생성합니다. COPY 명령을 사용하여 Amazon S3의 데이터를 재무 테이블에 로드합니다. 최다 투표
  • B. DynamoDB 스트림을 처리하기 위한 AWS Lambda 함수를 생성합니다. 재무팀을 위해 제한된 S3 버킷에 출력을 저장합니다. 재무팀만 액세스할 수 있는 Amazon Redshift 재무 테이블을 생성합니다. KMS 키에 액세스할 수 있는 IAM 역할과 함께 COPY 명령을 사용하여 S3에서 재무 테이블로 데이터를 로드합니다. 최다 투표
  • C. KMS 키에 액세스할 수 있는 EMR_EC2_DefaultRole 역할을 사용하여 Amazon EMR 클러스터를 생성합니다. DynamoDB에 저장된 데이터와 Amazon Redshift의 재무 테이블을 참조하는 Apache Hive 테이블을 생성합니다. Hive에서 DynamoDB의 데이터를 선택한 다음 Amazon Redshift의 재무 테이블에 출력을 삽입합니다.
  • D. Amazon EMR 클러스터를 생성합니다. DynamoDB에 저장된 데이터를 참조하는 Apache Hive 테이블을 생성합니다. 재무팀을 위해 제한된 Amazon S3 버킷에 출력을 삽입합니다. KMS 키에 액세스할 수 있는 IAM 역할과 함께 COPY 명령을 사용하여 Amazon S3의 데이터를 Amazon Redshift의 금융 테이블에 로드합니다.

Exam question from Amazon's AWS Certified Data Analytics - Specialty
Question #: 41
Topic #: 1

A mortgage company has a microservice for accepting payments. This microservice uses the Amazon DynamoDB encryption client with AWS KMS managed keys to encrypt the sensitive data before writing the data to DynamoDB. The finance team should be able to load this data into Amazon Redshift and aggregate the values within the sensitive fields. The Amazon Redshift cluster is shared with other data analysts from different business units.
Which steps should a data analyst take to accomplish this task efficiently and securely?

  • A. Create an AWS Lambda function to process the DynamoDB stream. Decrypt the sensitive data using the same KMS key. Save the output to a restricted S3 bucket for the finance team. Create a finance table in Amazon Redshift that is accessible to the finance team only. Use the COPY command to load the data from Amazon S3 to the finance table. Most Voted
  • B. Create an AWS Lambda function to process the DynamoDB stream. Save the output to a restricted S3 bucket for the finance team. Create a finance table in Amazon Redshift that is accessible to the finance team only. Use the COPY command with the IAM role that has access to the KMS key to load the data from S3 to the finance table. Most Voted
  • C. Create an Amazon EMR cluster with an EMR_EC2_DefaultRole role that has access to the KMS key. Create Apache Hive tables that reference the data stored in DynamoDB and the finance table in Amazon Redshift. In Hive, select the data from DynamoDB and then insert the output to the finance table in Amazon Redshift.
  • D. Create an Amazon EMR cluster. Create Apache Hive tables that reference the data stored in DynamoDB. Insert the output to the restricted Amazon S3 bucket for the finance team. Use the COPY command with the IAM role that has access to the KMS key to load the data from Amazon S3 to the finance table in Amazon Redshift.
