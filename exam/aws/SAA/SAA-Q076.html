A bicycle sharing company is developing a multi-tier architecture to track the location of its bicycles during peak operating hours. The company wants to use these data points in its existing analytics platform. A solutions architect must determine the most viable multi-tier option to support this architecture. The data points must be accessible from the REST API.
Which action meets these requirements for storing and retrieving location data?

  • A. Use Amazon Athena with Amazon S3.
  • B. Use Amazon API Gateway with AWS Lambda.
  • C. Use Amazon QuickSight with Amazon Redshift.
  • D. Use Amazon API Gateway with Amazon Kinesis Data Analytics.
Show Suggested Answer Hide Answer

Eliminating the invalid options A. Amazon Athena with Amazon S3 - AWS Athena is a S3 query service and hence it is irrelavant. S3 can be used to collect data and existing data analytics platform can use it theoritcally. However it will be very inefficient and costly. Consider two factor 1. the bicycles will be sending a lot of data, possibly tiny 'byte' sized data 2. S3 put operation incurs cost. S3 put cost is only $0.005 per 1k 'put' requests but just consider 1000 bicycles, each making only 100 request per day can add upto $500/day. This problem is addressed by batching byte size data into big large data chunks. Kinesis Firehose is a solution provided by Amazon to specifically address this problem. B. API Gateway with lambda - Irrelavant. API gateway can provide an endpoint (a http url) to interact with lambda function. Lamda function is compute resource. Obiviously data is needed to be persisted to be further used by existing analytics system. To bring it any close to working solution lambda should write data to low cost storage, preferable to a EBS via app running on EC2. (note: Lamda cost only $0.20 per 1M request)
