A solutions architect is helping a developer design a new ecommerce shopping cart application using AWS services. The developer is unsure of the current database schema and expects to make changes as the ecommerce site grows. The solution needs to be highly resilient and capable of automatically scaling read and write capacity.
Which database solution meets these requirements?

  • A. Amazon Aurora PostgreSQL
  • B. Amazon DynamoDB with on-demand enabled
  • C. Amazon DynamoDB with DynamoDB Streams enabled
  • D. Amazon SQS and Amazon Aurora PostgreSQL

Rereading the question after some time, I changed my mind. --------- Two key concepts for NoSQL design. Designing a NoSQL requires a different approach than designing an RDBMS. For an RDBMS you can create a normalized data model without thinking about access models. You can extend it later when new questions and query requirements arise. You can organize each type of data in its table. How NoSQL design is different You don't have to start designing your own schema for DynamoDB until you know what questions it will answer. It is essential to understand business problems and application use cases early on. ------------ The question says that the developer is unsure about the scheme to adopt, so as per best practices you shouldn't use NoSql so you shouldn't use DynamoDb. Consequently, the correct answer is A. Does anyone agree?
