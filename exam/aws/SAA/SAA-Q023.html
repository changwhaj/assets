  • B. Launch a new EC2 instance from an Amazon Machine Image (AMI) in a new Region.

  • D. Copy an Amazon Machine Image (AMI) of an EC2 instance and specify a different Region for the destination.
  • E. Copy an Amazon Elastic Block Store (Amazon EBS) volume from Amazon S3 and launch an EC2 instance in the destination Region using that EBS volume.

FYI most of these answers are incorrect. B and D is the correct answer. Ami's are region specific and need to be copied and then reproduced in another region. Copying an AMI will automatically save a snapshot of whatever volume is available, so there is no need to directly save any volumes directly to S3. All is needed for DR is go copy the AMI to a new region, then launch a new EC2 instance using the copied AMI. All of the same resources will be provisioned from the original EC2 instance.
