회사 경영진은 모든 사용자가 자신의 직무 역할 및 기능에 따라 권한을 받기를 원합니다.
최소한의 운영 오버헤드로 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. AWS Single Sign-On 배포를 생성합니다. 온프레미스 Active Directory에 연결하여 회사 전체의 사용자 및 권한을 중앙에서 관리하세요. 최다 투표
  • B. 각 직무에 대한 IAM 역할을 생성합니다. 각 직원이 자신의 직무를 수행하려면 AWS Management Console에서 sts:AssumeRole 작업을 호출해야 합니다.
  • C. 각 직원에 대한 개별 IAM 사용자 계정을 생성합니다. 각 직무에 대한 IAM 정책을 생성하고 해당 직무에 따라 모든 IAM 사용자에게 정책을 연결합니다.
  • D. 각 직원에 대한 개별 IAM 사용자 계정을 생성합니다. 각 직무에 대한 IAM 정책을 생성합니다. IAM 그룹을 생성하고 각 그룹에 관련 정책을 연결합니다. IAM 사용자를 해당 직무 역할에 따라 그룹에 할당합니다. 최다 투표

Topic #: 1

A company wants to enforce strict security guidelines on accessing AWS Cloud resources as the company migrates production workloads from its data centers.
Company management wants all users to receive permissions according to their job roles and functions.
Which solution meets these requirements with the LEAST operational overhead?

  • A. Create an AWS Single Sign-On deployment. Connect to the on-premises Active Directory to centrally manage users and permissions across the company. Most Voted
  • B. Create an IAM role for each job function. Require each employee to call the sts:AssumeRole action in the AWS Management Console to perform their job role.
  • C. Create individual IAM user accounts for each employee. Create an IAM policy for each job function, and attach the policy to all IAM users based on their job role.
  • D. Create individual IAM user accounts for each employee. Create IAM policies for each job function. Create IAM groups, and attach associated policies to each group. Assign the IAM users to a group based on their job role. Most Voted
Show Suggested Answer Hide Answer
