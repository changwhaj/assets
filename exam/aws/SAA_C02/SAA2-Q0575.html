회사는 하나의 AWS 지역에서 엔터프라이즈 콘텐츠 관리 플랫폼을 호스팅하지만 여러 지역에서 플랫폼을 운영해야 합니다. 이 회사에는 마이크로서비스를 실행하는 Amazon Elastic Kubernetes Service(Amazon EKS) 클러스터가 있습니다. EKS 클러스터는 Amazon S3에서 객체를 저장하고 검색합니다. EKS 클러스터는 또한 Amazon DynamoDB에서 메타데이터를 저장하고 검색합니다.
여러 지역에 플랫폼을 배포하려면 솔루션 아키텍트가 수행해야 하는 단계 조합은 무엇입니까? (2개를 선택하세요.)

  • A. 교차 지역 복제를 사용하여 EKS 클러스터를 복제합니다.
  • B. Amazon API Gateway를 사용하여 EKS 클러스터에 대한 글로벌 엔드포인트를 생성합니다.
  • C. AWS Global Accelerator 엔드포인트를 사용하여 트래픽을 여러 지역에 분산시킵니다. 최다 투표
  • D. Amazon S3 액세스 포인트를 사용하여 여러 지역의 객체에 대한 액세스 권한을 부여합니다. DynamoDB Accelerator(DAX)를 구성합니다. DAX를 관련 테이블에 연결합니다.
  • E. EKS 클러스터와 S3 버킷을 다른 지역에 배포합니다. 두 S3 버킷 모두에서 리전 간 복제를 구성합니다. DynamoDB에 대한 전역 테이블을 활성화합니다. 최다 투표

Topic #: 1

A company hosts its enterprise content management platform in one AWS Region but needs to operate the platform across multiple Regions. The company has an Amazon Elastic Kubernetes Service (Amazon EKS) cluster that runs its microservices. The EKS cluster stores and retrieves objects from Amazon S3. The EKS cluster also stores and retrieves metadata from Amazon DynamoDB.
Which combination of steps should a solutions architect take to deploy the platform across multiple Regions? (Choose two.)

  • A. Replicate the EKS cluster with cross-Region replication.
  • B. Use Amazon API Gateway to create a global endpoint to the EKS cluster.
  • C. Use AWS Global Accelerator endpoints to distribute the traffic to multiple Regions. Most Voted
  • D. Use Amazon S3 access points to give access to the objects across multiple Regions. Configure DynamoDB Accelerator (DAX). Connect DAX to the relevant tables.
  • E. Deploy an EKS cluster and an S3 bucket in another Region. Configure cross-Region replication on both S3 buckets. Turn on global tables for DynamoDB. Most Voted
