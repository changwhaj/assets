Which solution will meet these requirements?

  • B. Configure a placement group for EC2 instances that have the same instance type. Most Voted

  • D. Configure AWS PrivateLink for the EC2 instances.

A and B are both correct. ============ Placement groups: -cluster(EC2s on same rack/server) -spread(each Ec2 on own rack, can do multi AZ) -partition(concept of partitions, each partition on diff rack, multiple ec2 can belong to same rack//can do multi AZ) ============ Aim is to use same underlying server ie control the EC2 placement in Dedicated host (to reduce latency). 'Dedicated Hosts' allow us to rent entire server and control placement of individual EC2 instances.We also have visibility into cores/sockets(and can make use of server licensing tied to sockets). Contrast this with 'Dedicated Instances' where we still rent complete server but cannot control placement of Ec2 instances.
