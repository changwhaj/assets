회사는 부서 수준 VPC의 AWS에서 웹 기반 애플리케이션을 호스팅합니다. 이 회사는 AWS CloudFormation 스택을 사용하여 AWS 클라우드에 필요한 모든 리소스를 프로비저닝했습니다. 회사는 이 애플리케이션을 회사 수준 VPC로 마이그레이션해야 합니다. 애플리케이션과 네트워크 구성요소는 별도의 엔터티로 관리되어야 합니다. 회사에서는 변경 사항을 구현하기 전에 변경 사항이 이미 실행 중인 리소스에 어떤 영향을 미치는지 이해하려고 합니다.
이러한 요구 사항을 충족하기 위해 솔루션 설계자는 어떤 조치 조합을 취해야 합니까? (2개를 선택하세요.)

  • A. CloudFormation 스택에서 드리프트 감지를 수행하여 변경 사항을 추적합니다.
  • B. AWS Config를 활성화하여 CloudFormation 스택의 변경 사항을 추적합니다.
  • C. CloudFormation 스택을 업데이트하기 전에 변경 세트를 생성합니다. 최다 투표
  • D. 두 개의 중첩 스택(애플리케이션 계층용 스택 하나와 네트워크 계층용 스택 하나)이 있는 CloudFormation 루트 스택을 생성합니다.
  • E. CloudFormation 교차 스택 참조를 생성합니다. 회사 수준 VPC 스택 출력을 내보냅니다. 스택 출력을 애플리케이션 스택으로 가져옵니다. 최다 투표

Topic #: 1

A company hosts a web-based application on AWS in a departmental level VPC. The company has used AWS CloudFormation stacks to provision all the required resources in the AWS Cloud. The company needs to migrate this application to the company level VPC. The application and network components need to be managed as separate entities. Before the company implements changes, the company wants to understand how the changes will affect resources that are already running.
Which combination of actions should a solutions architect take to meet these requirements? (Choose two.)

  • A. Perform drift detection on the CloudFormation stacks to track the changes.
  • B. Enable AWS Config to track the changes to the CloudFormation stacks.
  • C. Create change sets before updating the CloudFormation stacks. Most Voted
  • D. Create a CloudFormation root stack that has two nested stacks: one stack for the application layer and one stack for the network layer.
  • E. Create a CloudFormation cross-stack reference. Export company level VPC stack outputs. Import the stack outputs into the application stack. Most Voted
