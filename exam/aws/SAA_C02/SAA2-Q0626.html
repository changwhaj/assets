주제 #: 1

회사에는 동일한 AWS 리전에 두 개의 AWS 계정이 있습니다. 한 계정은 게시자 계정이고 다른 계정은 구독자 계정입니다. 각 계정에는 자체 Amazon S3 버킷이 있습니다.
애플리케이션은 미디어 객체를 게시자 계정의 S3 버킷에 넣습니다. 객체는 고객 제공 암호화 키(SSE-C)를 사용하는 서버 측 암호화로 암호화됩니다. 회사에는 객체를 구독자 계정의 S3 버킷에 자동으로 복사하는 솔루션이 필요합니다.
최소한의 운영 오버헤드로 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. 게시자 계정의 S3 버킷에서 S3 버전 관리를 활성화합니다. 구독자 계정의 S3 버킷에 대한 객체의 S3 동일 리전 복제를 구성합니다. 최다 투표
  • B. 게시자 계정의 S3 버킷에 객체가 게시될 때 호출되는 AWS Lambda 함수를 생성합니다. 객체를 구독자 계정의 S3 버킷에 복사하도록 Lambda 함수를 구성합니다.
  • C. 게시자 계정의 S3 버킷에 객체가 게시될 때 AWS Lambda 함수를 호출하도록 Amazon EventBridge(Amazon CloudWatch Events)를 구성합니다. 객체를 구독자 계정의 S3 버킷에 복사하도록 Lambda 함수를 구성합니다.
  • D. 게시자 계정의 S3 버킷에 객체가 게시될 때 Amazon Simple 알림 서비스(Amazon SNS) 알림을 게시하도록 Amazon EventBridge(Amazon CloudWatch Events)를 구성합니다. 알림이 수신되면 S3 콘솔을 사용하여 객체를 구독자 계정의 S3 버킷에 복사합니다.

A company has two AWS accounts in the same AWS Region. One account is a publisher account, and the other account is a subscriber account. Each account has its own Amazon S3 bucket.
An application puts media objects into the publisher account's S3 bucket. The objects are encrypted with server-side encryption with customer-provided encryption keys (SSE-C). The company needs a solution that will automatically copy the objects to the subscriber's account's S3 bucket.
Which solution will meet these requirements with the LEAST operational overhead?

  • A. Enable S3 Versioning on the publisher account's S3 bucket. Configure S3 Same-Region Replication of the objects to the subscriber account's S3 bucket. Most Voted
  • B. Create an AWS Lambda function that is invoked when objects are published in the publisher account's S3 bucket. Configure the Lambda function to copy the objects to the subscriber account's S3 bucket.
  • C. Configure Amazon EventBridge (Amazon CloudWatch Events) to invoke an AWS Lambda function when objects are published in the publisher account's S3 bucket. Configure the Lambda function to copy the objects to the subscriber account's S3 bucket.
  • D. Configure Amazon EventBridge (Amazon CloudWatch Events) to publish Amazon Simple Notification Service (Amazon SNS) notifications when objects are published in the publisher account's S3 bucket. When notifications are received, use the S3 console to copy the objects to the subscriber account's S3 bucket.

Requirements for replication: Both source and destination buckets must have versioning enabled. Replication enables automatic, asynchronous copying of objects across Amazon S3 buckets. Buckets that are configured for object replication can be owned by the same AWS account or by different accounts. You can replicate objects to a single destination bucket or to multiple destination buckets. The destination buckets can be in different AWS Regions or within the same Region as the source bucket. To automatically replicate new objects as they are written to the bucket use live replication, such as Same-Region Replication (SRR) or Cross-Region Replication (CRR). https://docs.aws.amazon.com/AmazonS3/latest/userguide/replication.html
