A company's security policy requires that all AWS API activity in its AWS accounts be recorded for periodic auditing. The company needs to ensure that AWS
CloudTrail is enabled on all of its current and future AWS accounts using AWS Organizations.
Which solution is MOST secure?

  • A. At the organization's root, define and attach a service control policy (SCP) that permits enabling CloudTrail only.
  • B. Create IAM groups in the organization's management account as needed. Define and attach an IAM policy to the groups that prevents users from disabling CloudTrail.
  • C. Organize accounts into organizational units (OUs). At the organization's root, define and attach a service control policy (SCP) that prevents users from disabling CloudTrail. Most Voted
  • D. Add all existing accounts under the organization's root. Define and attach a service control policy (SCP) to every account that prevents users from disabling CloudTrail.

SCP = AWS Svc/리소스/API 호출에 대한 최대 액세스 권한에 대한 중앙 제어를 시행합니다. I select A over C and D. With option A, users are only allowed to enable CloudTrail (denying other actions, such as modifying or disabling CloudTrail). Option C will prevent disabling CloudTrail, but will not allow enabling it. Option D requires a lot of administration to attach an SCP to each account (a step that may be forgotten when new accounts are created).

SCP = To enforce central control over max permission to access to AWS Svc/Resource/API Call; Apply to member AWS Account Root Users/IAM Users/IAM Roles under the same AWS Organization at the AWS Organization level; Don't affect IAM Users/Roles/Root User in the management AWS Account, affect only member AWS Accounts only; Doesn't affect Service-Linked IAM Role, but only IAM Role assigned to IAM User/Group; SCP permission rules supercede all IAM Policy; SCP (eg DenyRedShift SCP) inherited from a parent OU supercedes the SCP of the child OU.

As a best practice, do not use the AWS account root user for any task where it's not required. Instead, create a new IAM user for each person that requires administrator access. Then make those users administrators by placing the users into an "Administrators" user group to which you attach the AdministratorAccess managed policy. https://docs.aws.amazon.com/IAM/latest/UserGuide/getting-started_create-admin-group.html
