주제 #: 1

회사는 온프레미스 SAN(Storage Area Network)에서 Amazon S3로 대량의 데이터를 마이그레이션해야 합니다. SAN에는 현재 200TB의 데이터가 있으며 매달 20TB의 데이터를 추가로 수신하고 있습니다. 회사의 인터넷 연결 속도는 500Mbps입니다.
회사는 가장 짧은 시간 내에 데이터를 Amazon S3로 마이그레이션하려면 어떻게 해야 합니까?

  • A. 파일 동기화 애플리케이션을 사용하여 퍼블릭 S3 엔드포인트를 통해 인터넷을 통해 Amazon S3에 데이터를 동기화합니다. SAN이 폐기될 때까지 변경된 데이터를 동일한 방식으로 동기화합니다.
  • B. AWS Snowball Edge Storage Optimized 디바이스를 사용하여 초기 200TB의 데이터를 Amazon S3로 마이그레이션합니다. SAN이 폐기될 때까지 AWS DataSync를 사용하여 변경 사항 데이터를 동기화합니다.
  • C. AWS Site-to-Site VPN 연결을 설정합니다. 파일 동기화 애플리케이션을 사용하여 프라이빗 S3 엔드포인트를 통해 Amazon S3에 데이터를 동기화합니다. SAN이 폐기될 때까지 변경된 데이터를 동일한 방식으로 동기화합니다.
  • D. 10Gbps AWS Direct Connection 연결을 설정합니다. 파일 동기화 애플리케이션을 사용하여 초기 200TB의 데이터를 Amazon S3로 마이그레이션합니다. SAN이 폐기될 때까지 변경된 데이터를 동일한 방식으로 동기화합니다. 최다 투표

Exam question from Amazon's AWS Certified Solutions Architect - Associate SAA-C02
Question #: 582
Topic #: 1

A company needs to migrate a large amount of data from an on-premises storage area network (SAN) to Amazon S3. The SAN currently has 200 TB of data and is receiving an additional 20 TB of data each month. The company has a 500 Mbps connection to the internet.
What should the company do to migrate the data to Amazon S3 in the LEAST amount of time?

  • A. Use a file syncing application to sync the data to Amazon S3 over the internet through a public S3 endpoint. Sync any changed data the same way until the SAN is decommissioned.
  • B. Use an AWS Snowball Edge Storage Optimized device to migrate the initial 200 TB of data to Amazon S3. Sync any changes data by using AWS DataSync until the SAN is decommissioned.
  • C. Set up an AWS Site-to-Site VPN connection. Use a file syncing application to sync the data to Amazon S3 through a private S3 endpoint. Sync any changed data the same way until the SAN is decommissioned.
  • D. Set up a 10 Gbps AWS Direct Connection connection. Migrate the initial 200 TB of data to Amazon S3 by using a file syncing application. Sync any changed data the same way until the SAN is decommissioned. Most Voted
