  • D. Place the EC2 instances in an EC2 Auto Scaling group. Most Voted
  • E. Run the EC2 instances in a partition placement group. Most Voted

https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/placement-groups.html Depending on the type of workload, you can create a placement group using **one** of the following placement strategies: Cluster (C)– not an option because it groups the nodes having the same underlying hardware. Partition (E)– spreads your instances across logical partitions such that groups of instances in one partition do not share the underlying hardware with groups of instances in different partitions. This means that the partition might share the underlying hardware. Spread (B)– strictly places a small group of instances across distinct underlying hardware to reduce correlated failures. --> THIS IS THE VALID OPTION IN TERMS OF "underlying hardware" constrain. Is one placement group or the other... we cannot choose both. I would go with D for scalability but I am not sure if a Spread Placement Group is "Extremely Scalable" since we only have 7 nodes per AZ.
