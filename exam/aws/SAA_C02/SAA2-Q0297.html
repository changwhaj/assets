  • D. Configure Amazon Elastic Block Store (Amazon EBS) to store the users' home directories. Configure AWS Single Sign-On with Active Directory.

대답은 B입니다. https://aws.amazon.com/fsx/windows/ Windows 파일 서버용 Amazon FSx는 업계 표준 SMB(서버 메시지 블록) 프로토콜을 통해 액세스할 수 있는 안정성과 확장성이 뛰어난 완전 관리형 파일 스토리지를 제공합니다. . Windows Server를 기반으로 구축되어 사용자 할당량, 최종 사용자 파일 복원 및 "Microsoft Active Directory(AD) 통합"과 같은 광범위한 관리 기능을 제공합니다. 단일 AZ 및 "다중 AZ 배포 옵션", 완전 관리형 백업, 저장 및 전송 중인 데이터 암호화를 제공합니다. https://aws.amazon.com/fsx/windows/ 사용 사례 중 하나: 홈 디렉터리 많은 조직에는 홈 디렉터리와 팀 공유가 있습니다. 이러한 조직에는 일반적으로 수백 또는 수천 명의 사용자가 공통 데이터 세트에 대한 공유 액세스가 필요합니다. Amazon FSx for Windows File Server를 사용하면 최대 수십만 명의 사용자가 액세스할 수 있는 파일 시스템을 생성하고 파일 또는 폴더 수준에서 권한을 설정할 수 있습니다. AWS DataSync를 사용하면 기존 홈 디렉터리 파일 공유를 Amazon FSx로 쉽게 이동할 수 있습니다.

Answer is B: https://aws.amazon.com/fsx/windows/ Amazon FSx for Windows File Server provides fully managed, highly reliable, and scalable file storage that is accessible over the industry-standard Server Message Block (SMB) protocol. It is built on Windows Server, delivering a wide range of administrative features such as user quotas, end-user file restore, and " Microsoft Active Directory (AD) integration ". It offers single-AZ and " multi-AZ deployment options", fully managed backups, and encryption of data at rest and in transit. https://aws.amazon.com/fsx/windows/ One of the use case: Home directories Many organizations have home directories and team shares. These organizations typically need shared access to common data sets with hundreds or thousands of users. You can use Amazon FSx for Windows File Server to create a file system that is accessible by up to hundreds of thousands of users, and establish permissions at the file or folder level. You can easily move existing home directory file shares to Amazon FSx using AWS DataSync.
