질문 #: 111
주제 #: 1

네트워크 엔지니어는 회사 사무실 위치와 AWS 계정 간에 듀얼 스택 연결을 제공해야 합니다. 회사의 온프레미스 라우터는 듀얼 스택 연결을 지원하며, VPC는 ​​듀얼 스택 지원으로 구성되었습니다. 회사는 사무실 위치에 대한 두 개의 AWS Direct Connect 연결을 설정했습니다. 이 연결은 가용성이 높아야 하며 대기 시간에 민감한 트래픽에 대해 안정적이어야 합니다.

어떤 솔루션이 이러한 요구 사항을 충족합니까? (2개를 선택하세요.)

  • A. 각 Direct Connect 연결에 단일 프라이빗 VIF를 구성합니다. 각 프라이빗 VIF에 IPv4 및 IPv6 피어링을 모두 추가합니다. IPv4 피어링의 IPv4 경로와 IPv6 피어링의 IPv6 경로를 알리기 위해 AWS가 제공하는 BGP 이웃으로 온프레미스 장비를 구성합니다. 모든 피어링 세션에서 BFD(양방향 전달 감지)를 활성화합니다. 최다 투표
  • B. 각 Direct Connect 연결에 두 개의 프라이빗 VIF를 구성합니다. 하나는 IPv4 주소 계열이 있는 프라이빗 VIF이고 다른 하나는 IPv6 주소 계열이 있는 프라이빗 VIF입니다. IPv4 피어링의 IPv4 경로와 IPv6 피어링의 IPv6 경로를 알리기 위해 AWS가 제공하는 BGP 이웃으로 온프레미스 장비를 구성합니다. 모든 피어링 세션에서 BFD(양방향 전달 감지)를 활성화합니다. 최다 투표
  • C. 각 Direct Connect 연결에서 단일 프라이빗 VIF 및 IPv4 피어링을 구성합니다. 동일한 BGP 인접 구성에서 IPv6 경로를 광고하려면 이 피어링을 사용하여 온프레미스 장비를 구성하세요. 모든 피어링 세션에서 BFD(양방향 전달 감지)를 활성화합니다.
  • D. 각 Direct Connect 연결에 두 개의 프라이빗 VIF를 구성합니다. 하나는 IPv4 주소 계열이 있는 프라이빗 VIF이고 다른 하나는 IPv6 주소 계열이 있는 프라이빗 VIF입니다. 모든 피어링 세션에서 모든 IPv4 경로와 IPv6 경로를 알리기 위해 AWS가 제공하는 BGP 이웃으로 온프레미스 장비를 구성합니다. BFD(양방향 전달 감지) 구성을 변경하지 않고 유지합니다.
  • E. 각 Direct Connect 연결에 두 개의 프라이빗 VIF를 구성합니다. 하나는 IPv4 주소 계열이 있는 프라이빗 VIF이고 다른 하나는 IPv6 주소 계열이 있는 프라이빗 VIF입니다. IPv4 피어링의 IPv4 경로와 IPv6 피어링의 IPv6 경로를 알리기 위해 AWS가 제공하는 BGP 이웃으로 온프레미스 장비를 구성합니다. 온프레미스 장비와 Direct Connect 구성 모두에서 BGP Hello 타이머를 5초로 줄입니다.

Exam question from Amazon's AWS Certified Advanced Networking - Specialty ANS-C01
Question #: 111
Topic #: 1

A network engineer needs to provide dual-stack connectivity between a company's office location and an AWS account. The company's on-premises router supports dual-stack connectivity, and the VPC has been configured with dual-stack support. The company has set up two AWS Direct Connect connections to the office location. This connectivity must be highly available and must be reliable for latency-sensitive traffic.

Which solutions will meet these requirements? (Choose two.)

  • A. Configure a single private VIF on each Direct Connect connection. Add both IPv4 and IPv6 peering to each private VIF. Configure the on- premises equipment with the AWS provided BGP neighbors to advertise IPv4 routes on the IPv4 peering and IPv6 routes on the IPv6 peering. Enable Bidirectional Forwarding Detection (BFD) on all peering sessions. Most Voted
  • B. Configure two private VIFs on each Direct Connect connection: one private VIF with the IPv4 address family and one private VIF with the IPv6 address family. Configure the on-premises equipment with the AWS provided BGP neighbors to advertise IPv4 routes on the IPv4 peering and IPv6 routes on the IPv6 peering. Enable Bidirectional Forwarding Detection (BFD) on all peering sessions. Most Voted
  • C. Configure a single private VIF and IPv4 peering on each Direct Connect connection. Configure the on-premises equipment with this peering to advertise the IPv6 routes in the same BGP neighbor configuration. Enable Bidirectional Forwarding Detection (BFD) on all peering sessions.
  • D. Configure two private VIFs on each Direct Connect connection: one private VIF with the IPv4 address family and one private VIF with the IPv6 address family. Configure the on-premises equipment with the AWS provided BGP neighbors to advertise all IPv4 routes and IPv6 routes on all peering sessions. Keep the Bidirectional Forwarding Detection (BFD) configuration unchanged.
  • E. Configure two private VIFs on each Direct Connect connection: one private VIF with the IPv4 address family and one private VIF with the IPv6 address family. Configure the on-premises equipment with the AWS provided BGP neighbors to advertise IPv4 routes on the IPv4 peering and IPv6 routes on the IPv6 peering. Reduce the BGP hello timer to 5 seconds on both the on-premises equipment and the Direct Connect configuration.
