주제 #: 1

회사의 AWS 아키텍처는 여러 개의 VPC로 구성됩니다. VPC에는 공유 서비스 VPC와 여러 애플리케이션 VPC가 포함됩니다. 회사는 모든 VPC에서 온프레미스 DNS 서버로의 네트워크 연결을 설정했습니다.

애플리케이션 VPC에 배포된 애플리케이션은 온프레미스에서 내부적으로 호스팅된 도메인에 대한 DNS를 확인할 수 있어야 합니다. 또한 애플리케이션은 로컬 VPC 도메인 이름과 Amazon Route 53 프라이빗 호스팅 영역에서 호스팅되는 도메인을 확인할 수 있어야 합니다.

이러한 요구 사항을 충족하려면 네트워크 엔지니어가 무엇을 해야 합니까?

  • A. 공유 서비스 VPC에 새로운 Route 53 Resolver 인바운드 엔드포인트를 생성합니다. 온프레미스 호스팅 도메인에 대한 전달 규칙을 만듭니다. 규칙을 새로운 확인자 엔드포인트 및 각 애플리케이션 VPC와 연결합니다. DNS 확인이 새 확인자 엔드포인트를 가리키도록 각 애플리케이션 VPC의 DHCP 구성을 업데이트합니다.
  • B. 공유 서비스 VPC에 새로운 Route 53 Resolver 아웃바운드 엔드포인트를 생성합니다. 온프레미스 호스팅 도메인에 대한 전달 규칙을 만듭니다. 규칙을 새로운 확인자 엔드포인트 및 각 애플리케이션 VPC와 연결합니다. 최다 투표
  • C. 온프레미스 호스팅 도메인에 대한 공유 서비스 VPCreate 전달 규칙에서 새로운 Route 53 Resolver 아웃바운드 엔드포인트를 생성합니다. 규칙을 새 확인자 엔드포인트와 연결하고 각 애플리케이션 VP업데이트 각 애플리케이션 VPC의 DHCP 구성을 업데이트하여 DNS 확인이 새 확인자 엔드포인트를 가리키도록 합니다.
  • D. 공유 서비스 VPC에 새로운 Route 53 Resolver 인바운드 엔드포인트를 생성합니다. 온프레미스 호스팅 도메인에 대한 전달 규칙을 만듭니다. 규칙을 새로운 확인자 엔드포인트 및 각 애플리케이션 VPC와 연결합니다.
