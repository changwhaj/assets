주제 #: 1

한 회사가 AWS 클라우드에 다중 VPC 환경을 배포했습니다. 회사는 전송 게이트웨이를 사용하여 모든 VPC를 함께 연결합니다. 과거에 회사는 VPC의 보안 그룹, 네트워크 ACL 및 라우팅 테이블을 변경한 후 애플리케이션 간 연결이 끊어지는 문제를 경험했습니다. 이러한 변경이 발생하면 회사는 단일 VPC의 서로 다른 리소스 간에 연결이 여전히 존재하는지 자동으로 확인하려고 합니다.

  • A. VPC 도달성 분석기에서 확인할 다양한 리소스 간의 경로 목록을 생성합니다. Amazon CloudWatch에 변경 사항이 발생하고 기록되는 시기를 모니터링하는 Amazon EventBridge 규칙을 생성합니다. 도달성 분석기의 다양한 경로를 테스트하기 위해 AWS Lambda 함수를 호출하도록 규칙을 구성합니다.
  • B. VPC 도달성 분석기에서 확인할 다양한 리소스 간의 경로 목록을 생성합니다. Amazon EventBridge 규칙을 생성하여 변경 사항이 발생하고 AWS에 기록되는 시기를 모니터링합니다. CloudTrail. 도달성 분석기의 다양한 경로를 테스트하기 위해 AWS Lambda 함수를 호출하도록 규칙을 구성합니다. 최다 투표
  • C. AWS Transit Gateway 네트워크 관리자 경로 분석기에서 확인할 경로 목록을 생성합니다. Amazon CloudWatch에 변경 사항이 발생하고 기록되는 시기를 모니터링하는 Amazon EventBridge 규칙을 생성합니다. Route analyzer에서 다른 경로를 테스트하기 위해 AWS Lambda 함수를 호출하도록 규칙을 구성합니다.
  • D. AWS Transit Gateway 네트워크 관리자 경로 분석기에서 확인할 경로 목록을 생성합니다. 변경 사항이 발생하고 AWS CloudTrail에 기록되는 시기를 모니터링하는 Amazon EventBridge 규칙을 생성합니다. Route Analyser의 다양한 경로를 테스트하기 위해 AWS Lambda 함수를 호출하도록 규칙을 구성합니다.

Exam question from Amazon's AWS Certified Advanced Networking - Specialty ANS-C01
Question #: 153
Topic #: 1

A company has deployed a multi-VPC environment in the AWS Cloud. The company uses a transit gateway to connect all the VPCs together. In the past, the company has experienced a loss of connectivity between applications after changes to security groups, network ACLs, and route tables in a VPC. When these changes occur, the company wants to automatically verify that connectivity still exists between different resources in a single VPC.

  • A. Create a list of paths between different resources to check in VPC Reachability Analyzer. Create an Amazon EventBridge rule to monitor when a change is made and logged in Amazon CloudWatch. Configure the rule to invoke an AWS Lambda function to test the different paths in Reachability Analyzer.
  • B. Create a list of paths between different resources to check in VPC Reachability Analyzer. Create an Amazon EventBridge rule to monitor when a change is made and logged in AWS. CloudTrail. Configure the rule to invoke an AWS Lambda function to test the different paths in Reachability Analyzer. Most Voted
  • C. Create a list of paths to check in AWS Transit Gateway Network Manager Route Analyzer. Create an Amazon EventBridge rule to monitor when a change is made and logged in Amazon CloudWatch. Configure the rule to invoke an AWS Lambda function to test the diffident paths in Route Analyzer.
  • D. Create a list of paths to check in AWS Transit Gateway Network Manager Route Analyzer. Create an Amazon EventBridge rule to monitor when a change is made and logged in AWS CloudTrail. Configure the rule to invoke an AWS Lambda function to test the different paths in Route Analyzer.

"When a security group change is made, the change event is logged in AWS CloudTrail. CloudTrail then forwards the change event to Amazon EventBridge, which evaluates the change against a series of rules to determine if any actions must be taken. Within EventBridge, a rule will be created to forward all security group change events from CloudTrail to an AWS Lambda function. The Lambda function is responsible for determining if any EC2 instances are impacted by the security group change, and if any Reachability Analyzer paths assessing the connectivity from the internet to the instance exist. " [https://aws.amazon.com/blogs/networking-and-content-delivery/automating-connectivity-assessments-with-vpc-reachability-analyzer/]
