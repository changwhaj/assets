Amazon AWS 공인 SysOps 관리자 – 어소시에이트 의 시험 문제
질문 #: 368
주제 #: 1

회사는 Application Load Balancer 뒤에서 Amazon EC2 인스턴스에서 애플리케이션을 실행합니다. EC2 인스턴스는 Auto Scaling 그룹에 있습니다. 응용프로그램이 느려지고 응답하지 않는 경우가 있습니다. Amazon CloudWatch 지표에 따르면 일부 EC2 인스턴스에 높은 CPU 로드가 발생하고 있는 것으로 나타났습니다.

SysOps 관리자는 모든 EC2 인스턴스의 CPU 지표를 자동으로 표시할 수 있는 CloudWatch 대시보드를 생성해야 합니다. 지표에는 Auto Scaling 그룹의 일부로 시작되는 새 인스턴스가 포함되어야 합니다.

SysOps 관리자는 가장 운영 효율적인 방식으로 이러한 요구 사항을 충족하기 위해 무엇을 해야 합니까?

  • A. CloudWatch 대시보드를 생성합니다. Auto Scaling 그룹의 활동 알림을 사용하여 사용자 지정 AWS Lambda 함수를 호출합니다. Lambda 함수를 사용하여 CloudWatch 대시보드를 업데이트하여 새 인스턴스 ID에 대한 CPUUtilization 지표를 모니터링합니다.
  • B. CloudWatch 대시보드를 생성합니다. 각 EC2 인스턴스에서 사용자 지정 스크립트를 실행하여 CPU 사용률을 대시보드로 스트리밍합니다.
  • C. CloudWatch 지표 탐색기를 사용하여 aws:autoscaling:groupName 태그로 필터링하고 CPUUtilization 지표에 대한 시각화를 생성합니다. CloudWatch 대시보드에 시각화를 추가합니다. 최다 투표
  • D. CloudWatch 지표 탐색기를 사용하여 인스턴스 상태별로 필터링하고 CPUUtilization 지표에 대한 시각화를 생성합니다. CloudWatch 대시보드에 시각화를 추가합니다.

Exam question from Amazon's AWS Certified SysOps Administrator - Associate
Question #: 368
Topic #: 1

A company runs an application on Amazon EC2 instances behind an Application Load Balancer. The EC2 instances are in an Auto Scaling group. The application sometimes becomes slow and unresponsive. Amazon CloudWatch metrics show that some EC2 instances are experiencing high CPU load.

A SysOps administrator needs to create a CloudWatch dashboard that can automatically display CPU metrics of all the EC2 instances. The metrics must include new instances that are launched as part of the Auto Scaling group.

What should the SysOps administrator do to meet these requirements in the MOST operationally efficient way?

  • A. Create a CloudWatch dashboard. Use activity notifications from the Auto Scaling group to invoke a custom AWS Lambda function. Use the Lambda function to update the CloudWatch dashboard to monitor the CPUUtilization metric for the new instance IDs.
  • B. Create a CloudWatch dashboard. Run a custom script on each EC2 instance to stream the CPU utilization to the dashboard.
  • C. Use CloudWatch metrics explorer to filter by the aws:autoscaling:groupName tag and to create a visualization for the CPUUtilization metric. Add the visualization to a CloudWatch dashboard. Most Voted
  • D. Use CloudWatch metrics explorer to filter by instance state and to create a visualization for the CPUUtilization metric. Add the visualization to a CloudWatch dashboard.
