Amazon AWS 공인 SysOps 관리자 – 어소시에이트 의 시험 문제
질문 #: 128
주제 #: 1

SysOps 관리자가 새 AWS 계정에 Amazon EC2 Auto Scaling 그룹을 생성하고 있습니다. 일부 인스턴스를 추가한 후 SysOps 관리자는 그룹이 최소 인스턴스 수에 도달하지 않았음을 확인합니다. SysOps 관리자는 다음과 같은 오류 메시지를 받습니다.

새 EC2 인스턴스를 시작하는 중입니다. 상태 이유: 귀하의 할당량은 0개의 추가 인스턴스 실행을 허용합니다.
최소 1개를 요청했습니다. EC2 인스턴스 시작에 실패했습니다.

이 문제를 해결하려면 어떤 조치를 취해야 합니까?

  • A. AWS Billing and Cost Management 콘솔에서 Amazon EC2에 대한 계정 지출 한도를 조정합니다.
  • B. EC2 콘솔의 EC2 설정 섹션에서 해당 AWS 지역에 대한 EC2 할당량을 수정합니다.
  • C. AWS Management Console에서 서비스 할당량을 사용하여 인스턴스 유형 패밀리에 대한 할당량 증가를 요청합니다. 최다 투표
  • D. AWS Management Console의 Auto Scaling 그룹에서 재조정 작업을 사용합니다.

Question #: 128
Topic #: 1

A SysOps administrator is creating an Amazon EC2 Auto Scaling group in a new AWS account. After adding some instances, the SysOps administrator notices that the group has not reached the minimum number of instances. The SysOps administrator receives the following error message:

Launching a new EC2 instance. Status Reason: Your quota allows for 0 more running instance(s).
You requested at least 1. Launching EC2 instance failed.

Which action will resolve this issue?

  • A. Adjust the account spending limits for Amazon EC2 on the AWS Billing and Cost Management console.
  • B. Modify the EC2 quota for that AWS Region in the EC2 Settings section of the EC2 console.
  • C. Request a quota increase for the instance type family by using Service Quotas on the AWS Management Console. Most Voted
  • D. Use the Rebalance action in the Auto Scaling group on the AWS Management Console.
