What should a SysOps administrator do to resolve this issue?

  • B. Enable sticky sessions (session affinity) for the target group of EC2 instances.
  • C. Redeploy the EC2 instances in a spread placement group.
  • D. Replace the ALB with a Network Load Balancer.

B. Enable sticky sessions (session affinity) for the target group of EC2 instances. Explanation: When users are frequently prompted to log in, it might indicate that their requests are being directed to different instances, leading to a loss of session state. Enabling sticky sessions (session affinity) ensures that requests from the same client are consistently directed to the same EC2 instance. This is important for applications that rely on maintaining session state.
