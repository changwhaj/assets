질문 #: 402
주제 #: 1

한 회사가 AWS에서 CI/CD(지속적 통합 및 지속적 전달) 환경을 호스팅하고 있습니다. CI/CD 환경에는 Amazon EC2 인스턴스에서 호스팅되는 Jenkins 서버가 포함되어 있습니다. 500GB 범용 SSD(gp2) Amazon Elastic Block Store(Amazon EBS) 볼륨이 EC2 인스턴스에 연결됩니다.

디스크 처리량 제한으로 인해 Jenkins 서버는 서버 빌드 속도를 저하시키는 성능 문제를 보고합니다. EBS 볼륨은 야간 빌드 작업을 수행하는 동안 3,000 IOPS를 유지해야 합니다.

SysOps 관리자는 Amazon CloudWatch에서 서버 기록을 검사합니다. BurstBalance 지표는 야간 빌드 중에 0 값을 가졌습니다. SysOps 관리자는 성능을 향상하고 지속적인 처리량 요구 사항을 충족해야 합니다.

이러한 요구 사항을 가장 비용 효율적으로 충족하는 솔루션은 무엇입니까?

  • A. gp2 EBS 볼륨 크기를 500GB에서 1,000GB로 두 배로 늘립니다.
  • B. 볼륨 유형을 gp2에서 범용 SSD(gp3)로 변경합니다.
  • C. 볼륨 유형을 gp2에서 처리량 최적화 HDD(st1)로 변경합니다.
  • D. 볼륨 유형을 gp2에서 프로비저닝된 IOPS SSD(io2)로 변경합니다.

Exam question from Amazon's AWS Certified SysOps Administrator - Associate
Question #: 402
Topic #: 1

A company hosts a continuous integration and continuous delivery (CI/CD) environment on AWS. The CI/CD environment includes a Jenkins server that is hosted on an Amazon EC2 instance. A 500 GB General Purpose SSD (gp2) Amazon Elastic Block Store (Amazon EBS) volume is attached to the EC2 instance.

Because of disk throughput limitations, the Jenkins server reports performance issues that are resulting in slower builds on the server. The EBS volume needs to sustain 3,000 IOPS while performing nightly build tasks.

A SysOps administrator examines the server's history in Amazon CloudWatch. The BurstBalance metric has had a value of 0 during nightly builds. The SysOps administrator needs to improve the performance and meet the sustained throughput requirements.

Which solution will meet these requirements MOST cost-effectively?

  • A. Double the gp2 EBS volume size from 500 GB to 1,000 GB.
  • B. Change the volume type from gp2 to General Purpose SSD (gp3).
  • C. Change the volume type from gp2 to Throughput Optimized HDD (st1).
  • D. Change the volume type from gp2 to Provisioned IOPS SSD (io2).

Selected Answer: B
B. Change the volume type from gp2 to General Purpose SSD (gp3). Explanation: The gp3 volume type is designed to provide baseline performance of 3,000 IOPS and a throughput of up to 125 MB/s. This aligns with the requirement to sustain 3,000 IOPS during nightly build tasks. gp3 volumes are a good fit for a wide range of transactional workloads and offer a more cost-effective solution compared to Provisioned IOPS SSD (io2) volumes.
