Amazon AWS 공인 SysOps 관리자 – 어소시에이트 의 시험 문제
질문 #: 315
주제 #: 1

회사는 EC2 인스턴스 중에 비프로덕션 Amazon EC2 인스턴스를 중지하기로 결정했습니다. 회사의 IT 관리자는 환경 유형 태그 값이 비프로덕션인 EC2 인스턴스가 밤에 시작될 때마다 거의 실시간으로 알림을 받아야 합니다.

가장 효율적인 운영 효율성으로 이 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. SMTP 클라이언트 라이브러리를 사용하여 AWS Lambda 함수를 구성합니다. EC2 인스턴스가 실행 상태일 때마다 알림을 받으려면 AWS 상태 대시보드에 Lambda 함수를 구독하세요. Amazon Pinpoint를 사용하여 IT 관리자에게 이메일 알림을 보내도록 Lambda 함수를 구성합니다. 낮 동안 첫 번째 Lambda 함수의 호출을 조절하기 위해 두 번째 Lambda 함수를 배포합니다.
  • B. Amazon EC2 API를 쿼리하여 각 EC2 인스턴스의 상태를 확인하는 AWS Lambda 함수를 배포합니다. EC2 인스턴스 스케줄러를 사용하여 야간에 1분마다 실행되도록 Lambda 함수를 구성하고 실행 상태에 있는 각 비프로덕션 EC2 인스턴스에 대해 IT 관리자에게 이메일 알림을 보내도록 합니다.
  • C. EC2 인스턴스 상태 변경 알림 이벤트 유형을 포함하는 Amazon EventBridge 규칙을 생성합니다. 실행 상태만 캡처하도록 이벤트를 필터링합니다. 규칙의 대상으로 AWS Lambda 함수를 생성합니다. 환경 유형을 결정하기 위해 현재 시간과 EC2 인스턴스의 태그를 확인하도록 Lambda 함수를 구성합니다. 알림을 위한 Lambda 함수의 대상으로 Amazon Simple 알림 서비스(Amazon SNS) 주제를 생성합니다. IT 관리자의 이메일 주소로 SNS 주제를 구독하세요. 최다 투표
  • D. 환경 유형을 포함한 EC2 인스턴스 메타데이터를 Amazon DynamoDB 테이블에 저장합니다. EC2 인스턴스에 사용자 지정 애플리케이션을 배포합니다. 밤 동안 매분마다 DynamoDB 데이터를 폴링하고 Amazon EC2 API에 쿼리하여 각 인스턴스의 상태를 확인하도록 사용자 지정 애플리케이션을 구성합니다. 또한 실행 중인 각 비프로덕션 EC2 인스턴스에 대해 IT 관리자에게 이메일 알림을 보내도록 사용자 지정 애플리케이션을 구성합니다.

Exam question from Amazon's AWS Certified SysOps Administrator - Associate
Question #: 315
Topic #: 1

A company decides to stop non-production Amazon EC2 instances during the EC2 instances. The company's IT manager must receive notification in near real time whenever an EC2 instance that has an environment type tag value of non-production is started during the night.

Which solution will meet this requirement with the MOST operational efficiency?

  • A. Configure an AWS Lambda function with an SMTP client library. Subscribe the Lambda function to the AWS Health Dashboard to receive notification whenever an EC2 instance is in the running state. Configure the Lambda function to use Amazon Pinpoint to send email notifications to the IT manager. Deploy a second Lambda function to throttle calls from the first Lambda function during the daytime.
  • B. Deploy an AWS Lambda function that queries the Amazon EC2 API to determine the state of each EC2 instance. Use the EC2 instance scheduler to configure the Lambda function to run every minute during the night and to send an email notification to the IT manager for each non-production EC2 instance that is in the running state.
  • C. Create an Amazon EventBridge rule that includes the EC2 Instance State-change Notification event type. Filter the event to capture only the running state. Create an AWS Lambda function as a target of the rule. Configure the Lambda function to check the current time and the EC2 instances’ tags to determine the environment type. Create an Amazon Simple Notification Service (Amazon SNS) topic as a target of the Lambda function for notifications. Subscribe the IT manager's email address to the SNS topic. Most Voted
  • D. Store the EC2 instance metadata, including the environment type, in an Amazon DynamoDB table. Deploy a custom application to an EC2 instance. Configure the custom application to poll the DynamoDB data every minute during the night and to query the Amazon EC2 API to determine the state of each instance. Additionally, configure the custom application to send an email notification to the IT manager for each non-production EC2 instance that is in the running state.
