이러한 요구 사항을 충족하는 가장 운영 효율적인 솔루션은 무엇입니까?

  • A. StatusCheckFailed_System 지표에 대한 Amazon CloudWatch 경보를 생성하여 EC2 인스턴스를 중지하고 시작하는 AWS Lambda 함수를 호출합니다.
  • B. MySQL 다중 AZ DB 인스턴스용 Amazon RDS를 생성합니다. Amazon S3에 저장된 MySQL 기본 백업을 사용하여 데이터를 새 데이터베이스로 복원합니다. 웹 애플리케이션에서 연결 문자열을 업데이트합니다. 최다 투표
  • C. 읽기 전용 복제본을 사용하여 MySQL 단일 AZ DB 인스턴스용 Amazon RDS를 생성합니다. Amazon S3에 저장된 MySQL 기본 백업을 사용하여 데이터를 새 데이터베이스로 복원합니다. 웹 애플리케이션에서 연결 문자열 업데이트
  • D. Amazon Data Lifecycle Manager(Amazon DLM)를 사용하여 매시간 Amazon Elastic Block Store(Amazon EBS) 볼륨의 스냅샷을 찍습니다. EC2 인스턴스 오류가 발생하는 경우 스냅샷에서 EBS 볼륨을 복원합니다. 최다 투표

Question #: 330
Topic #: 1

A company has a web application with a database tier that consists of an Amazon EC2 instance that runs MySQL. A SysOps administrator needs to minimize potential data loss and the time that is required to recover in the event of a database failure.

What is the MOST operationally efficient solution that meets these requirements?

  • A. Create an Amazon CloudWatch alarm for the StatusCheckFailed_System metric to invoke an AWS Lambda function that stops and starts the EC2 instance.
  • B. Create an Amazon RDS for MySQL Multi-AZ DB instance. Use a MySQL native backup that is stored in Amazon S3 to restore the data to the new database. Update the connection string in the web application. Most Voted
  • C. Create an Amazon RDS for MySQL Single-AZ DB instance with a read replica. Use a MySQL native backup that is stored in Amazon S3 to restore the data to the new database. Update the connection string in the web application
  • D. Use Amazon Data Lifecycle Manager (Amazon DLM) to take a snapshot of the Amazon Elastic Block Store (Amazon EBS) volume every hour. In the event of an EC2 instance failure, restore the EBS volume from a snapshot. Most Voted
