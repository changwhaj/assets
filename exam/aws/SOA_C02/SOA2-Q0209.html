Amazon AWS 공인 SysOps 관리자 – 어소시에이트 의 시험 문제
질문 #: 209
주제 #: 1

회사는 AWS에 새로운 워크로드를 배포해야 합니다. 회사는 모든 저장 데이터를 암호화해야 하며 매년 한 번씩 암호화 키를 교체해야 합니다. 워크로드는 데이터 저장을 위해 MySQL 다중 AZ 데이터베이스용 Amazon RDS를 사용합니다.

이러한 요구 사항을 충족하는 구성 접근 방식은 무엇입니까?

  • A. MySQL 구성 파일에서 TDE(투명한 데이터 암호화)를 활성화합니다. 12개월마다 키를 수동으로 순환합니다.
  • B. Amazon RDS용 AWS 관리형 키를 사용하여 생성 시 데이터베이스에서 RDS 암호화를 활성화합니다.
  • C. 새로운 AWS Key Management Service(AWS KMS) 고객 관리형 키를 생성합니다. 자동 키 순환을 활성화합니다. KMS 키를 사용하여 생성 시 데이터베이스에서 RDS 암호화를 활성화합니다. 최다 투표
  • D. 새로운 AWS Key Management Service(AWS KMS) 고객 관리형 키를 생성합니다. 자동 키 순환을 활성화합니다. RDS DB 인스턴스에 연결된 Amazon Elastic Block Store(Amazon EBS) 볼륨에서 암호화를 활성화합니다.

A company needs to deploy a new workload on AWS. The company must encrypt all data at rest and must rotate the encryption keys once each year. The workload uses an Amazon RDS for MySQL Multi-AZ database for data storage.

Which configuration approach will meet these requirements?

  • A. Enable Transparent Data Encryption (TDE) in the MySQL configuration file. Manually rotate the key every 12 months.
  • B. Enable RDS encryption on the database at creation time by using the AWS managed key for Amazon RDS.
  • C. Create a new AWS Key Management Service (AWS KMS) customer managed key. Enable automatic key rotation. Enable RDS encryption on the database at creation time by using the KMS key. Most Voted
  • D. Create a new AWS Key Management Service (AWS KMS) customer managed key. Enable automatic key rotation. Enable encryption on the Amazon Elastic Block Store (Amazon EBS) volumes that are attached to the RDS DB instance.

Automatic key rotation is supported only on symmetric encryption KMS keys. You cannot enable automatic rotation of asymmetric KMS keys, HMAC KMS keys, KMS keys with imported key material, or KMS keys in a custom key store. To enable or disable automatic rotation of a set of related multi-Region keys, set the property on the primary key.
