Amazon AWS 공인 SysOps 관리자 – 어소시에이트 의 시험 문제
질문 #: 114
주제 #: 1

SysOps 관리자에게는 암호화된 Amazon 머신 이미지(AMI)를 배포하는 데 사용되는 AWS CloudFormation 템플릿이 있습니다. CloudFormation 템플릿은 두 번째 계정에서 사용되므로 SysOps 관리자는 암호화된 AMI를 두 번째 계정에 복사합니다. 두 번째 계정에서 새 CloudFormation 스택을 시작하면 실패합니다.
문제를 해결하려면 SysOps 관리자가 어떤 조치를 취해야 합니까?

  • A. AMI를 공개로 표시하도록 AMI 권한을 변경합니다.
  • B. 원본 계정에서 AMI 등록을 취소합니다.
  • C. 대상 계정의 AWS Key Management Service(AWS KMS) 키를 사용하여 대상 AMI를 다시 암호화합니다. 최다 투표
  • D. 대상 계정의 AMI ID로 CloudFormation 템플릿을 업데이트합니다.

Question #: 114
Topic #: 1

A SysOps administrator has an AWS CloudFormation template that is used to deploy an encrypted Amazon Machine Image (AMI). The CloudFormation template will be used in a second account so the SysOps administrator copies the encrypted AMI to the second account. When launching the new CloudFormation stack in the second account, it fails.
Which action should the SysOps administrator take to correct the issue?

  • A. Change the AMI permissions to mark the AMI as public.
  • B. Deregister the AMI in the source account.
  • C. Re-encrypt the destination AMI with an AWS Key Management Service (AWS KMS) key from the destination account. Most Voted
  • D. Update the CloudFormation template with the ID of the AMI in the destination account.
