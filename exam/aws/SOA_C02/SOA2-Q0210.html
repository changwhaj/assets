질문 #: 210
주제 #: 1

회사에는 활성-수동 구성으로 두 개의 AWS 지역에 배포된 애플리케이션이 있습니다. 애플리케이션은 각 지역의 ALB(Application Load Balancer) 뒤의 Amazon EC2 인스턴스에서 실행됩니다. 인스턴스는 각 지역의 Amazon EC2 Auto Scaling 그룹에 있습니다. 애플리케이션은 DNS에 Amazon Route 53 호스팅 영역을 사용합니다. SysOps 관리자는 보조 지역에 대한 자동 장애 조치를 구성해야 합니다.

SysOps 관리자는 이러한 요구 사항을 충족하기 위해 무엇을 해야 합니까?

  • A. 각 ALB를 가리키는 Route 53 별칭 레코드를 구성합니다. 장애 조치 라우팅 정책을 선택합니다. 대상 상태 평가를 예로 설정합니다. 최다 투표
  • B. 각 AL을 가리키는 CNAME 레코드를 구성합니다. 장애 조치 라우팅 정책을 선택합니다. 대상 상태 평가를 예로 설정합니다.
  • C. Auto Scaling 그룹에 대한 Elastic Load Balancing(ELB) 상태 확인을 구성합니다. 기본 지역의 ALB에 대상 그룹을 추가합니다. 보조 리전에 EC2 인스턴스를 대상으로 포함합니다.
  • D. Auto Scaling 그룹에 대한 EC2 상태 확인을 구성합니다. 기본 지역의 ALB에 대상 그룹을 추가합니다. 보조 리전에 EC2 인스턴스를 대상으로 포함합니다.

Topic #: 1

A company has an application that is deployed to two AWS Regions in an active-passive configuration. The application runs on Amazon EC2 instances behind an Application Load Balancer (ALB) in each Region. The instances are in an Amazon EC2 Auto Scaling group in each Region. The application uses an Amazon Route 53 hosted zone for DNS. A SysOps administrator needs to configure automatic failover to the secondary Region.

What should the SysOps administrator do to meet these requirements?

  • A. Configure Route 53 alias records that point to each ALB. Choose a failover routing policy. Set Evaluate Target Health to Yes. Most Voted
  • B. Configure CNAME records that point to each ALChoose a failover routing policy. Set Evaluate Target Health to Yes.
  • C. Configure Elastic Load Balancing (ELB) health checks for the Auto Scaling group. Add a target group to the ALB in the primary Region. Include the EC2 instances in the secondary Region as targets.
  • D. Configure EC2 health checks for the Auto Scaling group. Add a target group to the ALB in the primary Region. Include the EC2 instances in the secondary Region as targets.

To configure automatic failover to the secondary Region for an application that is deployed to two AWS Regions in an active-passive configuration, the following steps should be taken: Configure Route 53 alias records that point to each ALB in the two Regions. Choose a failover routing policy, such as Failover or Geolocation. Set Evaluate Target Health to Yes to ensure that Route 53 only responds to DNS queries with healthy ALB endpoints.
