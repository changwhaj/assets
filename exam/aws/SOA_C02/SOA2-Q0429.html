주제 #: 1

회사는 AWS Organizations를 사용하여 여러 AWS 계정에 걸쳐 여러 애플리케이션을 호스팅합니다. 여러 팀이 AWS 계정 전반에 걸쳐 애플리케이션 인프라를 구축하고 유지 관리하는 일을 담당합니다.

SysOps 관리자는 사용자 계정과 권한이 중앙에서 관리되도록 하는 솔루션을 구현해야 합니다. 솔루션은 회사의 기존 온프레미스 Active Directory 환경과 통합되어야 합니다. SysOps 관리자는 이미 AWS IAM Identity Center(AWS Single Sign-On)를 활성화했으며 AWS Direct Connect 연결을 설정했습니다.

이러한 요구 사항을 충족하는 가장 운영 효율적인 솔루션은 무엇입니까?

  • A. Simple AD 도메인을 생성하고 온프레미스 Active Directory 도메인과 포리스트 신뢰 관계를 설정합니다. Simple AD 도메인을 IAM ID 센터의 ID 소스로 설정합니다. 필요한 역할 기반 권한 집합을 만듭니다. 각 사용자 그룹을 해당 그룹이 관리할 AWS 계정에 할당합니다.
  • B. 온프레미스 Active Directory 도메인에 조인된 Amazon EC2 인스턴스에 Active Directory 도메인 컨트롤러를 생성합니다. Active Directory 도메인 컨트롤러를 IAM ID 센터의 ID 소스로 설정합니다. 필요한 역할 기반 권한 집합을 만듭니다. 각 사용자 그룹을 해당 그룹이 관리할 AWS 계정에 할당합니다.
  • C. 온프레미스 Active Directory 도메인과 연결된 AD 커넥터를 생성합니다. AD 커넥터를 IAM ID 센터의 ID 소스로 설정합니다. 필요한 역할 기반 권한 집합을 만듭니다. 각 사용자 그룹을 해당 그룹이 관리할 AWS 계정에 할당합니다. 최다 투표
  • D. 내장된 SSO 디렉터리를 IAM ID 센터의 ID 소스로 사용합니다. 온프레미스 Active Directory 도메인에서 사용자 및 그룹을 복사합니다. 필요한 역할 기반 권한 집합을 만듭니다. 각 사용자 그룹을 해당 그룹이 관리할 AWS 계정에 할당합니다.

Topic #: 1

A company uses AWS Organizations to host several applications across multiple AWS accounts. Several teams are responsible for building and maintaining the infrastructure of the applications across the AWS accounts.

A SysOps administrator must implement a solution to ensure that user accounts and permissions are centrally managed. The solution must be integrated with the company's existing on-premises Active Directory environment. The SysOps administrator already has enabled AWS IAM Identity Center (AWS Single Sign-On) and has set up an AWS Direct Connect connection.

What is the MOST operationally efficient solution that meets these requirements?

  • A. Create a Simple AD domain, and establish a forest trust relationship with the on-premises Active Directory domain. Set the Simple AD domain as the identity source for IAM Identity Center. Create the required role-based permission sets. Assign each group of users to the AWS accounts that the group will manage.
  • B. Create an Active Directory domain controller on an Amazon EC2 instance that is joined to the on-premises Active Directory domain. Set the Active Directory domain controller as the identity source for IAM Identity Center. Create the required role-based permission sets. Assign each group of users to the AWS accounts that the group will manage.
  • C. Create an AD Connector that is associated with the on-premises Active Directory domain. Set the AD Connector as the identity source for IAM Identity Center. Create the required role-based permission sets. Assign each group of users to the AWS accounts that the group will manage. Most Voted
  • D. Use the built-in SSO directory as the identity source for IAM Identity Center. Copy the users and groups from the on-premises Active Directory domain. Create the required role-based permission sets. Assign each group of users to the AWS accounts that the group will manage.

Here's why this option is the most efficient: Centralized Management: An AD Connector provides a centralized way to manage user identities from the existing on-premises Active Directory, reducing the need to create and manage separate user accounts in AWS. Reduced Operational Overhead: Compared to managing a separate AD domain controller on an EC2 instance (Option B) or copying users/groups to the built-in directory (Option D), using an AD Connector simplifies administration. Existing Integration: Since AWS Direct Connect is already established, connecting the on-premises AD through an AD Connector leverages the existing network connectivity.
