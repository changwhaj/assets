Amazon AWS 공인 SysOps 관리자 – 어소시에이트 의 시험 문제
질문 #: 183
주제 #: 1

회사에는 어떤 보안 그룹도 SSH 포트가 모든 IP 주소에 열려 있도록 허용할 수 없다는 규정 준수 요구 사항이 있습니다. SysOps 관리자는 보안 그룹 규칙이 이 요구 사항을 위반할 때 회사의 SysOps 팀에 알리는 솔루션을 구현해야 합니다. 또한 솔루션은 보안 그룹 규칙을 자동으로 수정해야 합니다.

어떤 솔루션이 이러한 요구 사항을 충족합니까?

  • A. 보안 그룹이 변경될 때 AWS Lambda 함수를 호출하는 Amazon EventBridge(Amazon CloudWatch Events) 규칙을 생성합니다. 보안 그룹의 규정 준수를 평가하고, 모든 포트에서 모든 인바운드 보안 그룹 규칙을 제거하고, 보안 그룹이 규정을 준수하지 않는 경우 SysOps 팀에 알리도록 Lambda 함수를 구성합니다.
  • B. 보안 그룹 변경 사항에 대한 AWS CloudTrail 지표 필터를 생성합니다. 지표가 0보다 클 때 Amazon Simple Notification Service(Amazon SNS) 주제를 통해 SysOps 팀에 알리는 Amazon CloudWatch 경보를 생성합니다. 규칙을 제거하여 보안 그룹 규칙을 수정하려면 SNS 주제에 대한 AWS Lambda 함수를 구독하세요.
  • C. AWS Config 제한된 SSH 관리형 규칙을 활성화합니다. AWS 시스템 관리자 자동화 AWS-DisablePublicAccessForSecurityGroup Runbook을 사용하여 AWS Config 규칙에 자동 문제 해결을 추가합니다. 규칙이 규정을 준수하지 않는 경우 SysOps 팀에 알리는 Amazon EventBridge(Amazon CloudWatch Events) 규칙을 생성합니다. 최다 투표
  • D. 보안 그룹 변경 사항에 대한 AWS CloudTrail 지표 필터를 생성합니다. 지표가 0보다 큰 경우에 대한 Amazon CloudWatch 경보를 생성합니다. 경보가 ALARM 상태일 때 시스템 관리자 자동화 AWS-DisablePublicAccessForSecurityGroup Runbook을 사용하여 보안 그룹을 일시 중단하려면 CloudWatch 경보에 AWS 시스템 관리자 작업을 추가합니다. SysOps 팀에 알리려면 Amazon Simple 알림 서비스(Amazon SNS) 주제를 두 번째 대상으로 추가하세요.

Exam question from Amazon's AWS Certified SysOps Administrator - Associate
Question #: 183
Topic #: 1

A company has a compliance requirement that no security groups can allow SSH ports to be open to all IP addresses. A SysOps administrator must implement a solution that will notify the company’s SysOps team when a security group rule violates this requirement. The solution also must remediate the security group rule automatically.

Which solution will meet these requirements?

  • A. Create an Amazon EventBridge (Amazon CloudWatch Events) rule that invokes an AWS Lambda function when a security group changes. Configure the Lambda function to evaluate the security group for compliance, remove all inbound security group rules on all ports, and notify the SysOps team if the security group is noncompliant.
  • B. Create an AWS CloudTrail metric filter for security group changes. Create an Amazon CloudWatch alarm to notify the SysOps team through an Amazon Simple Notification Service (Amazon SNS) topic when the metric is greater than 0. Subscribe an AWS Lambda function to the SNS topic to remediate the security group rule by removing the rule.
  • C. Activate the AWS Config restricted-ssh managed rule. Add automatic remediation to the AWS Config rule by using the AWS Systems Manager Automation AWS-DisablePublicAccessForSecurityGroup runbook. Create an Amazon EventBridge (Amazon CloudWatch Events) rule to notify the SysOps team when the rule is noncompliant. Most Voted
  • D. Create an AWS CloudTrail metric filter for security group changes. Create an Amazon CloudWatch alarm for when the metric is greater than 0. Add an AWS Systems Manager action to the CloudWatch alarm to suspend the security group by using the Systems Manager Automation AWS-DisablePublicAccessForSecurityGroup runbook when the alarm is in ALARM state. Add an Amazon Simple Notification Service (Amazon SNS) topic as a second target to notify the SysOps team.
