Amazon AWS 공인 SysOps 관리자 – 어소시에이트 의 시험 문제
질문 #: 414
주제 #: 1

한 회사에 50개의 AWS 계정이 있고 각 계정에 동일한 Amazon VPC를 생성하려고 합니다. 회사가 향후 VPC에 적용하는 모든 변경 사항은 모든 VPC에서 구현되어야 합니다.

각 계정에서 VPC를 배포하고 업데이트하는 가장 운영상 효율적인 방법은 무엇입니까?

  • A. VPC를 정의하는 AWS CloudFormation 템플릿을 생성합니다. 각 계정에서 AWS Management Console에 로그인합니다. 템플릿에서 스택을 생성합니다.
  • B. AWS CLI를 사용하여 VPC를 구성하는 셸 스크립트를 생성합니다. 텍스트 파일에서 셸 스크립트에 계정 목록을 제공합니다. 목록의 모든 계정에 VPC를 생성합니다.
  • C. Amazon DynamoDB에 계정 정보를 VPStore로 구성하는 AWS Lambda 함수를 생성합니다. DynamoDB 테이블에 대한 Lambda 액세스 권한을 부여합니다. 목록의 모든 계정에 VPC를 생성합니다.
  • D. VPC를 정의하는 AWS CloudFormation 템플릿을 생성합니다. 템플릿을 기반으로 AWS CloudFormation StackSet을 생성합니다. 스택 세트를 사용하여 모든 계정에 템플릿을 배포합니다. 최다 투표

Topic #: 1

A company has 50 AWS accounts and wants to create an identical Amazon VPC in each account. Any changes the company makes to the VPCs in the future must be implemented on every VPC.

What is the MOST operationally efficient method to deploy and update the VPCs in each account?

  • A. Create an AWS CloudFormation template that defines the VPC. Sign in to the AWS Management Console under each account. Create a stack from the template.
  • B. Create a shell script that configures the VPC using the AWS CLI. Provide a list of accounts to the shell script from a text file. Create the VPC in every account in the list.
  • C. Create an AWS Lambda function that configures the VPStore the account information in Amazon DynamoDB. Grant Lambda access to the DynamoDB table. Create the VPC in every account in the list.
  • D. Create an AWS CloudFormation template that defines the VPC. Create an AWS CloudFormation StackSet based on the template. Deploy the template to all accounts using the stack set. Most Voted
