주제 #: 1

한 회사에 수천 명의 사용자가 있는 팟캐스트 플랫폼이 있습니다. 회사는 팟캐스트 청취, 일시 중지, 종료 등 사용자 이벤트의 10분 실행 기간을 기반으로 낮은 팟캐스트 참여를 감지하는 이상 감지 알고리즘을 구현했습니다. 기계 학습(ML) 전문가는 추론 전에 이벤트 페이로드에 약간의 작은 변환이 필요하다는 사실을 알고 이러한 이벤트의 데이터 수집을 설계하고 있습니다.

ML 전문가는 최소한의 운영 오버헤드로 이러한 요구 사항을 충족하기 위해 데이터 수집을 어떻게 설계해야 합니까?

  • A. AWS AppSync에서 GraphQLAPI를 사용하여 이벤트 데이터를 수집합니다. Amazon DynamoDB 테이블에 데이터를 저장합니다. DynamoDB 스트림을 사용하여 AWS Lambda 함수를 호출하여 추론 전 가장 최근 10분 동안의 데이터를 변환합니다.
  • B. Amazon Kinesis Data Streams를 사용하여 이벤트 데이터를 수집합니다. Amazon Kinesis Data Firehose를 사용하여 Amazon S3에 데이터를 저장합니다. AWS Glue를 사용하여 추론 전 가장 최근 10분 동안의 데이터를 변환합니다. 최다 투표
  • C. Amazon Kinesis Data Streams를 사용하여 이벤트 데이터를 수집합니다. Apache Flink용 Amazon Kinesis Data Analytics 애플리케이션을 사용하여 추론하기 전에 가장 최근 10분 동안의 데이터를 변환합니다. 최다 투표
  • D. Amazon Managed Streaming for Apache Kafka(Amazon MSK)를 사용하여 이벤트 데이터를 수집합니다. AWS Lambda 함수를 사용하여 추론 전에 가장 최근 10분 동안의 데이터를 변환합니다.

Exam question from Amazon's AWS Certified Machine Learning - Specialty
Question #: 197
Topic #: 1

A company has a podcast platform that has thousands of users. The company has implemented an anomaly detection algorithm to detect low podcast engagement based on a 10-minute running window of user events such as listening, pausing, and exiting the podcast. A machine learning (ML) specialist is designing the data ingestion of these events with the knowledge that the event payload needs some small transformations before inference.

How should the ML specialist design the data ingestion to meet these requirements with the LEAST operational overhead?

  • A. Ingest event data by using a GraphQLAPI in AWS AppSync. Store the data in an Amazon DynamoDB table. Use DynamoDB Streams to call an AWS Lambda function to transform the most recent 10 minutes of data before inference.
  • B. Ingest event data by using Amazon Kinesis Data Streams. Store the data in Amazon S3 by using Amazon Kinesis Data Firehose. Use AWS Glue to transform the most recent 10 minutes of data before inference. Most Voted
  • C. Ingest event data by using Amazon Kinesis Data Streams. Use an Amazon Kinesis Data Analytics for Apache Flink application to transform the most recent 10 minutes of data before inference. Most Voted
  • D. Ingest event data by using Amazon Managed Streaming for Apache Kafka (Amazon MSK). Use an AWS Lambda function to transform the most recent 10 minutes of data before inference.
