금융 서비스 회사는 Amazon SageMaker를 기본 데이터 과학 환경으로 채택하려고 합니다. 회사의 데이터 과학자들은
기밀 재무 데이터에 대해 기계 학습(ML) 모델을 실행합니다. 회사는 데이터 유출을 걱정하고 있으며 ML 엔지니어가 환경을 보호하기를 원합니다.
ML 엔지니어는 SageMaker의 데이터 송신을 제어하기 위해 어떤 메커니즘을 사용할 수 있습니까? (3개를 선택하세요.)

  • A. AWS PrivateLink에서 제공하는 VPC 인터페이스 엔드포인트를 사용하여 SageMaker에 연결합니다. 최다 투표
  • B. SCP를 사용하여 SageMaker에 대한 액세스를 제한합니다.
  • C. SageMaker 노트북 인스턴스에서 루트 액세스를 비활성화합니다.
  • D. 훈련 작업 및 모델에 대해 네트워크 격리를 활성화합니다. 최다 투표
  • E. 노트북에 미리 서명된 URL을 회사에서 사용하는 특정 IP로 제한합니다. 최다 투표
  • F. 저장 및 전송 중인 데이터를 암호화하여 보호합니다. AWS Key Management Service(AWS KMS)를 사용하여 암호화 키를 관리합니다.

Exam question from Amazon's AWS Certified Machine Learning - Specialty
Question #: 131
Topic #: 1

A financial services company wants to adopt Amazon SageMaker as its default data science environment. The company's data scientists run machine learning
(ML) models on confidential financial data. The company is worried about data egress and wants an ML engineer to secure the environment.
Which mechanisms can the ML engineer use to control data egress from SageMaker? (Choose three.)

  • A. Connect to SageMaker by using a VPC interface endpoint powered by AWS PrivateLink. Most Voted
  • B. Use SCPs to restrict access to SageMaker.
  • C. Disable root access on the SageMaker notebook instances.
  • D. Enable network isolation for training jobs and models. Most Voted
  • E. Restrict notebook presigned URLs to specific IPs used by the company. Most Voted
  • F. Protect data with encryption at rest and in transit. Use AWS Key Management Service (AWS KMS) to manage encryption keys.
