데이터 과학자는 기존 온프레미스 ETL 프로세스를 클라우드로 마이그레이션해야 합니다. 현재 프로세스는 정기적인 시간 간격으로 실행되며 PySpark를 사용하여 여러 대규모 데이터 소스를 다운스트림 처리를 위한 단일 통합 출력으로 결합하고 형식을 지정합니다.
데이터 과학자에게는 클라우드 솔루션에 대한 다음 요구 사항이 주어졌습니다.
✑ 여러 데이터 소스를 결합합니다.
✑ 기존 PySpark 논리를 재사용합니다.
✑ 기존 일정에 따라 솔루션을 실행합니다.
✑ 관리해야 할 서버 수를 최소화합니다.
데이터 과학자는 이 솔루션을 구축하기 위해 어떤 아키텍처를 사용해야 합니까?

  • A. Amazon S3에 원시 데이터를 씁니다. 기존 일정에 따라 영구 Amazon EMR 클러스터에 Spark 단계를 제출하도록 AWS Lambda 함수를 예약합니다. 기존 PySpark 로직을 사용하여 EMR 클러스터에서 ETL 작업을 실행합니다. 다운스트림 사용이 가능한 Amazon S3의 "처리된" 위치에 결과를 출력합니다.
  • B. Amazon S3에 원시 데이터를 씁니다. 입력 데이터에 대해 ETL 처리를 수행하는 AWS Glue ETL 작업을 생성합니다. 기존 로직을 활용하려면 PySpark에서 ETL 작업을 작성하세요. 기존 일정에 따라 ETL 작업을 트리거하는 새로운 AWS Glue 트리거를 생성합니다. 다운스트림 사용을 위해 액세스할 수 있는 Amazon S3의 "처리된" 위치에 쓰도록 ETL 작업의 출력 대상을 구성합니다. 최다 투표
  • C. Amazon S3에 원시 데이터를 씁니다. 기존 일정에 따라 실행되도록 AWS Lambda 함수를 예약하고 Amazon S3의 입력 데이터를 처리합니다. Python으로 Lambda 논리를 작성하고 기존 PySpark 논리를 구현하여 ETL 프로세스를 수행합니다. Lambda 함수가 다운스트림 사용이 가능한 Amazon S3의 "처리된" 위치에 결과를 출력하도록 합니다.
  • D. Amazon Kinesis Data Analytics를 사용하여 입력 데이터를 스트리밍하고 스트림에 대해 실시간 SQL 쿼리를 수행하여 스트림 내에서 필요한 변환을 수행합니다. 다운스트림 사용이 가능한 Amazon S3의 "처리된" 위치로 출력 결과를 전달합니다.

A Data Scientist needs to migrate an existing on-premises ETL process to the cloud. The current process runs at regular time intervals and uses PySpark to combine and format multiple large data sources into a single consolidated output for downstream processing.
The Data Scientist has been given the following requirements to the cloud solution:
✑ Combine multiple data sources.
✑ Reuse existing PySpark logic.
✑ Run the solution on the existing schedule.
✑ Minimize the number of servers that will need to be managed.
Which architecture should the Data Scientist use to build this solution?

  • A. Write the raw data to Amazon S3. Schedule an AWS Lambda function to submit a Spark step to a persistent Amazon EMR cluster based on the existing schedule. Use the existing PySpark logic to run the ETL job on the EMR cluster. Output the results to a ג€processedג€ location in Amazon S3 that is accessible for downstream use.
  • B. Write the raw data to Amazon S3. Create an AWS Glue ETL job to perform the ETL processing against the input data. Write the ETL job in PySpark to leverage the existing logic. Create a new AWS Glue trigger to trigger the ETL job based on the existing schedule. Configure the output target of the ETL job to write to a ג€processedג€ location in Amazon S3 that is accessible for downstream use. Most Voted
  • C. Write the raw data to Amazon S3. Schedule an AWS Lambda function to run on the existing schedule and process the input data from Amazon S3. Write the Lambda logic in Python and implement the existing PySpark logic to perform the ETL process. Have the Lambda function output the results to a ג€processedג€ location in Amazon S3 that is accessible for downstream use.
  • D. Use Amazon Kinesis Data Analytics to stream the input data and perform real-time SQL queries against the stream to carry out the required transformations within the stream. Deliver the output results to a ג€processedג€ location in Amazon S3 that is accessible for downstream use.
