주제 #: 1

한 전자상거래 회사가 웹사이트, 모바일 앱, IoT 장치에서 정형 데이터와 비정형 데이터를 수집하고 있습니다. 데이터는 여러 데이터베이스와 Amazon S3 버킷에 저장됩니다. 회사는 구조화된 데이터와 구조화되지 않은 데이터를 저장하기 위해 확장 가능한 저장소를 구현하고 있습니다. 회사는 중앙 데이터 카탈로그, 데이터에 대한 셀프 서비스 액세스, 데이터 보호를 위한 세분화된 데이터 액세스 정책 및 암호화를 제공하는 솔루션을 구현해야 합니다.

최소한의 설정으로 이러한 요구 사항을 충족하는 작업 조합은 무엇입니까? (3개를 선택하세요.)

  • A. 데이터베이스와 S3 버킷의 기존 데이터를 식별합니다. 데이터를 AWS Lake Formation에 연결합니다. 최다 투표
  • B. 데이터베이스와 S3 버킷에 있는 기존 데이터를 식별합니다. 데이터를 AWS Glue에 연결합니다.
  • C. 연결된 데이터 소스에서 AWS Glue 크롤러를 실행하여 중앙 데이터 카탈로그를 생성합니다. 최다 투표
  • D. AWS Identity and Access Management(1AM)를 사용하여 세분화된 액세스 정책을 적용합니다. 각 데이터 원본에 대해 서버 측 암호화를 구성합니다.
  • E. AWS Lake Formation을 사용하여 세분화된 액세스 정책 및 암호화를 적용합니다. 최다 투표
  • F. AWS Glue를 사용하여 세분화된 액세스 정책 및 암호화를 적용합니다.

Topic #: 1

An ecommerce company is collecting structured data and unstructured data from its website, mobile apps, and IoT devices. The data is stored in several databases and Amazon S3 buckets. The company is implementing a scalable repository to store structured data and unstructured data. The company must implement a solution that provides a central data catalog, self-service access to the data, and granular data access policies and encryption to protect the data.

Which combination of actions will meet these requirements with the LEAST amount of setup? (Choose three.)

  • A. Identify the existing data in the databases and S3 buckets. Link the data to AWS Lake Formation. Most Voted
  • B. Identify the existing data in the databases and S3 buckets. Link the data to AWS Glue.
  • C. Run AWS Glue crawlers on the linked data sources to create a central data catalog. Most Voted
  • D. Apply granular access policies by using AWS Identity and Access Management (1AM). Configure server-side encryption on each data source.
  • E. Apply granular access policies and encryption by using AWS Lake Formation. Most Voted
  • F. Apply granular access policies and encryption by using AWS Glue.

Lake Formation provides a single place to manage access controls for data in your data lake. You can define security policies that restrict access to data at the database, table, column, row, and cell levels. These policies apply to IAM users and roles, and to users and groups when federating through an external identity provider. You can use fine-grained controls to access data secured by Lake Formation within Amazon Redshift Spectrum, Athena, AWS Glue ETL, and Amazon EMR for Apache Spark. Whenever you create IAM identities, make sure to follow IAM best practices.
