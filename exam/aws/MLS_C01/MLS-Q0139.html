개발자는 앱의 미리 보기 기능을 구독한 제한된 수의 사용자를 위해 새 버전의 모델을 도입하려고 합니다. 새 버전의 모델이 미리보기로 테스트된 후 개발자는 모델의 정확성을 평가합니다. 모델의 새 버전의 정확도가 더 높으면 개발자는 일정 기간 동안 모든 사용자를 위해 새 버전을 점진적으로 출시할 수 있어야 합니다.
회사는 어떻게 운영 오버헤드를 최소화하면서 테스트 모델을 구현할 수 있습니까?

  • A. InitialVariantWeight 매개 변수를 0으로 설정한 CreateEndpointConfig 작업을 사용하여 ProductionVariant 데이터 형식을 모델의 새 버전으로 업데이트합니다. 미리 보기 기능을 구독한 사용자에 대해 InvokeEndpoint 호출에 대해 TargetVariant 매개 변수를 지정합니다. 모델의 새 버전이 출시될 준비가 되면 모든 사용자가 업데이트된 버전을 가질 때까지 InitialVariantWeight를 점차적으로 늘립니다. 최다 투표
  • B. 다양한 버전의 모델을 제공하는 두 개의 SageMaker 호스팅 엔드포인트를 구성합니다. TargetVariant 쿼리 문자열 매개변수를 기반으로 두 끝점 모두에 트래픽을 라우팅하는 ALB(Application Load Balancer)를 생성합니다. 미리 보기 기능을 구독한 사용자에게 TargetVariant 쿼리 문자열 매개 변수를 보내도록 앱을 재구성합니다. 모델의 새 버전이 출시될 준비가 되면 모든 사용자가 업데이트된 버전을 가질 때까지 ALB의 라우팅 알고리즘을 가중치 적용으로 변경합니다.
  • C. DesiredWeight 매개변수가 0으로 설정된 UpdateEndpointWeightsAndCapacities 작업을 사용하여 DesiredWeightsAndCapacity 데이터 유형을 모델의 새 버전으로 업데이트합니다. 미리 보기 기능을 구독한 사용자에 대해 InvokeEndpoint 호출에 대해 TargetVariant 매개변수를 지정합니다. 모델의 새 버전이 출시될 준비가 되면 모든 사용자가 업데이트된 버전을 가질 때까지 DesiredWeight를 점차적으로 늘립니다. 최다 투표
  • D. 다양한 버전의 모델을 제공하는 두 개의 SageMaker 호스팅 엔드포인트를 구성합니다. 단순 라우팅 정책으로 구성되고 모델의 현재 버전을 가리키는 Amazon Route 53 레코드를 생성합니다. 미리 보기 기능을 구독한 사용자에게는 엔드포인트 URL을 사용하고 다른 사용자에게는 Route 53 레코드를 사용하도록 모바일 앱을 구성합니다. 모델의 새 버전이 출시될 준비가 되면 Route 53에 새 모델 버전 엔드포인트를 추가하고 모든 사용자가 업데이트된 버전을 가질 때까지 정책을 가중치 적용으로 전환합니다.

A telecommunications company is developing a mobile app for its customers. The company is using an Amazon SageMaker hosted endpoint for machine learning model inferences.
Developers want to introduce a new version of the model for a limited number of users who subscribed to a preview feature of the app. After the new version of the model is tested as a preview, developers will evaluate its accuracy. If a new version of the model has better accuracy, developers need to be able to gradually release the new version for all users over a fixed period of time.
How can the company implement the testing model with the LEAST amount of operational overhead?

  • A. Update the ProductionVariant data type with the new version of the model by using the CreateEndpointConfig operation with the InitialVariantWeight parameter set to 0. Specify the TargetVariant parameter for InvokeEndpoint calls for users who subscribed to the preview feature. When the new version of the model is ready for release, gradually increase InitialVariantWeight until all users have the updated version. Most Voted
  • B. Configure two SageMaker hosted endpoints that serve the different versions of the model. Create an Application Load Balancer (ALB) to route traffic to both endpoints based on the TargetVariant query string parameter. Reconfigure the app to send the TargetVariant query string parameter for users who subscribed to the preview feature. When the new version of the model is ready for release, change the ALB's routing algorithm to weighted until all users have the updated version.
  • C. Update the DesiredWeightsAndCapacity data type with the new version of the model by using the UpdateEndpointWeightsAndCapacities operation with the DesiredWeight parameter set to 0. Specify the TargetVariant parameter for InvokeEndpoint calls for users who subscribed to the preview feature. When the new version of the model is ready for release, gradually increase DesiredWeight until all users have the updated version. Most Voted
  • D. Configure two SageMaker hosted endpoints that serve the different versions of the model. Create an Amazon Route 53 record that is configured with a simple routing policy and that points to the current version of the model. Configure the mobile app to use the endpoint URL for users who subscribed to the preview feature and to use the Route 53 record for other users. When the new version of the model is ready for release, add a new model version endpoint to Route 53, and switch the policy to weighted until all users have the updated version.
