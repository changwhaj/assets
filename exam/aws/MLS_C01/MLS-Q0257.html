가장 효율적인 운영 효율성으로 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

  • A. Amazon SageMaker Feature Store를 사용하여 모델 훈련 및 추론을 위한 기능을 저장하십시오. 온라인 추론을 위한 온라인 상점을 만듭니다. 모델 학습을 위한 오프라인 매장을 만드세요. 데이터 과학자가 기능 그룹에 액세스하고 검색할 수 있는 IAM 역할을 만듭니다. 최다 투표
  • B. Amazon SageMaker Feature Store를 사용하여 모델 훈련 및 추론을 위한 기능을 저장합니다. 온라인 추론과 모델 교육을 모두 위한 온라인 상점을 만드세요. 데이터 과학자가 기능 그룹에 액세스하고 검색할 수 있는 IAM 역할을 만듭니다.
  • C. 온라인 추론 기능을 저장할 하나의 Amazon S3 버킷을 생성합니다. 오프라인 모델 훈련 기능을 저장할 두 번째 S3 버킷을 생성합니다. S3 버킷의 버전 관리를 활성화하고 태그를 사용하여 온라인 추론 기능용 태그와 오프라인 모델 교육 기능용 태그를 지정합니다. Amazon Athena를 사용하여 온라인 추론을 위해 S3 버킷을 쿼리합니다. 오프라인 모델 훈련을 위한 S3 버킷을 SageMaker 훈련 작업에 연결합니다. 데이터 과학자가 두 버킷 모두에 액세스할 수 있도록 허용하는 IAM 정책을 생성합니다.
  • D. 온라인 추론 기능과 오프라인 모델 교육 기능을 저장하기 위해 두 개의 별도 Amazon DynamoDB 테이블을 생성합니다. 두 테이블 모두에서 시간 기반 버전 관리를 사용합니다. 온라인 추론을 위해 DynamoDB 테이블을 쿼리합니다. 새로운 SageMaker 훈련 작업이 시작되면 DynamoDB에서 Amazon S3로 데이터를 이동합니다. 데이터 과학자가 두 테이블 모두에 액세스할 수 있도록 허용하는 IAM 정책을 생성합니다.

Which solution will meet these requirements with the MOST operational efficiency?

Turn on versioning for the S3 buckets and use tags to specify which tags are for online inference features and which are for offline model training features. Use Amazon Athena to query the S3 bucket for online inference. Connect the S3 bucket for offline model training to a SageMaker training job. Create an IAM policy that allows data scientists to access both buckets.
  • D. Create two separate Amazon DynamoDB tables to store online inference features and offline model training features. Use time-based versioning on both tables. Query the DynamoDB table for online inference. Move the data from DynamoDB to Amazon S3 when a new SageMaker training job is launched. Create an IAM policy that allows data scientists to access both tables.
